# 行動心理学の基礎

行動変容デザインの基盤となる心理学の原則と意思決定メカニズムをまとめる。

---

## 二重プロセス理論

人間の心は2つの異なるモードで動作する。

| システム | 特徴 | 動作 |
|---------|------|------|
| **システム1（直感の心理）** | 瞬間的・自動的・無意識 | 過去の経験・経験則・感情に基づく |
| **システム2（熟慮の心理）** | 遅い・集中・自覚的 | 論理的分析・複雑な問題解決 |

### システム1の制約

- 同時に保持できる情報は7±2個のチャンク
- 注意力は限られている（ゴリラを見落とす実験）
- 意志力は消耗する（選択疲れ）
- 満足化（Satisfice）する - 最適解ではなく「よさそう」で決める

### システムの相互作用

- システム2が考えていても、実際に行動するのはシステム1
- 象（システム1）と象使い（システム2）の比喩：象使いは常に自分が操縦していると思っているが、実際は象が動かしている
- 意識的な判断と直感的反応が一致しないことがある

## 意思決定の主要な学び

| 学び | 説明 | プロダクト設計への示唆 |
|------|------|---------------------|
| **ほとんど選択していない** | 生活の約半分は習慣・直感的反応に支配される | 意識的な説得だけでは行動は変わらない |
| **過去の経験が直感を形成** | 連想ネットワークによる瞬時の評価 | 初回体験での直感的評価を把握する |
| **習慣は予測可能** | キュー→ルーティン→リワードのループ | 習慣形成のメカニズムを活用する |
| **文脈に依存する** | マインドセットがその場の反応を左右する | ユーザーの心の状態を活性化する |
| **心のリソースは限られる** | 記憶・注意・意志・決断の容量は小さい | 認知負荷を最小化する |
| **安易な課題を探す** | 難しい問題を簡単な問題にすり替える | ショートカットを理解し活用する |
| **社会的手がかりに依存** | 他者の行動・専門家・規範を参照する | 社会的証明を組み込む |
| **セルフナラティブが行動を左右** | 自己概念が行動の道しるべになる | ユーザーの自己概念を構築する |

## 認知バイアスとヒューリスティクス

### 損失回避（Loss Aversion）

- 同じ金額でも、得るより失う方が心理的に2倍強く感じる
- 現状維持バイアス - 変化を避けようとする
- **活用例**: 「失うもの」をフレーミングで強調

### フレーミング効果（Framing Effect）

- 同じ情報でも提示方法で判断が変わる
- 「90%成功」vs「10%失敗」で印象が異なる
- **活用例**: ポジティブフレームで価値を伝える

### アンカリング（Anchoring）

- 最初に提示された数値が基準になる
- たとえ根拠がなくても強く影響する
- **活用例**: 価格比較・進捗表示の基準設定

### 社会的証明（Social Proof）

- 他者の行動を真似る傾向
- 「みんながやっている」が強い動機になる
- **活用例**: 「いいね」数・レビュー・専門家推薦

### 現在バイアス（Present Bias / 時間的近視眼）

- 今の快楽を過大評価、将来の報酬を過小評価
- 先延ばしの原因
- **活用例**: 即座のリワード設計・緊急性の創出

### デフォルト効果（Default Effect）

- デフォルト選択肢がそのまま選ばれやすい
- 401(k)自動登録で加入率2倍
- **活用例**: 望ましい選択をデフォルトに設定

### 希少性ヒューリスティック（Scarcity Heuristic）

- 手に入りづらいものほど価値があると感じる
- 「期間限定」「残り5個」が購買意欲を高める
- **活用例**: 限定性の演出（倫理的に）

### 利用可能性ヒューリスティック（Availability Heuristic）

- 思い出しやすいほど頻度・確率を高く見積もる
- 有名人の名前は多いと思われる
- **活用例**: 記憶に残るキュー設計

### 曖昧さ効果（Ambiguity Effect）

- 結果が不確実な選択肢を避ける
- 明確な見込みがある方を選ぶ
- **活用例**: 明確な期待値を提示する

### 注意バイアス（Attentional Bias）

- 内面の状態によって注意の向き方が変わる
- 依存症患者は依存対象に敏感
- **活用例**: キューのタイミング調整

## 習慣の心理メカニズム

### 習慣の定義

- **環境のキューによって引き起こされる繰り返し行動**
- 意識的制御の外で自動的に生じる
- 心の働きを「アウトソーシング」する仕組み

### 習慣形成の2つの方法

#### 1. 単純な繰り返し

```
キュー → ルーティン → 強化された連想 → 習慣
```

- 朝起きる（キュー）→ 同じ側からベッドを下りる（ルーティン）
- 繰り返すことで連想が強化される

#### 2. リワードによる形成

```
キュー → ルーティン → リワード → 渇望 → 習慣
```

**3要素の詳細**:

| 要素 | 説明 | 特性 |
|------|------|------|
| **キュー** | 環境または身体の信号 | 明快で曖昧さがないこと |
| **ルーティン** | 単純～複雑な一連の行動 | 意識が介入しないこと |
| **リワード** | 行動直後の報酬 | ランダム性も効果的 |

### リワードの特性

- **ルーティン直後が最適** - キューとの連想を強化
- **習慣形成後は不要** - 自動化される
- **渇望は持続** - リワードがなくなっても欲求は残る
- **時間がかかる** - 数週間～数ヶ月

### 欲望エンジン（Desire Engine）

キューを見るとリワードを予期し、ルーティンを実行したくなる心理メカニズム。

## 心のリソースの制約

### 記憶力の限界

- 短期記憶は7±2チャンクが限界
- 複雑な情報の同時処理は困難

### 注意力の限界

- 選択的注意 - 他のものを探すとゴリラを見落とす
- マルチタスクの困難さ

### 意志力の限界

- 精神的タスク・誘惑への抵抗で消耗
- 疲れると直感プロセスに依存
- 選択のたびに一時的に疲弊（選択疲れ）
- **意志力は養えるスキル**でもある

### 決断力の限界

- 最適解ではなく満足化（Satisfice）する
- 「最初のよさそうなもの」で決める
- 情報過多で選択できなくなる

## 基本的な6つの原則

| 原則 | 説明 | 理由 |
|------|------|------|
| **簡単なのがいい** | 心身の労力が少ない方が実行されやすい | チャネル因子・摩擦の克服 |
| **慣れているほうがいい** | 以前に見た・やったことを好む | 単純接触効果・自己効力感 |
| **きれいなのがいい** | 見た目がよいものを好む | ハロー効果・処理流暢性 |
| **リワードを繰り返したい** | 報酬体験を再現したくなる | 直感的反応・損益計算 |
| **失敗したくない** | 成功しなさそうなことは避ける | リワード欠如・社会的評価 |
| **急ぎのことをまずやる** | 緊急性が優先される | 生存本能・リソース配分 |

## プロダクトデザインへの示唆

### 直感と熟慮の両方に訴える

- **直感（システム1）**: 習慣・連想・ヒューリスティクスを活用
- **熟慮（システム2）**: 論理的説得・費用対効果の提示

### 認知負荷を最小化

- 一度に要求する情報を7個以内に
- デフォルトを賢く設定
- 選択肢を厳選する

### 社会的要素の組み込み

- 社会的証明（レビュー・いいね数）
- 専門家推薦
- ピア比較

### 心の状態をプライミング

- 適切なマインドセットを活性化
- セルフナラティブを構築
- ステレオタイプの影響を理解

### ヒューリスティクスの戦略的活用

- アンカリング効果を利用
- フレーミングで価値を伝える
- 損失回避を活用（倫理的に）

---

## まとめ

- 人間の行動はシステム1（直感）が大部分を支配している
- 過去の経験・習慣・社会的手がかりが行動を決める
- 認知バイアスとヒューリスティクスは予測可能なパターン
- 心のリソース（記憶・注意・意志・決断）は限られている
- これらの理解がプロダクトデザインの基礎となる

# 市場分析・フィードバック分析

Claude Codeを使った競合分析、市場調査、顧客フィードバック分析の体系的ワークフロー。

---

## 競合分析ワークフロー

### ワークスペース構成

```bash
mkdir -p research/competitors
mkdir -p research/pricing
mkdir -p research/market-sizing
```

### 競合プロファイル作成

プロンプト:
```
[競合名] をWebサーチで調査し、research/competitors/[competitor-name]-profile.md に構造化プロファイルを作成してください。

以下のセクションを含めてください:
- **概要**: 事業内容、対象顧客、企業ステージ（1段落）
- **プロダクト**: コア製品と主要機能（箇条書き）
- **価格モデル**: 課金方法、ティア構成、概算価格
- **ターゲット市場**: 企業規模、業界、ユーザーロール
- **主要差別化要因**: ポジショニングで強調している点（3-5項目）
- **強み**: 自社比での優位点
- **弱み**: ギャップや弱点（3-5項目）
- **最近の動向**: 直近6ヶ月のリリース、資金調達、提携
- **最終更新日**: [今日の日付]

簡潔なリファレンスドキュメントとして。深掘りではなく参照用。
```

**所要時間**: 5-10分/社、$0.15-0.35

### 比較マトリクス生成

個別プロファイル作成後:
```
research/competitors/ の競合プロファイルに基づき、
research/competitors/competitive-matrix-[YYYY-MM].md に比較マトリクスを作成。

機能比較テーブル:
- 行: 主要機能（認証、権限管理、インテグレーション、モバイル対応、API、レポート等）
- 列: 自社、競合A、競合B、競合C
- 値: ✓（完全対応）、~（部分/限定）、x（未対応）、?（不明）

価格比較テーブル:
- 行: Free、Starter、Professional、Enterprise
- 列: 同上
- 値: 月額 or "要問合せ" or "N/A"

各テーブル後に2-3文の分析コメント。
```

### バージョン管理による時系列追跡

四半期ごとにプロファイル更新＋新マトリクス生成。gitで差分追跡:
```bash
# Q1 vs Q2 の変化を確認
git diff competitive-matrix-2026-01.md competitive-matrix-2026-04.md
```

これにより: 競合の機能追加、価格変更、新規参入を時系列で把握可能。

## 価格追跡

### 価格詳細調査

```
[競合名] の現在の価格モデルをWebサーチで調査し、
research/pricing/[competitor-name]-pricing-[YYYY-MM].md に詳細を作成。

含めるべき項目:
- **ティア構成**: 名称、月額、年額（異なる場合）
- **各ティアの制限**: ユーザー数、ストレージ、API呼出し、プロジェクト数等
- **機能ゲート**: 上位ティアでのみ利用可能な機能
- **アドオン**: オプション機能とその価格
- **割引**: 年払い割引率、ボリュームディスカウント、非営利/教育機関向け
- **Freeティア**: 含まれる内容と制限
- **Enterpriseティア**: 公開価格 or "要問合せ"
- **課金モデル**: ユーザー単位、機能単位、従量制、定額
```

### 価格比較テーブル

```
research/pricing/ のファイルに基づき、pricing-comparison-[YYYY-MM].md を作成。

テーブル:
- 行: 価格ティア
- 列: 自社、競合A、競合B、競合C
- 値: ユーザー/月額
```

## フィードバック分析

### データ準備

Claude Codeが処理可能な形式:

| ソース | 推奨フォーマット | 備考 |
|--------|---------------|------|
| Zendesk | CSV | チケット件名、説明、コメント、タグ |
| Intercom | CSV/JSON | 会話スレッド、メタデータ |
| アンケート (Typeform等) | CSV | 1行=1回答者、自由記述列が分析対象 |
| NPS | CSV | スコア + 「この評価の理由」テキスト |
| Slack | テキスト/JSON | スレッドをテキストファイルにコピー |
| インタビュー書起こし | テキスト/Markdown | プレーンテキスト推奨 |
| メール | テキスト | `---` で区切って1ファイルに統合 |

### PII除去（必須）

分析前に以下を除去/匿名化:
- 顧客名、メールアドレス、電話番号、アカウントID
- `Customer_1`, `Enterprise_user` 等のラベルに置換
- Claude Codeは自動的にPIIをスクラブしない。ユーザーの責任

### テーマ特定プロンプト

```
[feedback-folder] のフィードバックをすべて読み、繰り返し出現するトップ10-15のテーマを特定。

各テーマについて:
- テーマ名
- 説明（1-2文）
- 出現頻度（何件言及されたか）
- 代表的な引用（3-5件、原文のまま）

頻度順にソート。
```

### フィードバック分類プロンプト

```
[feedback-folder] のフィードバックを以下のカテゴリに分類:
[カテゴリ1], [カテゴリ2], [カテゴリ3], [その他]

カテゴリ別の件数、センチメント内訳、注目すべきパターンを含む要約を作成。
```

### インタビュー分析プロンプト

```
[interview-folder] のインタビュー書起こしを読み、インタビュー横断のパターンを分析。

特定すべき項目:
- 共通のペインポイント
- 満たされていないニーズ
- ユーザーが作成した回避策
- 感情的な反応
- 機能リクエスト（明示的・暗黙的）
```

### インサイト→アクションマッピング

```
[レポートファイル] のテーマレポートをレビューし、各主要テーマをプロダクトアクションにマッピング。

各テーマについて:
- Quick Win（低コスト・短期で実施可能なアクション）
- 大規模イニシアティブ（ロードマップに含めるべき取り組み）
- モニタリング対象（今はアクション不要だが注視すべき事項）
```

## 分析が効果的でないケース

- **20件以下のフィードバック**: 手作業で読んだ方が早い
- **定量データのみ**: スプレッドシートが適切
- **ニッチ業界（公開情報が少ない）**: 営業コール、アナリストレポート、顧客インタビューで補完が必要
- **価格非公開の競合**: 「要問合せ」が多い場合、Claude Codeの価値は限定的
- **急速に変化する市場**: 週単位で情報が陳腐化する場合

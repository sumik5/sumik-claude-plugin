# 問題タイプ別パターン

Serenaはキーワードベースで最適な思考パターンを自動選択します。

## デバッグパターン（5-8思考ステップ）

**キーワード**: error, bug, issue, broken, failing, エラー, バグ, 問題

```bash
/serena "APIエラーが発生している"
```

**思考フロー**:
症状分析 → 環境確認 → 原因仮説 → 検証 → 解決策 → 実装 → 検証戦略 → 予防措置

## 設計パターン（8-12思考ステップ）

**キーワード**: architecture, system, structure, plan, 設計, アーキテクチャ

```bash
/serena "ユーザー認証システム設計"
```

**思考フロー**:
要件整理 → 制約確認 → 選択肢生成 → 評価 → 技術選定 → 設計決定 → 実装フェーズ → リスク軽減

## 実装パターン（6-10思考ステップ）

**キーワード**: build, create, add, feature, implement, 作成, 追加, 実装

```bash
/serena "リアルタイムチャット機能追加"
```

**思考フロー**:
仕様確認 → 技術選定 → 設計 → 依存関係 → 実装順序 → テスト戦略 → エラーハンドリング

## 最適化パターン（4-7思考ステップ）

**キーワード**: performance, slow, improve, refactor, optimize, 最適化, 改善

```bash
/serena "データベースクエリ最適化"
```

**思考フロー**:
現状分析 → ボトルネック特定 → 改善機会 → 解決選択肢 → 優先度 → 実装影響評価

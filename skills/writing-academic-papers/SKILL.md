# アカデミックライティング

## 🎯 概要と使用タイミング

このスキルは学術的文章作成を包括的にガイドします。

**使用場面**:
- エッセイ、論文、レポート作成
- 修士論文（dissertation）・博士論文執筆
- リフレクティブジャーナル作成
- 試験でのエッセイライティング

**対象外**:
- 技術文書・コードコメント → `writing-technical-docs` を使用

---

## 🏛️ コアプリンシプル（5つの柱）

### 1. ライティングプロセス

反復的サイクルで進行：

```
計画（40%） → 下書き（20%） → 推敲（40%）
```

**重要な活動**:
- **課題分析**: コマンドワード（discuss, analyse, evaluate等）を特定
- **リサーチクエスチョン設定**: トピック・制限ワードからキーワードリスト作成
- **計画手法**: マインドマップ（発散）→ リニアプラン（収束）
- **批判的思考**: 分析・比較・評価・統合のレベルで文献を扱う

### 2. 一貫性と論証

#### 組織化原則
- **Classification**: カテゴリ分類
- **Comparison and Contrast**: 類似点と相違点
- **Cause and Effect**: 原因と結果
- **Problem-Solution**: 問題分析と解決策

#### Coherence（一貫性）
- **General → Specific**: 一般的記述から具体的詳細へ
- **Old → New**: 既知情報への参照 → 新情報の提示

#### 論証構築
```
Claim（主張）
  ↓
Evidence（証拠）
  ↓
Warrant（根拠）- なぜその証拠が主張を支持するか
  ↓
Qualifier（限定）- 主張の適用範囲
  ↓
Rebuttal（反論への対応）
```

### 3. 引用・参照（Harvard System）

#### 基本原則
- **One-to-one match**: 本文引用とReferencesの一対一対応
- **著者名・年の記載**: 本文中は (Surname, Year) 形式
- **参考文献リストはアルファベット順**: References（読んだものだけ記載）

#### In-text Citation
- **3名以上の著者**: 最初の著者名 + "et al" （例: Johnson et al, 2003）
- **直接引用**: 引用符 + ページ番号必須（例: (Thomas, 2016, p. 36)）
- **複数参照**: 時系列順、セミコロンで区切る（例: (Li, 2002; James, 2007; Jones, 2007)）

#### 批判的ソース活用
- **Author-prominent**: 著者を強調（Butler (2007) questions...）
- **Information-prominent**: 情報を強調（Scientific accuracy may be sacrificed (Butler, 2007)）
- **報告動詞の選択**: argues, suggests, advocates, identifies 等で態度を示す

### 4. アカデミックスタイル

#### 3要素
| 要素 | 説明 |
|------|------|
| **明確性（Clarity）** | 論理的順序、短文、リンクバック |
| **簡潔・正確性（Conciseness & Precision）** | 冗長表現排除、正確な語彙選択 |
| **フォーマル性（Formality）** | 客観的トーン、非人称構造、短縮形禁止 |

#### フォーマリティを高める戦略
- ❌ 会話表現（actually, by the way, to be honest）
- ✅ フォーマルな否定（little/few → not much/many）
- ✅ 接続詞の適切配置（文頭の"And"/"But"を避ける）
- ✅ 客観性の保持（I/you/we を慎重に使用、受動態・非人称構造を活用）
- ✅ 短縮形の排除（they're → they are）

#### 批判的言語
- **確信度**: will, may, possibly, to some extent
- **程度**: entirely, largely, to a great extent, marginally
- **評価語彙**: innovative, enhance（ポジティブ）/ problems, struggle（ネガティブ）
- **コメント副詞**: Crucially, Importantly, Worryingly, Clearly

### 5. 提出準備

#### 編集 vs 校正
| プロセス | フォーカス |
|---------|-----------|
| **最終編集（Editing）** | 意味・流れ・論理性・構成 |
| **校正（Proofreading）** | スペリング・文法・句読点・タイポ |

#### フォーマット
- **行間隔**: 1.5またはダブルスペース
- **段落分離**: 空白または字下げで明確化
- **ワードカウント**: ±10%以内（例: 2,500語課題 → 2,250～2,750語）
- **フォント**: 12pt Times New Roman / Arial

## 🗂️ クイックリファレンス

### エッセイ構造（標準）

| セクション | 割合 | 必須要素 |
|-----------|------|---------|
| **Introduction** | 10-15% | Hook, Context, Thesis Statement, Scope, Structure |
| **Main Body** | 70-80% | PEEL法（Point, Evidence, Explanation, Link） |
| **Conclusion** | 10-15% | Restatement, Summary, Synthesis, Implications |

### 文書タイプ別構造

| 文書タイプ | 構造推奨 |
|-----------|---------|
| **エッセイ** | Introduction（TAOS） → Body（ファネル型・PEEL法） → Conclusion（4ステップ） |
| **リフレクティブエッセイ** | 経験 → 感情 → 評価 → 分析（理論接続） → 結論 → 行動計画（詳細は [REFLECTIVE-WRITING.md](references/REFLECTIVE-WRITING.md)） |
| **Dissertation** | Introduction(10%) → Literature Review(30%) → Methodology(15%) → Findings(20%) → Discussion(20%) → Conclusion(5%)（詳細は [DISSERTATION-WRITING.md](references/DISSERTATION-WRITING.md)） |
| **試験エッセイ** | 計画(15%) → 執筆(70%) → 見直し(15%)、即座のブレインダンプ |

### 引用形式の選択

| 文献タイプ | Harvard形式例 |
|-----------|--------------|
| **書籍** | Surname, Initial (Year) Title. Place: Publisher. |
| **編集書籍の章** | Surname, Initial (Year) Title of Chapter. In Surname, Initial (ed/eds) Title of Book. Place: Publisher. |
| **ジャーナル論文** | Surname, Initial (Year) Title of Article. Journal Title, Volume(Part), Pages. |
| **組織著者** | DfE (2011) Title. Place: Publisher. |
| **電子ソース** | [情報]. [online] Available at: [URL] (accessed [日付]). |

### フォーマル表現変換（代表例）

| インフォーマル | フォーマル |
|----------------|------------|
| not much / not many | little / few |
| got worse | deteriorated |
| about | approximately |
| show | demonstrate |
| make sure | ensure |
| like | such as |
| a lot of | many / much |
| can't | cannot |

## 🤔 ユーザー確認の原則（AskUserQuestion）

**判断分岐がある場合、推測で進めず必ずAskUserQuestionツールでユーザーに確認する。**

### 確認すべき場面

| 確認項目 | 例 |
|---------|-----|
| **引用形式** | Harvard / APA / Vancouver / その他 |
| **文書タイプ** | エッセイ / Dissertation / リフレクティブ / 試験 |
| **対象読者レベル** | 学部生 / 修士 / 博士 / 学術誌 |
| **英語バリエーション** | British English / American English |
| **ワードカウント範囲** | 厳密に守るか、±10%許容か |

### 確認不要な場面

- **Plagiarism回避**: すべてのソースを明示（常に必須）
- **一貫性確保**: 選択したスタイルを文書全体で統一（常に必須）
- **文法的正確性**: 正しい文法・句読点を使用（常に必須）

## 🔄 作業フロー例

### エッセイ作成（標準）

```
1. 課題分析
   ├─ コマンドワード特定（discuss, evaluate等）
   ├─ トピック・制限ワード抽出
   └─ リサーチクエスチョン設定

2. 計画（40%の時間）
   ├─ マインドマップでブレインストーミング
   ├─ リニアプランで構造整理
   └─ Thesis Statement 作成

3. 下書き（20%の時間）
   ├─ Introduction: Hook → Context → Thesis → Scope → Structure
   ├─ Body: PEEL法で各段落（Point → Evidence → Explanation → Link）
   └─ Conclusion: Restatement → Summary → Synthesis → Implications

4. 推敲（40%の時間）
   ├─ 最終編集（意味・流れ・論理性）
   ├─ 批判的言語の確認（確信度、程度、評価語彙）
   ├─ 引用・参照の正確性確認（One-to-one match）
   └─ 校正（スペリング・文法・句読点）

5. 提出準備
   ├─ ワードカウント確認（±10%以内）
   ├─ フォーマット確認（1.5/ダブルスペース、12pt）
   └─ タイトルページ・ページ番号追加
```

### Dissertation作成

```
1. Research Questions 設定（Introduction）
2. Literature Review（30%）
   ├─ Gap identification
   ├─ Theoretical framework
   └─ Critical synthesis
3. Methodology（15%）
4. Findings（20%）- データの客観的提示
5. Discussion（20%）- データの解釈・理論との対話
6. Conclusion（5%）- 要約・含意・限界・今後の研究
```

## ⚡ 実践のポイント

### 優先順位
1. **課題への適合**: 設問に直接回答しているか
2. **批判的思考**: 記述でなく分析・評価・統合
3. **エビデンス**: すべての主張に信頼できるソースを提示
4. **一貫性**: 引用形式・スタイル・論理展開の統一
5. **明確性**: 読者が容易に理解できる構造と表現

### よくある落とし穴
- ❌ 下書きに時間をかけすぎる（計画と推敲を軽視）
- ❌ 記述的ライティング（批判的分析が不足）
- ❌ 引用の乱用（自分の声が弱い）
- ❌ パラフレーズで単語を同義語に置き換えるだけ
- ❌ ソースを「パッチワーク」状に並べるだけ
- ❌ ワードカウント制限を無視

### 成功の鍵
- ✅ 計画に十分な時間を投資（40%）
- ✅ 批判的言語を駆使（確信度・程度・評価語彙）
- ✅ ソースを比較・対照し、自分の議論に統合
- ✅ 音読で流れと自然さを確認
- ✅ 複数回読み通して推敲

## 📊 チェックリスト

### 計画段階
- [ ] 課題のコマンドワード、トピックワード、制限ワードを特定した
- [ ] キーワードリストを作成し、同義語・関連概念を洗い出した
- [ ] リサーチクエスチョンを明確に定式化した
- [ ] 主要な文献を特定し、ノートを取った

### 執筆段階
- [ ] 明確なThesis Statementを作成した
- [ ] Introductionに必要な要素（Hook, Context, Thesis, Scope, Structure）を含めた
- [ ] 各パラグラフにトピックセンテンスがある
- [ ] 主張にはすべて証拠（引用、データ、事例）が付随している
- [ ] 情報源への距離を示す動詞（claims, demonstrates等）を使い分けた
- [ ] 異なる視点を対照・評価した
- [ ] Conclusionで新しい証拠を導入していない

### 推敲段階
- [ ] Thesis Statementに対して一貫した論証ができているか
- [ ] 各パラグラフが一つの明確な論点を扱っているか
- [ ] 記述的内容と批判的分析のバランスは適切か
- [ ] 引用・参照が正確で一貫しているか（One-to-one match）
- [ ] ワードカウント制限を守っているか（±10%以内）

### 提出準備
- [ ] 最終編集（意味・流れ・論理性）を実施
- [ ] 校正（スペリング・文法・句読点）を実施
- [ ] フォーマット（1.5/ダブルスペース、12pt、段落分離）を確認
- [ ] タイトルページ・ページ番号を追加
- [ ] 提出方法・形式・締切を確認

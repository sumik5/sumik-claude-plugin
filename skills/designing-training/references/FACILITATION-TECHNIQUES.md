# ファシリテーション実践技法

研修ファシリテーションの実践的手法。アクティビティ設計（オープニング・クロージング・リビジット・エナジャイザー）、質問技法、グループワーク運営、困難な状況への対処法、対応が難しい参加者への接し方を網羅する。

---

## 1. アクティビティのデザイン原則

### 1-1. すべてのアクティビティに「目的」を持たせる

研修で行うことすべてに**目的**がある。ディスカッションの内容、部屋のセッティング、使用する道具、順序、使う言葉まで、細部にいたるまで意図を持たせる。

**よくある失敗：**
- 「参画させること」自体が目的になり、本来の意図を見失う
- ディスカッション後に個人への落とし込みが設計されていない
- 発表を「聞く」だけで終わり、各自への適用が促されない

**改善の原則：**

```
悪いデザイン例（接客スキル研修）：
①うまくいっていること・課題をチームで話し合う
②各チームから発表する
→ 聞く作業に集中すると「他者の成功事例を自分に適用する」同時処理は困難

良いデザイン例：
①うまくいっていること・課題をチームで話し合う
②各チームから発表する
③個人で「他者の成功事例から何を学べるか、自分の課題のヒントはあったか」を考察する
④考察をペア/チームでシェアして深める
```

脳はマルチタスクが苦手。「聞く」と「分析・適用」を同時に求めない設計にする。

### 1-2. 5つのアクティビティシーン

| シーン | タイミング | 目的 |
|--------|----------|------|
| **オープニング** | 研修開始時・各トピック開始時 | 興味喚起・場づくり・参画意欲の醸成 |
| **クロージング** | 研修終了時・各トピック終了時 | 振り返り・達成感・アクションプラン |
| **リビジット** | 20分ごと | 記憶定着・重要点の整理 |
| **エナジャイザー** | 集中力が落ちてきた時 | エネルギー回復・場の活性化 |
| **EATの「E」** | 各トピックのオープニング | 体験・経験を通じた主体的な学び |

## 2. オープニングのデザイン

### 2-1. オープニングの目的

| 目的 | 内容 |
|------|------|
| **メリットを感じてもらう** | この研修で学ぶことが、参加者自身にとって価値があると感じてもらう |
| **居心地の良さを生む** | 「この場に存在していたい」「このメンバーと時間を過ごしたい」と感じてもらう |
| **主体的参画を体験させる** | 「全員が能動的にかかわる研修」であることを体験を通じて感じてもらう |

人は**最初と最後に触れた情報を記憶しやすい**。研修の最初であるオープニングは、重要なメッセージを入れる好機。

### 2-2. 効果的なオープニングの構成順序

```
1. 研修内容と関連性があり、インパクトのあるアクティビティ
   └─ 問いかけ・クイズ・手を動かすワーク（参加者が「口と手を使う」）

2. 研修目的・アジェンダの説明
   └─ 講師が一方的に話すのではなく、参加者が能動的に確認する方法を検討

3. グラウンドルールの確認
   └─ 講師からいくつか提示し、参加者が追加する形も有効

4. 参加者同士の自己紹介（チーム内など少人数で）
   └─ 研修内容に関連する情報を含める自己紹介が効果的

5. 講師の自己紹介
   └─ このテーマで講師を務めるのにふさわしい理由が伝わる内容
```

**注意：** 多くの研修で「1」がなく「2」から始まっている。または「5→2」の順になっている。

### 2-3. オープニングの時間目安

| 研修時間 | オープニング推奨時間 |
|---------|------------------|
| 1〜3時間 | 5〜10分 |
| 半日（4時間） | 15〜20分 |
| 1日（7〜8時間） | 20〜30分 |

短時間の研修では「3.グラウンドルール」「4.自己紹介」を割愛可。ただし、少しでも隣の人と話すきっかけをつくる。

### 2-4. オープニングのDOとDON'T

| DO（推奨） | DON'T（避ける） |
|-----------|--------------|
| 開始前から参加者に笑顔で話しかける | 講師が開始時刻直前に入ってくる |
| 興味を引く掲示物やグッズを置く | 淡々と事務局や部長が話し続ける |
| 最初に問いかけやクイズを出す | 講師が一方的に長々と話す |
| チーム内など少人数での自己紹介 | 全員の前で一人ずつ自己紹介 |
| 事前課題の自己採点（成績を公開しない） | 事前課題の理解度を指名確認する |

### 2-5. 「何を学びたいか」を参加者自身に語らせる方法

```
AskUserQuestion:
- 上司と部下の対話が整っており、参加者が明確な学習目的を持っている → 直接「今日何を学びたいか？」を問いかけ可
- そうでない場合（多くの場合） → 下記の間接的な方法を使う
```

**間接的な方法：**
- ワークブックに目をとおして、学びたいページに印をつけ、チーム内でシェアする
- アジェンダ一覧を見て、とくに興味があるトピックに投票する

いきなりオープンクエスチョン「今日何を学びたいですか？」は「よくわかりません」という回答を引き出すリスクがある。

## 3. クロージングのデザイン

### 3-1. クロージングの目的

| 目的 | 内容 |
|------|------|
| **学習内容の確認** | 学んだことを整理・定着させる |
| **達成感・充実感** | 職場に戻って実践しようという気持ちを生む |
| **アクションプランの策定** | 実践に向けた具体的な計画を研修内で立てる |

### 3-2. 効果的なクロージングの構成順序

```
1. アンケート記入（コンテンツ終了後、最初に行う）
2. 習得したことの確認（参加者が主体的に行うアクティビティで）
3. アクションプランを立てる（個人で書いてペア/チームでシェア）
4. メッセージ性のあるコンテンツで締め（成功事例・行動を促す言葉等）
```

**重要：** アンケートと事務連絡を最後にしない。「人は最初と最後を記憶しやすい」ため、最後は重要なメッセージで終わる。

### 3-3. アクションプランのポイント

- 「～を意識する」「～に気をつける」など抽象的な表現のままにしない
- **5W1Hで具体化**する（いつ・どこで・誰と・何を・どのように）
- うまくいかない状況も想定し、乗り越え方のイメージも描く
- アクションプランをシェアし、参加者同士で励ましのコメントを述べ合う

**なぜ研修内でアクションプランを立てるのか：**
「職場に戻ってから」にすると忘却が進む。翌日には多くのことを忘れている。研修内でアクションプランを立てることで、実践率が格段に上がる。

## 4. リビジット（復習）のデザイン

### 4-1. リビジットの目的

| 目的 | 内容 |
|------|------|
| **重要ポイントの繰り返し** | 短期記憶から長期記憶への移行を促す |
| **重要ポイントの整理** | 学んだことを体系化・構造化する |

20分に1回のリビジットが基本（90/20/8の法則に基づく）。1〜2分程度の短いアクティビティで十分。

### 4-2. リビジットのアクティビティ例

| 方法 | 内容 |
|------|------|
| 要点を3つ挙げる | 「ここまでで最も重要だと思った点を3つ挙げてください」 |
| ペアで説明 | 「学んだことをペアの人に2分で説明してください」 |
| テキストへの印 | 「最も重要だと思う箇所に線を引いてください」 |
| クイズ形式 | 「次のうち正しいのはどれか？」 |

**リビジットで避けること：** 新しいコンテンツを追加しない（「ちなみに...」「余談ですが...」は次のコンテンツとして扱う）。

## 5. エナジャイザー

### 5-1. エナジャイザーとは

集中力の低下、食後の眠気、長時間研修での疲れなど、エネルギーが落ちてきた時に実施する短い活動。内容は研修テーマと必ずしも関係なくて良い。

**目的：**
- 身体を動かして眠気や倦怠感を解消する
- 場の雰囲気を明るく軽くする
- 脳に新鮮な刺激を与える

**実施の判断基準：**

```
AskUserQuestion:
- 研修が午後に差し掛かり、参加者の集中力が明らかに落ちている → エナジャイザーを実施
- 深刻なテーマ（コンプライアンス等）の研修 → 慎重に判断（場の雰囲気に合わせる）
- 半日以下の短い研修 → 通常不要
```

## 6. 質問技法

### 6-1. 質問の種類と使い分け

| 種類 | 特徴 | 使用タイミング |
|------|------|-------------|
| **クローズド質問** | Yes/No で答えられる | 参加者が答えやすく、安心感がある |
| **オープン質問** | 自由に答えられる | 思考を深めたい時（ただし広すぎると注意） |
| **クイズ形式** | 選択肢から選ぶ | 知識確認・EAT の「E」段階 |
| **リダイレクト** | 参加者の発言を他の参加者に転送 | 一人への集中を避け、全員を巻き込む |

### 6-2. タキソノミー（認知過程次元）に基づく問いかけ設計

コンテンツの到達目標レベルに合わせて問いかけのレベルを調整する。

| レベル | 名称 | 問いかけのキーワード | 例 |
|--------|------|-------------------|-----|
| 1 | **記憶** | 「〜は何でしたか？」「〇点挙げてください」 | 「3つのポイントの1つめは何でしたか？」 |
| 2 | **理解** | 「例を挙げる」「選ぶ」「自分の言葉でまとめる」 | 「5つのポイントで最も重要なのはどれで、理由は？」 |
| 3 | **活用** | 「実践する」「活用する」「当てはめる」 | 「今学んだことを自分の状況にどう当てはめますか？」 |
| 4 | **分析** | 「原因を分析する」「検証する」「比較する」 | 「なぜこのようなことが起きるのか？」 |
| 5 | **評価** | 「判断する」「比較評価する」「もっとも〇〇を選ぶ」 | 「A案とB案、どちらが優れているか？」 |
| 6 | **創造** | 「作成する」「計画する」「考案する」 | 「学んだことを活かして解決策を考案してください」 |

### 6-3. 経験を引き出す問いの設計

**オープンすぎる問い（避ける）：**
- 「良いリーダーとは？」 → 講師の意図とズレた回答が出やすい
- 「良い接客とは？」 → 持論が展開されやすい

**経験を引き出す問い（推奨）：**
- 「今までした買い物で、予定はなかったけれど何かを提案されて購入し、結果としてとても良い買い物ができたという経験はありますか？」
- 「最も印象に残っているチームの仕事の場面を思い出してください。どんなことが起きていましたか？」

**なぜ有効か：**
- 自分自身の成功/失敗ではなく第三者の話なので客観的に話せる
- 具体的・現実的な内容で話しやすい
- 研修内容から大きくズレた回答が出にくい

### 6-4. 知識を引き出す問い（クイズ形式）

クイズ形式の問いかけで参加者の知識を自然に引き出す：

```
効果的なクイズ形式：
- 「正しい文はどちらでしょうか？」（二択）
- 「A、B、Cのうち、正しいのはどれですか？」（三択）
- 「空欄に入る数字・言葉を埋めてください」（穴埋め）
- 「この文章の中に間違いがあります。どこでしょうか？」
- 「正しい手順に並べてください」

運用のコツ：
まずは個人で答えを考えてもらい、その後ペアや少人数チームで検討する
```

### 6-5. 経験学習サイクルに基づく問いかけ

体験型アクティビティ（ロールプレイ等）の後に行う振り返りの問い：

| ステップ | 目的 | 問いかけ例 |
|---------|------|-----------|
| 1. 具体的経験の確認 | 何が起きたかを正確に思い出す | 「誰が何を言った/行ったか、順を追って思い出してください」 |
| 2. 内省的観察 | 言動の評価を行う | 「目的達成に貢献したことは？マイナスになったことは？」 |
| 3. 抽象的概念化 | 一般化・抽象化する | 「○○において大切なのはどのようなことでしょうか？」 |
| 4. 能動的実験 | 今後への活用を考える | 「次の機会に何をどう活かしますか？」 |

## 7. グループワーク運営

### 7-1. グループサイズの選択基準

```
AskUserQuestion:
- 参加者の緊張が高い・考察タイプが多そう → ペア（2人）
- 通常のディスカッション → 3〜4人
- 多様な意見を出したい → 5〜6人
- 時間が少ない → 人数を減らす（ペアにする）
```

**人数と特性：**

| 人数 | メリット | デメリット |
|------|---------|----------|
| 2人（ペア） | 発言プレッシャーが低い・話しやすい | 発想が限られやすい |
| 3〜4人 | 発言機会が多く、多様性も確保 | バランスが取りやすい |
| 5〜6人 | 多様な視点・発想 | 一部が話さない可能性 |

### 7-2. チームメンバーの固定化を避ける理由

チームメンバーを固定し続けるデメリット：
- 発想が広がりにくくなる
- 新鮮さがなくなる
- 一部のメンバーが仕切りやすくなる
- 対応が難しい参加者がいた場合、特定の人がその影響を受け続ける

→ 定期的にメンバーをシャッフルする設計を推奨。

### 7-3. 時間管理のテクニック

**時間をオーバーさせないための工夫：**
- スクリーンにタイマーを表示する（キッチンタイマーを手元に置くだけでは参加者に見えない）
- 「残り○分」の合図を出す
- 必要に応じて個別に声をかける

**時間が足りない・余る場合の対策：**
- 足りない場合：あらかじめカットできるアクティビティを決めておく
- 余る場合：あらかじめ追加できる簡単なアクティビティを準備しておく

## 8. 困難な状況への対処法

### 8-1. ディスカッションが活発でない場合

まず原因を特定し、以下の3点を見直す：

| 見直しポイント | 対策 |
|------------|------|
| **設定時間** | 短くする（時間が長いとゆっくりしたモードになる） |
| **メンバー・人数** | シャッフルする・人数を減らす |
| **問いかけの質** | オープンすぎないか確認・具体的な問いに修正 |

**問いかけの改善：**
- 「深い自己開示を求める問い」→ グラウンドルール設定・ペアにする
- 「難易度の高い問い」→ 段階的に答えを導くデザインにする

### 8-2. 反応がない（誰も発言しない）場合

**根本的な原因：** 「全体に問いかけて誰かが挙手する」というスタイル自体に問題がある。

**解決するファシリテーションの基本ステップ：**

```
ステップ1：問いかけの内容を伝える
↓
ステップ2：チームのリーダーをランダムな方法で選ぶ
↓
ステップ3：チーム内で話す
↓
ステップ4：リーダーが全体にシェアする
```

「発表する役割」を事前に決めることで、「発言しよう」という意識が高まる。

### 8-3. 話が盛り上がって時間がオーバーする場合

**まず判断：**
- 意義のある話で盛り上がっている → 時間を延長して内容を優先する（記録して次回の設計に活かす）
- 意義が低い/脱線 → 介入して軌道修正する

**設定時間に合わせてもらう工夫：**
- スクリーンのタイマー表示
- 「残り○分」の声かけ
- 「ペースを上げましょう」と個別に伝える

### 8-4. 想定外のコメント・挑戦的な質問への対応

**基本のステップ：**

```
①発言に感謝する
  「シェアありがとうございます」

②考えたプロセスを聞き、受け止める
  「どのようなプロセスで話し合ってその結論が出ましたか？」
  「そう考えると、たしかにそうなりますね」

③他の参加者を巻き込む
  「他のチームはどうでしょうか？」
  「今のコメントについて、どう思いますか？」

④全体に問いかけてディスカッションに転換する

⑤答える
  「確認させてください」（時間を置く）
  不明な場合：「後日回答します」と約束する
```

**やってはいけないこと：**
- 発言を無視して講師のまとめに入る
- 挑戦的な参加者と講師が1対1で議論し続ける（場の空気が悪くなる）

## 9. 対応が難しい参加者への接し方

### 9-1. 対応が難しい参加者を生まないために

対応が難しい参加者を「生まない」設計が最善策：

| 設計要素 | 予防効果 |
|---------|---------|
| 会場レイアウト（円卓型推奨） | 対話が生まれやすい・講師の死角が少ない |
| オープニングでの関係構築 | 参加者と講師の信頼関係を早期に形成 |
| 問いかけの質 | オープンすぎる問いが挑戦的な発言を誘発しないよう設計 |
| グループダイナミクスの活用 | 場の力を利用して問題行動を抑制 |
| 事前学習と上司の巻き込み | 目的意識を持って参加してもらう |

### 9-2. ケース別対応

| ケース | 対応策 |
|--------|--------|
| 隣にこそこそ話す | 全体への問いかけで巻き込む・近くを歩く |
| メール等、他のことをする | 問いかけてチームに参画させる機会を作る |
| 寝る | 物理的に近くを歩く・グループに参画させる |
| 発言しない | チームで話す設計にし、全体発表役をランダムに選ぶ |
| 関係のないおしゃべり/脱線 | チームを移動させる・問いかけで本題に引き戻す |
| 否定的・挑戦的 | 9-3の「想定外のコメント対応」フローを使う |
| ポイントがずれた発言 | 「そのほかに」で続ける・適切な発言を引き出す |
| 同じ人がいつも発言する | チームサイズを小さく・他の人に振る構造を作る |
| 持論を展開する | 「貴重なご意見」として受け止め、チーム全体に振る |
| 質問攻め | 一問ずつ対応・後日回答を約束・他参加者へ転送 |

### 9-3. 許容範囲の設定

参加者全員が全く同じ反応をすることはない。一定の「ばらつき」は許容するという姿勢が重要。

```
AskUserQuestion:
- 問題行動が他の参加者の学習に影響している → 介入を検討
- 少し目立つだけで研修全体への悪影響は小さい → 許容・継続観察
- 一度対応して解決しない場合 → 休憩時に個別に話しかける
```

### 9-4. それでも解決しない場合

休憩中に個別に声をかけ、穏やかに状況を確認する。公衆の面前での指摘は避ける。

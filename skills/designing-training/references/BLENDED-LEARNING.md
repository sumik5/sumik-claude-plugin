# ブレンディッドラーニング・ハイブリッド研修 設計ガイド

対面とオンライン、同期と非同期を組み合わせたブレンディッドラーニングの設計・実施リファレンス。
研修を「1回限りのイベント」ではなく「学習プロセス」として設計するための方法論を提供する。

---

## 1. 研修は「イベント」ではなく「プロセス」

### なぜプロセスとして設計するか

従来の研修設計では「研修当日」に集中しがちだった。しかし効果的な学びは、研修前・研修中・研修後の継続的なプロセスとして設計されることで最大化される。

```
[研修前]      →      [研修中]      →      [研修後]
事前学習         集合学習（対面/オンライン）    フォローアップ・実践
(非同期型)          (同期型)               (非同期/同期)
```

**研修効果を最大化するために設計すべきこと：**
- 研修前：必要な前提知識を事前学習で取得させる
- 研修中：集合するからこそ可能な「参加者同士の対話」と「体験」に集中する
- 研修後：職場での実践をサポートするフォローアップを設計する

## 2. ブレンディッドラーニングの定義と4つの形態

### 2.1 学習形態の分類

| 軸 | 形態 |
|---|------|
| 同期/非同期 | 同時に同じ時間を共有するか否か |
| オンライン/対面 | 場所を共有するか否か |

### 2.2 4つの学習形態

| 形態 | 説明 | 例 |
|------|------|-----|
| **同期 × 対面** | 同じ場所・同じ時間に集まる | 集合研修、ワークショップ |
| **同期 × オンライン** | 時間を共有するがオンライン | Zoomを使ったオンライン研修 |
| **非同期 × オンライン** | 自分のペースでオンライン学習 | eラーニング、録画講義 |
| **非同期 × 対面** | 自分のペースで対面学習 | 資料を読む、OJT |

### 2.3 ブレンディッドラーニングとは

上記4形態を組み合わせた学習設計。単に「対面とオンラインを混ぜる」のではなく、**各形態の強みを活かした最適な組み合わせ**を設計すること。

---

## 3. 各学習形態の強み・弱み

### 3.1 同期型学習（対面・オンライン共通の強み）

**強み：**
- 参加者同士がリアルタイムで対話・学び合いができる
- 講師が状況に応じて調整できる
- 質疑応答をリアルタイムで行える
- チームビルディング・関係構築がしやすい

**弱み：**
- 全員が同じ時間に集まる必要がある（スケジュール調整が必要）
- 個人のペースに合わせにくい

### 3.2 非同期型学習の強み・弱み

**強み：**
- 自分のペースで学べる
- 何度でも繰り返し視聴・参照できる
- 場所・時間を選ばない
- 個人差に対応しやすい

**弱み：**
- 対話・学び合いが起きにくい
- モチベーション維持が難しい
- フィードバックにタイムラグが生じる

**非同期型学習の効果を高める2つのポイント：**
1. **マイクロラーニング：** 1コンテンツを5〜10分以内に分割。集中が持続しやすい
2. **インタラクティブな設計：** 問いかけ・クイズ・選択問題を組み込んで能動的な学びを促す

### 3.3 対面 vs. オンラインの比較

| 観点 | 対面 | オンライン |
|------|------|-----------|
| 講師のエネルギー伝達 | 高（空間共有） | 低（画面越し） |
| 参加者のグループ人数 | 5〜6人が最適 | 3〜4人が最適 |
| 臨機応変な対応 | 容易（場の空気を読める） | 難しい（視覚情報が限られる） |
| 地理的制約 | あり | なし |
| 事前デザインの重要度 | 高 | さらに高い |
| コスト（移動・会場） | 高 | 低 |

## 4. ブレンディッドラーニングの設計パターン

> **AskUserQuestion（実装時の判断）：** どのブレンディッドパターンを使うかは、研修目的・対象者・制約条件によって異なる。以下のパターン選択テーブルを参照。

### 4.1 パターン選択テーブル

| 条件 | 推奨パターン |
|------|------------|
| 前提知識にばらつきがある | パターン1（事前学習型） |
| 遠隔地の参加者がいる | パターン2（オンライン中心型） |
| スキル習得 + 知識習得が両方必要 | パターン3（ハイブリッド型） |
| 実践・フォローアップが重要 | パターン4（継続学習型） |

### 4.2 パターン1：事前学習型（フリップドラーニング）

```
[非同期オンライン（事前）]     [同期（集合研修）]     [非同期（事後）]
・基礎知識の動画・資料      →  ・応用・実践・対話    →  ・振り返り・実践記録
・eラーニング             →  ・ケーススタディ      →  ・フォローアップ面談
・事前課題                →  ・グループワーク      →
```

**効果的な場面：**
- 知識習得は個別で行い、集合では「活用・実践・対話」に集中したい場合
- 参加者の前提知識を揃えてから集合研修を始めたい場合

**設計の注意点：**
- 事前学習の量は「集合研修の20〜30%相当」が目安
- 事前学習の完了を確認できる仕組みを設ける（課題・小テスト等）
- 研修冒頭に事前学習内容を振り返るリビジットを必ず設ける

### 4.3 パターン2：オンライン中心型

```
[同期オンライン研修（メイン）]  →  [非同期フォロー]
・参加者主体の研修             →  ・実践記録・共有
・ブレイクアウトルーム活用       →  ・質問対応
・90/20/4の法則で設計         →  ・コミュニティ
```

**効果的な場面：**
- 遠隔地に参加者が分散している
- 移動コスト・時間を削減したい
- 継続的な学習プログラムを設計したい

### 4.4 パターン3：対面 + オンラインのハイブリッド型

```
[同期 × 対面]  +  [同期 × オンライン]  →  [非同期フォロー]
一部の参加者が    他の参加者がリモート      実践・振り返り
物理的に集合       で参加
```

> **⚠️ 注意：** ハイブリッド研修（同じ時間に対面とオンラインが混在）は設計が最も複雑。後述の「ハイブリッド研修の設計」参照。

### 4.5 パターン4：継続学習型（プロジェクト型・複数回型）

```
第1回（集合/オンライン）  →  中間課題（非同期）  →  第2回  →  実践発表
```

**効果的な場面：**
- 行動変容が目的（知識だけでなく実践が必要）
- 複数のスキルを段階的に習得させたい
- OJTと連動させたい

## 5. 対面研修をオンラインに変更する際のポイント

### 5.1 変更前の確認事項

まず「その研修を本当にオンラインで行う必要があるか」をゼロベースで検討する。

**オンラインに適した研修の特徴：**
- 対話・ディスカッションを通じた学びが中心
- 参加者同士の学び合いに価値がある
- インストラクショナルデザインを適切にオンライン化できる

**オンデマンド（非同期）に変更した方が良い研修の特徴：**
- 一方向的な知識伝達が中心
- 個人学習で完結できる内容
- スケジュールを合わせる必要性が低い

### 5.2 3つの変更ポイント

#### ポイント1：時間設定

| 対面研修 | オンライン変換の目安 |
|---------|-------------------|
| 1日（7〜8時間） | 2〜3回に分割（各2〜3時間） |
| 半日（3〜4時間） | 1〜2回（各90〜120分） |
| 2時間 | 90分（内容を絞る） |

- 対面で6時間の研修をそのままオンラインに移すと効果が大幅に落ちる
- オンラインでは「どこを非同期（eラーニング・事前課題）に移すか」を検討する
- 集合（同期）部分は参加者同士の対話と体験に特化させる

#### ポイント2：同期 / 非同期の切り分け

- 一方向的な知識インプットは非同期（事前学習）に移す
- 双方向の対話・ディスカッション・演習を同期（集合）に集中させる

**変換例：**

| 対面で行っていたこと | オンライン変換後 |
|-----------------|---------------|
| 講師の講義（30分） | 事前動画（15分）→ 集合で振り返り（5分） |
| グループディスカッション | オンラインブレイクアウトルーム（人数を3-4人に調整） |
| ケーススタディ演習 | 事前課題 → 集合で共有・議論 |
| 個人ワーク（書き込み） | デジタルワークシート → チャットで共有 |

#### ポイント3：インストラクショナルデザインのオンライン化

- 90/20/4 の法則を適用する（→ ONLINE-DELIVERY.md 参照）
- EAT の構成順序を維持する
- グループ人数を5-6人から3-4人に減らす
- 視覚・聴覚への訴えかけを意識したスライドデザインにする

## 6. ハイブリッド研修の設計

### 6.1 ハイブリッド研修とは

同じ時間に、一部の参加者は物理的に会場に集まり、他の参加者はオンラインで参加する形式。

**難しさの理由：**
- 対面参加者とオンライン参加者が「同じ体験」をするのが非常に困難
- 対面の参加者に合わせると、オンライン参加者が置いてけぼりになる
- カメラ・音響・グループ分け等の技術的な課題が複雑

### 6.2 ハイブリッド研修の設計原則

**全員を「オンライン参加者」として設計する原則：**

対面参加者もオンライン参加者も、同じようにデジタルツール（投票・チャット・ワークシート等）を使う設計にすることで、体験の格差を縮小できる。

**具体的な設計のポイント：**

| 課題 | 対策 |
|------|------|
| 音声がオンライン参加者に届かない | マイクを全員分用意、または会場にスピーカーを複数設置 |
| 会場のカメラが固定で動きを見せられない | 複数のカメラアングルを用意、講師にウェアラブルカメラ |
| グループワークが対面と分断される | 対面参加者もブレイクアウトルームを使い、オンライン参加者と混合グループを作る |
| ホワイトボードが見えない | デジタルホワイトボード（Miro等）を全員が使う形式にする |

### 6.3 ハイブリッド研修の判断基準

| 条件 | 推奨対応 |
|------|---------|
| オンライン参加者が少数（1〜2名） | 全員オンラインに切り替えるか、オンライン参加者専用のサポート役を配置 |
| オンライン参加者が半数程度 | ハイブリッド設計を本格的に行う価値あり |
| 対話・グループワークが重要な研修 | ハイブリッドを避け、全員オンラインまたは全員対面に統一 |
| 情報共有・発表が中心の研修 | ハイブリッドでも対応可能 |

## 7. 事前学習→集合研修→事後フォローの設計

### 7.1 事前学習の設計

**目的：** 集合研修の前に必要な前提知識を揃え、集合時間を「対話・実践」に最大化する。

**効果的な事前学習コンテンツ：**
- 5〜10分のマイクロ動画（1概念 = 1動画）
- 読み物・資料（簡潔にまとめる）
- 事前課題・ワークシート（「自分の課題を書いてくる」等）
- 事前アンケート（参加者の背景・課題感を把握する）

**設計の注意点：**
- 完了確認の仕組みを必ず設ける（修了テスト、確認メール等）
- 事前学習が「任意」だと参加率が下がる → 集合研修との連動を明確にする
- 量は集合研修の20〜30%相当を上限とする

### 7.2 集合研修の設計（事前学習後）

**事前学習を受けた参加者向けの設計変更点：**

| 従来の集合研修 | 事前学習後の集合研修 |
|-------------|------------------|
| 知識のインプット（講義）が中心 | 知識のインプットを最小化（事前済み） |
| 演習・事例紹介が後半 | 演習・ディスカッション・実践が中心 |
| 個別の学習が多い | グループ学習・対話を最大化 |

**リビジット（冒頭の振り返り）必須：**
```
「事前学習で学んだことで、印象に残ったことをチャットに書いてください」
「事前課題で書いてきた自分の課題を、グループでシェアしてください」
```

### 7.3 事後フォローの設計

**目的：** 研修での学びを職場での実践につなげ、行動変容を確実にする。

**フォローアップの手法：**

| 手法 | 内容 | タイミング |
|------|------|----------|
| アクションプランの提出 | 研修終了時に「職場で実践すること」を宣言させる | 研修直後 |
| 1週間後の振り返りレポート | 「何を実践したか・どう変化があったか」を記録 | 研修後1週間 |
| 上長面談 | アクションプランの実践状況を上司と確認 | 研修後2〜4週間 |
| フォローアップセッション | グループで実践事例を共有 | 研修後1〜3ヶ月 |
| オンラインコミュニティ | 継続的な質問・情報交換の場を提供 | 随時 |

## 8. 非同期型学習のコンテンツ作成ヒント

### 動画コンテンツの設計

| 要素 | 推奨 |
|------|------|
| 1動画の長さ | 5〜10分以内 |
| 1動画で扱う概念 | 1つに絞る |
| 構成 | 問いかけ → 説明 → 実例 → まとめ |
| インタラクション | クイズ・選択問題を挿入 |

### 非同期型学習の効果を高める仕掛け

- **進捗の可視化：** 学習の進み具合がわかる（xx/全xx完了）
- **締め切りの設定：** 集合研修の前日までに完了と明確に指定
- **完了報告の仕組み：** メールでの完了通知、管理者への自動報告
- **振り返り促進：** 学習後に「3つのキーポイントを書く」等のアウトプット課題

## 9. インストラクショナルデザインの基本原則

### 研修をゼロから設計する8ステップ

| ステップ | 内容 |
|---------|------|
| 1. ニーズ分析 | 「なぜこの研修が必要か」「どんな成果を期待するか」を明確にする |
| 2. 参加者分析 | 対象者の背景・前提知識・学習スタイルを把握する |
| 3. 目的・目標設定 | 研修終了時に参加者が「できること」を明確にする |
| 4. 全体構成設計 | ブレンディッドパターンを選択し、全体の流れを設計する |
| 5. コンテンツ作成 | 各セクションの内容・アクティビティを設計する |
| 6. 資料・教材準備 | スライド・ワークシート・配付資料を作成する |
| 7. 効果測定設計 | カークパトリック4段階での評価方法を決める |
| 8. 実施・改善 | 研修を実施し、評価結果をもとに改善する |

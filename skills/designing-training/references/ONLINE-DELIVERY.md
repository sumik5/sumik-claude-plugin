# オンライン研修 設計・実施ガイド

オンライン研修を効果的に設計・実施するための包括的なリファレンス。
3つの誤解の解消から、具体的な設計・運営手法までをカバーする。

---

## 1. オンライン研修に関する3つの誤解

オンライン研修を効果的に行うには、まず以下の誤解を解消することが重要。

### 誤解1：講師の講義を「配信」することがオンライン研修である

#### オンラインで集中力が持続しない理由
対面研修では講師のエネルギーや場の雰囲気が参加者に直接伝わり、引き込まれる状態が生まれやすい。しかしオンラインでは「人が同じ空間にいない」という根本的な違いがあり、同じ集中力を維持させることは難しい。

**講義配信型のデメリット：**
- 参加者の集中が途切れやすい（目や指先が他のことに向かう）
- 一方向的な情報伝達に終わり、「学び」が起きない
- 講師が「反応が見えない」と感じ、モチベーションが下がる

#### 「参加者主体の研修」が目指す状態

| 研修形態 | 状態 | 評価 |
|---------|------|------|
| 講義配信型 | 講師→参加者（一方向） | ❌ 効果が低い |
| 双方向型 | 講師↔参加者（質疑応答のみ） | △ 不十分 |
| 参加者主体型 | 参加者同士が対話・学び合い | ✅ 推奨 |

集合して学ぶ意味は「参加者同士が対話をして学び合うこと」にある。これはオンライン・対面どちらでも同じ。

#### 「講義配信型」を「参加者主体型」に変える手順

```
ステップ1: その時間で参加者に何を習得してほしいか（目的）を明確にする
    ↓
ステップ2: 目的を達成するためのインストラクショナルデザインを作成する
    ↓
ステップ3: オンライン特有のツールの使い方を工夫する
```

### 誤解2：オンライン研修では講師の存在をアピールすることが重要である

#### 講師に求められるのは「内容」であり「見栄え」ではない

講師にとって最高の評価は「研修内容を職場に持ち帰って実践し、成果を出してもらうこと」。「講師が誰だったか思い出せないが、学びは今でも活かしている」という状態が理想。

#### 対面とオンラインでの発想の転換

| 場面 | アプローチ |
|------|-----------|
| 対面での集合研修 | 講師の人としての魅力で参加者を引きつけ → 研修内容への興味につなげる |
| オンライン研修 | 研修の内容自体に興味をもってもらう → 参加するメリットを感じてもらう → 前のめりで参加してもらう |

#### オープニングで避けるべきこと・すべきこと

| ❌ NG例 | ✅ OK例 |
|---------|---------|
| 講師の自己紹介から始める | 研修内容に関係のあるクイズから始める |
| 参加者に全員自己紹介させる | 投票・アンケートで全員が気軽に参加できるアクティビティ |
| 説明・講義から入る | 「考える」「リアクションを示す」を最初に行う |

**脳科学的な根拠：** 脳は最初と最後に触れた情報を記憶に留めやすい。研修冒頭のこの貴重な時間を、自己紹介で使うのはもったいない。

### 誤解3：テクノロジーを使いこなすことが最重要である

#### テクノロジーはあくまで「ツール（道具）」

道具を使いこなすこと自体が目的ではない。目的を達成するために道具を選ぶという順序が重要。

**正しい優先順位：**
```
1. 研修の目的を明確にする（インストラクショナルデザイン優先）
2. 目的を達成するための内容・手法を決める
3. それを実現するためのツール・機器を選ぶ
```

#### 機器・環境についての基準

「声がクリアに聞こえる・好印象を与える映像が届く・雑音がない」レベルに達したら、それ以上の機器スペックにこだわる必要はない。時間と労力は「インストラクショナルデザインの向上」に使う。

## 2. 講師に必要な3つのスキル

### 2.1 研修デザイン

- 研修がビジネス課題の本当の解決策かどうかをゼロベースで検討する
- 同期・非同期を含めた学習プロセス全体をデザインする
- 「科学的な研修」（インストラクショナルデザインに基づく）に転換する

### 2.2 デリバリー

- テクノロジーを用いて「届ける」役割を担う
- カメラ目線で話す（スライドや参加者の映像を見て話さない）
- オンラインに適した伝え方を意識する

### 2.3 ファシリテーション

- 学習を「サポート」し「促進」する役割
- プラットフォームの機能を活用して参加者を巻き込む
- 安心・安全な学習環境をオンラインでつくる

## 3. 90/20/4 の法則（オンライン版時間配分）

### 3つの数字の意味

| 数字 | 意味 |
|------|------|
| **90分** | 最大の連続学習時間。90分に一度は休憩を設ける |
| **20分** | コンテンツの区切り単位。20分を1セクションとしてデザインする |
| **4分** | 参加者を巻き込む頻度。4分に1回は何らかのリアクションを求める |

> **注意:** 対面研修の「90/20/8」（8分に1回）より頻度を上げる必要がある。オンラインでは物理的に同じ空間にいないため、集中が途切れやすいため。

### インストラクショナルデザインの最低限のポイント

```
● 研修が90分以上の場合: 少なくとも90分に一度休憩を設ける
● 研修内容を意味のあるまとまりで区切り、1セクション=20分にする
● 20分のセクションが終わったら必ず振り返り（リビジット）を1分行う
● 20分のコンテンツの中では、4分講義→参加者を巻き込む を繰り返す
```

### 20分セクションの構成（CSR）

| 要素 | 説明 |
|------|------|
| **C（Content：コンテンツ）** | 新しい知識・スキルを提供する |
| **S（Sharing：参画）** | 参加者が考え・話し・動く |
| **R（Revisit：リビジット）** | そのセクションの学びを振り返る |

### 2時間研修のデザイン例

```
ソフトオープニング（開始3分前）
→ オープニング（1-2分）
→ セクション1（20分）: C→S→C→S→R
→ セクション2（20分）: C→S→C→S→R
→ 休憩（5-10分）
→ セクション3（20分）: C→S→C→S→R
→ 質疑応答・アクションプラン（10分）
→ クロージング（3-5分）
```

## 4. EAT（経験・気づき・理論）構成順序

### なぜ「理論から始める」のはNGか

脳は新しい情報を「すでに知っていること」に関連づけることで記憶する。先に参加者の経験・知識を引き出してから理論を提示することで、理解が深まり記憶に残りやすくなる。

### EATの順序

```
E（Experience：経験）
  ↓  参加者の過去の経験・知識を引き出す、または研修の場で体験を提供する
A（Awareness：気づき）
  ↓  問いかけによって気づきを深める
T（Theory：理論）
     理論や知識を提示して、経験・気づきと結びつける
```

### EATをオンラインでデザインする方法

| 段階 | オンラインでの実践例 |
|------|---------------------|
| E（経験） | チャットで「これまでの経験で〇〇だったことを書いてください」と問いかける |
| A（気づき） | 小グループ（3-4人）でディスカッションさせる。問いかけ：「どんな共通点があったか？」 |
| T（理論） | スライドで理論を提示し、先ほどの経験との関連を説明する |

## 5. オープニング・クロージングのデザイン

### 5.1 ソフトオープニング

- **開始時刻の3〜5分前から始める**
- 内容：研修テーマに関連したクイズや問いかけ
- 目的：参加者が手持ち無沙汰で緊張したまま待つ状態を防ぐ

### 5.2 オープニング（正式開始後）

**デザインのポイント：**
- 研修コンテンツで最も重要なメッセージを盛り込む
- ポジティブな内容でデザインする
- 徐々に参加者を巻き込む（最初は匿名性の高いツール活用）
- 「問いかけ」で参加者の主体性を引き出す

**効果的なオープニング例：**
```
● 研修に関係するデータ・数値を予測するクイズ
  「〇〇について『はい』と答えた顧客は何%だと思いますか？」

● 研修内容に関するクイズで複数選択肢から選ばせる
  「〇〇で一番効果的な方法はABCのうちどれだと思いますか？」

● チャット / 投票機能 / スタンプで全員が参加できる形式
```

### 5.3 クロージング

**クロージングの役割：**
- 研修の学びを統合・定着させる
- 職場での実践につなげるアクションプランを立てる
- 次への動機づけを行う

**1時間研修のクロージング例：**
```
40分経過時点: 質疑応答（チャット活用）
           ↓
アクションプランを書き出す（「今日学んだことで、職場で実践すること」）
           ↓
残りのコンテンツを進める
           ↓
終了5分前: アクションプランをもう1つ追記する
           ↓
アンケートの案内
           ↓
今後につながる問いかけ（チャットに記入してもらう）
```

## 6. グループ設計

### グループ人数の最適設定

| 形式 | 最適人数 | 理由 |
|------|---------|------|
| 対面での集合研修 | 5〜6人 | 場の空気感・声量で調整可 |
| オンライン研修（カメラオン） | 3〜4人 | 映像で表情が見えるため、やや多くても可 |
| オンライン研修（音声のみ） | 3人 | 声の聞き分け・発言タイミングの把握が難しくなるため |

### グループ設計の注意点

- **リーダーを事前に決める：** 「一番最近〇〇した人がリーダー」など、ランダムな方法で指定する
- **リーダーを固定しない：** 毎回違う人が担う
- **小グループに分かれる前にリーダーを決める：** 分かれた後に指定すると混乱する

**インストラクション例（ブレイクアウトルーム前）：**
```
「この後、みなさんにはグループに分かれてディスカッションをしていただきます。
話す内容は〇〇です。グループに分かれたら、リーダーを決めていただきます。
リーダーは、一番最近宅配で何かを受け取った人です。
ディスカッションの時間は4分です。
確認ですが、話すのは〇〇についてです。
グループに分かれる前に何か確認したいことはありますか？」
```

## 7. エナジャイザー（活性化アクティビティ）

### エナジャイザーとは

研修中の疲れ・飽きを軽減し、参加者にリフレッシュしてもらうためのアクティビティ。オンライン研修では対面より早く疲れが出やすいため、より重要。

### エナジャイザーの例

| アクティビティ | 方法 |
|--------------|------|
| **空欄を埋める** | キーワードが空欄のスライドを表示し、端に並んだ選択肢からどれが入るか線で結んでもらう |
| **投票でアンケート** | 「この研修のペースは？①速すぎる ②ちょうどいい ③遅すぎる」などで途中確認 |
| **窓の外を見てくる** | 最寄りの窓に行って外の様子をチャットに報告してもらう |
| **立つ** | アクションプラン2つ書いたら立ってもらい、30秒間立ったままにする |
| **30秒チャット** | 短い時間を設定して「今、一番解決したいことは？」などをチャットに入力してもらう |

### 変化をつける：ツールを組み合わせる

| ツール・機能 | 特徴 |
|-------------|------|
| チャット | 慣れ親しみやすい。全員が同時に入力可 |
| 投票・アンケート | 匿名性があり、率直な意見が集まりやすい |
| ホワイトボード | 視覚的に意見を整理できる |
| 小グループ（ブレイクアウトルーム） | 深いディスカッションが可能 |
| ワークシートへの書き込み | 手元の教材を使い、ツールに慣れていない人も参加しやすい |

ワンパターンにならないよう、毎回異なるツール・手法を組み合わせる。

## 8. プロデューサーの役割

### プロデューサーとは

オンライン研修において、講師の運営をサポートするスタッフのこと。テクノロジー面を担当することで、講師がファシリテーションに集中できる。

### プロデューサーの役割一覧

**研修前：**
- 参加者へのアクセス情報・資料の送付
- 技術環境の事前テスト
- リハーサルへの参加

**研修中：**
- 参加者がスムーズにログインできるようサポート
- 参加者が資料・データをダウンロードできるようサポート
- テクニカルトラブルへの対応
- 投票・小グループ（ブレイクアウトルーム）の操作
- チャットの監視・整理（多い質問を特定する）

**リハーサルで確認すること（10名超の場合は必須）：**
```
● どのタイミングでどのスライドを共有するか
● スライド共有に問題がないか
● 投票機能を使うタイミングと投票内容
● ブレイクアウトルームを使うタイミングと設定内容
● チャットに書き込んでもらうタイミングと内容
```

## 9. 参加者を巻き込むアクティビティの進め方

### アクティビティ実践の8つのテクニック

| # | テクニック | ポイント |
|---|-----------|---------|
| 1 | **結論を先に伝える** | 最終的なアウトプット（何を完成させるか）を最初に示す |
| 2 | **ゆっくり、はっきり、ていねいに** | 「わかっているだろう」は禁物 |
| 3 | **「たとえば...」で方向を示す** | 具体例で参加者が何をすべきかを明確にする |
| 4 | **注目を集めてから始める** | 参加者全員が講師に注目してからインストラクション開始 |
| 5 | **ていねいな表現で笑顔で言い切る** | 語尾を明確に。「〇〇してください」 |
| 6 | **文字化する** | 口頭＋チャットに文字で同じ内容を示す |
| 7 | **ツールの使い方を説明する** | オープニングで操作に慣れてもらう機会を設ける |
| 8 | **リーダーはあらかじめ決めておく** | 分かれてから決めると混乱が生じる |

### インストラクションの8ステップ（グループワーク前）

```
1. これから何をするかを言う
2. 具体的な内容を伝える
3. リーダーを決める（方法を指定する）
4. 時間を伝える
5. 内容を再確認する（「確認ですが...」）
6. 発表の有無を伝える
7. 質問がないかを確認する
8. グループへ送り出す
```

## 10. 問いかけ・質疑応答の設計

### クローズドクエスチョンのガイドライン

- 全員が同時に回答できる形式にする（投票、チャット、スタンプ等）
- 「OKな人は手を挙げて」ではなく「AかBかをチャットに入力してください」

### オープンクエスチョンのガイドライン

- 個人で考える時間（60秒など）を設けてから発言を求める
- チャットを活用して全員の意見を同時に引き出す
- 「何か質問はありますか？」はNG（誰も答えない雰囲気になりやすい）

**「何か質問はありますか？」の代替案：**
```
● 途中（40分経過時点など）で質問時間を設ける
● 「チャットに質問を書いてください」で書きやすい環境をつくる
● プロデューサーが質問を整理し、多い質問から回答する
● 回答しきれなかった質問は後日メール等でフォローする
```

## 11. 当日の運営チェックリスト

### 事前準備

```
□ 使用するプラットフォームの機能を練習済み
□ 30分前にログインして環境チェック
□ バックアップ機器（予備PCで参加者視点を確認）
□ タイマーとアジェンダを見やすい位置に配置
□ プロデューサーとリハーサル実施済み（10名超の場合）
□ カメラのすぐ近くに視線を向ける目印を配置（カメラ目線のため）
```

### 開始時

```
□ ソフトオープニング開始（3〜5分前）
□ 時間通りに正式開始
□ オープニングアクティビティ（1〜2分）
□ グラウンドルールの確認（必要に応じて）
```

### 進行中

```
□ 4分に1回は何らかの参加者巻き込みを実施
□ エナジャイザーを適切なタイミングで挿入
□ チャット・投票等のツールをバランスよく組み合わせる
□ 「間」・「沈黙」を恐れず、個人で考える時間を確保する
```

### 終了時

```
□ 2〜3分早めに終わることを目指す
□ クロージングアクティビティ実施
□ アクションプランを書いてもらう
□ アンケート案内
□ 今後につながる問いかけで締めくくる
```

## 12. 困った場面への対処法

### 研修設計の悩み

| 悩み | 対処法 |
|------|--------|
| どうしても一方的に話してしまう | まず「全員が答えられるクローズドクエスチョン」を設計する。投票・チャットを使う |
| 反応がないと話しにくい | 参加者の反応を「見る」のではなく「設計する」発想に転換する |
| 配付資料を印刷して送れない | デジタルワークシートを事前送付。研修中はチャット等で書き込みを促す |
| アクティビティのネタが思いつかない | チャット・投票・ホワイトボード・30秒チャット等の基本形を組み合わせて応用する |
| 一部が対面・一部がオンラインのリクエスト | ハイブリッド研修は設計が複雑。→ BLENDED-LEARNING.md 参照 |

### テクニカルトラブル

| 状況 | 対処法 |
|------|--------|
| 接続不安 | プロデューサーを配置。バックアップ機器を用意 |
| 初めてのプラットフォーム | 事前に参加者として別の研修に参加して体験する |
| 自分のカメラ映りが気になる | カメラ近くにモチベーションを上げる写真を貼り、カメラ目線で話す練習をする |

### 参加者対応

| 状況 | 対処法 |
|------|--------|
| 参加者の様子が見えない | 定期的に全員参加の仕組み（投票・チャット）を使って状況を把握 |
| ディスカッションが活発でない | リーダー指定 + 4分間のタイマー設定 + 「たとえば...」で方向を示す |
| 同じ人ばかり発言する | 全員がリアクションできる手法（チャット・投票）を使う。リーダーをランダムに変える |
| 集中していない参加者がいる | 4分ごとの巻き込みを徹底する。エナジャイザーを挿入する |

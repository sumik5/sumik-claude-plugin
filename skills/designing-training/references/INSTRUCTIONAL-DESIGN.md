# インストラクショナルデザイン（ID）リファレンス

研修設計を体系的・科学的に行うためのID理論体系。
習得主義・ADDIEモデル・TOTEモデル・ガニェ5分類から「教えない研修」の設計原則まで、IDの中核をカバーする。

---

## 1. IDの基本概念

### 「教えること ≠ 学ぶこと」

研修の本来の目的は「教えること」ではない。

> 「研修の目的とは、教えることではない。それは学習者が自ら学ぶことを手助けし、学習者に変化が起こることだ」

- 教えることはあくまで手段であり、目的ではない
- 成果につながる行動ができる人材育成のみならず、職場に「学習する文化」を拡げることが指向される

### 習得主義 vs 履修主義

| 概念 | 説明 | 問題点 |
|------|------|-------|
| **履修主義** | 一定時間学習したことを修了の条件とする（例：8時間出席で修了証） | 何も学ばなくても時間さえ経過すれば修了できる |
| **習得主義** | 学習目標を達成したことを修了の条件とする | IDが支持する考え方 |

キャロルの学校学習の時間モデル（Carroll, 1963）は「能力の差があるからできない子はいくら時間をかけてもむだだ」という考えから「人によって必要な学習時間の差があるから同じ時間だけ学習すれば差がつくのは当たり前だ」という考えへのパラダイムシフトを示した。これが習得主義の基盤となり「できるまで取り組み、できたら先に進む」完全習得学習（Mastery Learning）につながる。

## 2. ADDIEモデル

研修設計の標準的なプロセスモデル。5フェーズで構成される。

```
Analysis（分析）
    ↓
Design（設計）
    ↓
Development（開発）
    ↓
Implementation（実施）
    ↓
Evaluation（評価）
    ↓（フィードバックループ）
```

| フェーズ | 主な活動 | 主な成果物 |
|---------|---------|----------|
| **Analysis** | ニーズ分析・参加者分析・課題分析・学習環境分析 | ニーズ分析レポート |
| **Design** | 学習目標設定・評価設計・教授戦略・構成設計 | インストラクショナルデザイン文書 |
| **Development** | 教材開発・コンテンツ作成・パイロットテスト | 研修教材・ファシリテーターガイド |
| **Implementation** | 研修実施・ファシリテーター訓練 | 研修実施記録 |
| **Evaluation** | 効果測定・フィードバック収集・改善提案 | 評価レポート |

**ADDIEの特徴:** 各フェーズは順に進むが、前フェーズへのフィードバックループを持つ反復的なプロセス。「分析なきデザイン」「設計なき開発」が最も多い失敗パターン。

## 3. TOTEモデル

入口管理と出口管理の設計思想。

```
Test（入口評価：参加者はすでに知っているか？）
    ↓ 知らない
Operate（教育・学習活動）
    ↓
Test（出口評価：学習目標を達成したか？）
    ↓ 達成     ↓ 未達成
  Exit      → 再Operate（追加学習）
```

### 入口管理の考え方

参加者がすでに知っていることを繰り返し教えるのは「履修主義」の典型的な問題。

- 研修の入口で事前テストや確認を行い、知識レベルに応じて内容を調整する
- 「確実に理解してもらいたいので基本から丁寧に説明する」という姿勢は、参加者に「知っていることを長々と聞かされる」苦痛を与える

### 出口管理の考え方

- 研修の出口では、学習目標を確実に達成させるために何度でもテストの機会を与える
- テストは「能力評定」ではなく、「でき具合を見守り、次の手段を講ずるためのチェック」として機能させる
- 合格するまでは修了させないという覚悟が習得主義を実現する

### TOTEモデルが支持する設計

| ❌ 典型的な履修主義 | ✅ TOTEモデルに基づく設計 |
|------------------|----------------------|
| 8時間受講で修了証を発行 | 学習目標達成で修了 |
| 全員に同じ時間をかける | 達成に必要な時間は個人差がある |
| 入口テストなしで一斉講義 | 事前確認で知識レベルに応じた設計 |

## 4. ガニェの学習成果5分類

IDの産みの親と言われるロバート・M・ガニェ（Roberg M. Gagne, 1916-2002）が1970年代に提案した学習成果の分類枠組み。よく知られるブルームのタキソノミーと同時代の成果。

### 5分類の概要

| 分類 | 領域 | 説明 | 典型的な評価方法 |
|------|------|------|--------------|
| **言語情報** | 認知的（あたま） | 事実・概念を「知っている」状態 | 記憶確認テスト・穴埋め |
| **知的技能** | 認知的（あたま） | ルールを応用して問題を解ける能力 | 新しい問題を解かせる |
| **認知的方略** | 認知的（あたま） | 自分の学習・思考を管理する能力 | プロセス評価・ポートフォリオ |
| **運動技能** | 精神運動（からだ） | 身体を使ってできる能力 | 実技テスト |
| **態度** | 情意（こころ） | 特定の行動を選択しようとする傾向 | 行動観察・関連知識テスト |

### 知的技能の下位分類（単純→複雑の順）

1. **弁別** — AとBを区別する（例：正常品と不良品を見分ける）
2. **具体的概念** — 具体例を分類する（例：この図形は三角形か）
3. **定義された概念** — 定義を使って分類する（例：コンプライアンス違反かどうか判断する）
4. **ルール学習** — ルールを適用する（例：計算式を使って解く）
5. **問題解決** — 複数のルールを組み合わせて新しい解を導く

### 整合性（Alignment）の原則

> 「学習成果のタイプごとに妥当な評価方法と教え方が異なる」 — ガニェ

- 「よいプレゼンのコツを知っていること（言語情報）」と「実際にプレゼンができること（運動技能）」は別
- ルール応用力（知的技能）の評価には「新しい問題を解かせる」必要がある。過去に解いた問題では丸暗記で正答できてしまうため
- 研修でめざすこと（学習目標）・評価方法・研修内容の3つが合致していることが整合性であり、IDで最も重視する基本要件

**実践上の罠：**

```
よくある間違い: モチベーション研修でも態度（A）の問題として設計してしまう
正しい分析: 実際には「やり方を知らない」（知識K・スキルS不足）のに、
           「やる気がない」（態度A不足）と診断する誤り
```

## 5. 「教えない研修」の設計原則

### 自己啓発・OJTを主軸に考える

研修（フォーマル学習）への投資はパフォーマンス支援全体の2〜5%にすぎない（Rosenberg, 2006）。残りの95〜98%は職場での学習とその支援に向けられている。研修担当者が従来の担当職務にとどまることなく、責任範囲を広げることが必要。

### 熟達化レベルに応じた支援調整

| 熟達レベル | 特徴 | 適切な支援スタイル |
|-----------|------|----------------|
| **初心者** | 基礎知識が不足・戸惑いが多い | 構造化された教授・詳細なフィードバック |
| **中堅** | 基礎はあるが応用が難しい | ケーススタディ・問題解決演習 |
| **熟達者** | 状況の特異性に適応的に対応できる | 情報環境の整備・ネットワーク支援 |

熟達者レベルをめざす研修では、基礎知識習得より「自ら参照できる資源へのアクセスをしやすくする情報環境を整えること」や「学び合えるネットワーク作り」が適切な支援となる。

### 「教えない」ための設計変更4方向

| 変更の方向 | 問い | 具体例 |
|-----------|------|-------|
| **加える（Add）** | 研修以外の要素との連携に必要な研修は何か？ | 研修前の情報提供・基礎知識確認、研修後のアクションプランとフォローアップ |
| **削る（Delete）** | 研修以外の手段に移行できることは何か？ | 予習・自学自習に移せる知識インプット部分 |
| **動かす（Move）** | 基礎からではなく応用から入れないか？ | EAT（経験→気づき→理論）の順序に変更、受講時期の見直し |
| **変える（Modify）** | 集まってやる必要があるか？ | スリム化、eラーニングへの移行、ゴールの再設定 |

## 6. GBS（Goal-Based Scenario）理論

シャンク（Schank）によるゴールベース・シナリオ理論。「最も注目しているIDモデルのひとつ」（本書著者）として紹介されており、**ストーリー中心型カリキュラム**として発展。

### 設計の考え方

学習者が達成したいゴールを中心にシナリオを設計し、そのゴールを達成する過程で必要な知識・スキルを学ぶ構造。

```
明確なゴール設定
    ↓
ゴール達成に向けた実践的シナリオ
    ↓
失敗・挑戦を通じた学習
    ↓
フィードバックによる振り返り
    ↓
知識・スキルの定着
```

### 適用場面

| 形式 | 説明 |
|------|------|
| ビジネスシミュレーション型研修 | 現実的なビジネス課題をシナリオとして提示 |
| ケースメソッド型学習 | 実際の事例から分析・意思決定を学ぶ |
| プロジェクト型学習（PBL） | 実際のプロジェクト遂行を通じて学ぶ |

## 7. 肯定的学習環境の30指標（Tobin, 2000）

職場が学習を支援する肯定的な環境かどうかを評価するための指標。Mike Kunkle作成の15指標にTobinが15項目を追加した計30指標。

### 主要カテゴリ別指標

| カテゴリ | 代表的な指標 |
|---------|-----------|
| **対話・提言** | □ アイディアへの公式・非公式な反応がある □ 提言が歓迎される □ ブレインストーミングが一般的 |
| **失敗への態度** | □ 失敗が教育とみなされる |
| **学習投資** | □ 会社が業務関連の購読費・参加費等を払う □ 飲食持ち込み可のセミナーが定期的に開催される |
| **評価・成長** | □ 360度の調査が行われる □ 能力の評価が社員の学習・成長に関連づけられている |
| **OJT・コーチング** | □ OJTが使われている □ コーチングが一般的 □ Job shadowingが奨励される |
| **キャリア・メンタリング** | □ キャリアパスが部局を越えて開かれている □ すべての職位レベルでメンターのプログラムがある |
| **継続改善** | □ 社員が常によりよい実践を求めている □ 社員がミーティングを楽しみにしている |

**活用方法:** 研修担当者の努力と工夫によって、少しでも組織がより肯定的な学習環境に変貌することを目指す際のチェックリストとして使用する。

## 8. 研修以外の実現方法

### 研修は唯一の解決策ではない

人材育成を「研修で学ぶ」だけで考えるのは部分最適。4つの側面で現状と理想を把握する。

| 学習形態 | 説明 | 代表的なツール・方法 |
|---------|------|-----------------|
| **研修で学ぶ** | フォーマル学習（対面・eラーニング） | 集合研修、eラーニング、OJT計画的研修 |
| **情報で学ぶ** | ナレッジマネジメント・マニュアル | KMS、ジョブエイド、マニュアル集、EPSS |
| **経験して学ぶ** | 職場での実践・OJT | OJT、メンタリング、ストレッチアサインメント、コーチング |
| **仲間から学ぶ** | 実践コミュニティ・インフォーマル学習 | CoP、勉強会、ピアコーチング、相互コーチング |

**診断方法:** 各側面の現状実現度（0〜100%）を評価し、理想との差を確認することで改善優先度を把握できる。

### 改善のための6つの戦略（設計ハンドブックより）

| # | 戦略 | 特徴 |
|---|------|------|
| 1 | **システム改善** | 業務プロセス・ツール・環境の変更 |
| 2 | **組織開発プログラム** | 文化・構造・関係性の変革 |
| 3 | **採用** | 能力をもつ人材の獲得 |
| 4 | **人材配置** | 適材適所の配置変換 |
| 5 | **コーチング** | 個人への継続的支援 |
| 6 | **トレーニング（研修）** | 知識・スキル・態度の習得支援 |

基本的には①〜⑤のいずれかだけで解決できない場合に「⑥研修」という選択肢を選ぶ。

## 9. インフォーマル学習の活性化

研修担当者がインフォーマル学習を支援するためにできること（ロセット・ホフマン, 2013）。

| アクション | 内容 |
|-----------|------|
| **①見つける** | 組織内に現存するグループや活動内容、用いている手段（自作ジョブエイド・ブログ・Wiki・知識ベース等）、活動の成果や効用を調査する |
| **②学ぶ** | インフォーマルな学びの成果をとらえ、研修やKMSに組み込むべき（再利用可能な）内容を探す |
| **③高く評価する** | 活動の成果を認めて、その活動がやりやすくなるように援助する。研修の中にインフォーマル学習のリーダーを登用することも有効（インフォーマルであることのよさを失わないように） |
| **④支援する** | 活動が発展するように支援する。導入研修で紹介、グループ同士を結び付けて相乗効果を狙う、スペースを提供するなど |
| **⑤役割を再定義する** | 研修担当者・インストラクタの役割を見直し、インフォーマル学習の支援（コーチングなど）により多くの時間を割り当てる |

## 実践チェックリスト

### 研修設計開始時の確認

```
□ この問題は本当に研修で解決できるか確認したか？（6戦略で検討）
□ 習得主義の視点で学習目標を設定したか？
□ TOTEモデルで入口・出口管理の設計をしたか？
□ ガニェの5分類で学習目標のタイプを明確にしたか？
□ 評価方法と学習目標の整合性（Alignment）を確認したか？
□ 研修以外の実現方法（4側面）も検討したか？
□ 職場の学習環境（Tobin指標）を把握したか？
```

### 研修設計完了時の確認

```
□ 学習目標・評価方法・研修内容の整合性がとれているか？
□ 言語情報だけでなく知的技能・運動技能・態度の目標も設定したか？
□ 熟達レベルに応じた支援レベルの調整ができているか？
□ インフォーマル学習・OJTとの連携を設計したか？
□ 「教えない」ための余白（自己啓発・職場実践の機会）が確保されているか？
```

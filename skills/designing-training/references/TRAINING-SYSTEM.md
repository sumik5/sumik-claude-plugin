# 研修体系設計・Off-JT体系リファレンス

STEP 2-1: Off-JTとしての研修体系策定の実践手順。
4プロセスによる研修体系の構築から、次世代リーダー育成・360度サーベイ・eラーニング・研修会社選定まで包括的に解説。

---

## 研修体系とは

社員を階層・職種などの組み合わせにより分類して、どの階層・職種の社員に対してどんな研修を行うのかを整理・一覧化したもの。

**研修体系の縦軸と横軸の例**

| 縦軸（階層） | 横軸（分類） |
|------------|------------|
| 新人・若手層 | 共通研修 |
| 中堅・チームリーダー層 | 部門別研修 |
| 管理職層（部長級・課長級） | 選抜研修 |
| 役員層 | |

> 縦軸・横軸の切り分け方は会社によって異なる。自社の等級制度に合わせて設計する。

---

## 研修体系策定の4つのプロセス

```
プロセス① 人材強化課題の抽出
    ↓
プロセス② 現状の研修課題の抽出
    ↓
プロセス③ 研修体系の設計
    ↓
プロセス④ 実施計画の設計
```

---

## プロセス①：人材強化課題の抽出

### 3段階の進め方

#### ステップ1：等級定義とスキル要件の確認

等級制度は社員を位置づける階層（ランク）の枠組み。研修体系の策定にあたっては、等級定義・詳細定義をしっかり把握することが基本。

**確認すべき内容**
- 等級ごとの役割・職責の定義
- 各等級で求められる能力・スキル要件
- 求める人材像との対応関係

> 等級定義が明文化されていない場合は、この段階を省略して次に進むことも可。

#### ステップ2：内部環境の強化ポイントを分析する（バリューチェーン分析）

**バリューチェーンの分類**
- **主活動（直接部門）**: 開発・購買・製造・物流・マーケティング・セールス
- **支援活動（間接部門）**: 経理・人事・経営企画

**分析手順**
1. 各機能の活動内容を細分化
2. 細分化された各活動のコスト・付加価値・競合比較（強み・弱み）を把握
3. 現在の経営戦略を実現する際の、強化すべき機能・職種を特定

**経営戦略別の強化すべき機能の例**

| 経営戦略 | 強化すべき機能 |
|---------|------------|
| 既存商品の市場浸透 | マーケティング機能 |
| 販売代理店・小売店拡大 | セールス部門 |
| 新規事業・多角化 | 事業開発・新製品開発機能 |

#### ステップ3：強化すべき人材・課題を選定する

バリューチェーン分析で特定した機能・職種において、階層別にどこが課題かを検討する。

**課題の切り口例**
- 現場人材のスキル不足か
- 中間マネージャーのマネジメントスキル不足か
- 部門長の他部門との調整スキルが課題か

> 課題把握のためには部門責任者へのヒアリング・アンケートが有効。

---

## プロセス②：現状の研修課題の抽出

人材強化課題と照らして、現在の研修体系の見直しが必要な点を洗い出す。

**3つの観点**

| 観点 | 内容 |
|------|------|
| 誰に | 階層別・特定個人別の視点 |
| 何を | 研修テーマ別の視点 |
| 方法 | 期間・頻度・実施方法（オンライン/対面）の視点 |

**追加すべき研修の例**
- マネジメント研修は実施しているが、傾聴研修が実施されていない → 追加を検討

**変更すべき研修の例**
- キャリア研修を入社5年後 → 3年目に変更
- コーチング研修を部門長のみ → 主任クラスにも拡大
- 1日研修 → 2日研修に増量

---

## プロセス③：研修体系の設計

### 設計の3つの前提観点

1. **戦略との一貫性**: 経営戦略を実現するための人材育成であるという原点を確認
2. **人事制度内の整合性**: 等級・評価制度と齟齬なく実施されるか確認（特に人事企画と人材開発が別部署の場合は注意）
3. **選択と集中**: 限られた予算・時間の中で、経営戦略との関連において優先順位をつける

### 研修プログラム設計の3つの観点

#### 誰を対象とするか
- 全員一律か、選抜か
- 階層・部門・職種の限定

#### 何を目的とするか
- 成長目標（教育の目的）
- コンテンツのテーマ

#### どのように実施するか
- 講師: 社内講師 vs 外部講師
- 実施方法: オンライン vs 集合研修
- 期間・回数の設定

### 研修実施の注意点

| タイミング | よくある失敗 | 対策 |
|---------|------------|------|
| 実施前 | 研修の目的・目標が考えられていない | 人材課題から明確な目的を設定 |
| 実施中 | 実務とのつながりがない | 研修内でOJTと結びつける機会を用意 |
| 実施後 | 周りを巻き込む仕掛けがない | 上司・チームを巻き込む仕掛けを設計 |
| 実施後 | 定着の仕掛けがない | 2〜3年後まで繰り返し実践できるフォロー体制 |

### 研修体系図の作成

```
              │ 共通研修 │ 部門別研修 │
──────────────┼──────────┼────────────┤
役員層        │ 経営戦略  │           │
              │ 幹部育成  │           │
──────────────┼──────────┼────────────┤
管理職層      │ マネジメント│ 部門専門  │
              │ 360度サーベ│ 技術研修  │
──────────────┼──────────┼────────────┤
中堅・チームリーダー│ ロジカルシンキング│ OJT指導   │
              │ プロジェクト│ スキルアップ│
──────────────┼──────────┼────────────┤
新人・若手層  │ ビジネス基礎│ 職種別    │
              │ コンプライアンス│ 入門研修  │
```

**研修の種別判断**

| 種別 | 対象 | 内容 |
|------|------|------|
| 階層型 | その階層の全員 | 全員が受ける必須研修 |
| 選抜型 | その階層の一部 | 特定の能力開発・次世代リーダー育成等 |
| 職種別 | 職種に関係なく | 職種に必要な知識・スキルを学ぶ研修 |

## 各論①：次世代経営リーダー育成（7ステップ）

多くの企業で73%が取り組みができていないとされる重要課題。長期的視野でのロードマップと、Off-JT・OJT・モニタリングのサイクルが不可欠。

### 設計・運用の7ステップ

**ステップ1：後継者計画のロードマップ立案**

現経営陣の退陣からバックキャスティング（逆算）で必要人数を算出。
- 役員数の数倍程度の余裕のある候補者プールを設計
- 若手（20〜30代前半）から長期的な育成を開始

**ステップ2：後継ポジションの人材要件の策定**

次世代経営リーダーとしての「求める人材像」を設計。求める人材像フレームワーク（全社版）をベースに、経営リーダー固有の要件を追加。

**ステップ3：後継者候補の選抜**

人材要件の「スタンス」部分は後から変えにくいため、選抜時に重点的に見極める。

| 選抜方法 | メリット | デメリット |
|---------|---------|----------|
| 自薦 | やる気のある人材が選出 | 研修・勉強志向で実践力が低い場合あり |
| 他薦 | 客観的に優秀な人材が選出 | 部門長が「うちのエースを手放したくない」 |

> 専門家による外部アセスメントも活用を推奨（経営トップ候補の場合）。

**ステップ4：Off-JTによる育成**

一般的な1日研修とは大きく異なる、9か月程度の本格的プログラム。

| 要素 | 内容 |
|------|------|
| 事前学習 | 経営理論・業界分析等の自己学習 |
| 社外派遣 | 社外との人脈形成・多様な視点獲得 |
| フィールドワーク（OJT） | 実際のプロジェクト参加 |
| 中間プレゼンテーション | 進捗確認・フィードバック |
| 最終プレゼンテーション | 事業計画策定・提案 |

**ステップ5：OJTによる育成（最重要・最も難しい）**

修羅場経験（タフアサインメント）が次世代経営リーダー育成のキモ。

- 前ステップで提案した事業を実際にリーダーとして立ち上げ
- グループ会社への経営幹部として出向
- 事業部長・課長への昇格

> **重要**: OJTのポジション・チームを用意できなければ、ステップ4までは無駄になる。
> 経営トップの意思が必要。人事担当者は事前にグループ会社・現場部門と交渉しておく。

**ステップ6：育成状況のモニタリング**

ステップ2の要件定義を基に、Off-JT・OJTでの能力習得状況を定期的に評価。
実績（新規事業の成果・子会社業績等）も評価対象。

**ステップ7：指名・任用後のサポート**

エグゼクティブコーチングやメンタリングによる継続的サポート。
フィジカルとメンタルのコンディション管理も重要。

## 各論②：360度サーベイ

被評価者の上司・先輩・部下など多様な立場の周囲の人が評価者となり、評価内容をフィードバックする育成施策。

### 360度サーベイの特徴

**メリット**
- より客観的・公正な評価が可能
- ハラスメントなどの問題行動の発見
- 評価の納得感・気づきが得られやすい
- モチベーション向上・行動改善に結びつきやすい

**デメリット**
- 実施に多くの人が関わるため手間・時間がかかる
- 運用上の注意点が多く、準備が不十分だと失敗しやすい

### 360度サーベイの推奨用途

**人事評価より能力開発への活用を推奨**

人事評価（処遇）に紐づけると：評価が中庸化しやすい。
能力開発に使う場合：率直な意見が得られやすく、エンゲージメント向上に繋がりやすい。

### よくある失敗と対策

| 失敗パターン | 原因 | 対策 |
|------------|------|------|
| 評価を受け入れられない | 被評価者が評価に不満・不信感 | 事前ガイダンス・目的の説明を徹底 |
| 犯人捜しをしてしまう | 低評価者を特定しようとする | 匿名性の担保を明確に説明 |
| 誹謗中傷の記載 | 評価と人格攻撃の混同 | 評価と誹謗中傷の違いをマニュアル化 |
| 回答が全て標準点 | 雑な評価 | メリハリのある評価の重要性を説明 |
| やりっぱなし | 結果の活用なし | フィードバック研修・フォローアップ計画を含む一貫した計画を立案 |

## 各論③：eラーニングとLMS

### eラーニングの特徴

**メリット**
- 導入費用が比較的安価
- 受講者が自分の興味に沿ったコンテンツを手軽に学習
- 一方的な情報伝達型の集合研修より効率的

**デメリット**
- 社員の自主性に任せると形骸化しやすい
- 受講が続かなくなりやすい

### LMS（Learning Management System）の活用

eラーニングの継続的利用促進・研修管理の効率化ツール。

**受講者向け機能**
- 受講済み講座の確認・モチベーション維持
- 進捗・課題提出状況の確認
- おすすめ講座の提案
- 理解度確認のための小テスト

**人事担当者向け機能**
- 教材配信・受講状況・成績の一元管理
- 受講アラート・事前課題・アンケートの授受
- 業務効率化

**LMSの種類**

| タイプ | 特徴 |
|------|------|
| 汎用タイプ（単体システム） | 研修全般を管理可能。利用料は比較的高額 |
| eラーニング教材セット | eラーニングと合わせて購入 |
| eラーニング組み込み型 | eラーニングシステムに標準搭載 |

> 現在はクラウドサブスクリプションタイプが増加。導入時は最小限の契約でスタートし、利用実態に応じて変更可能。

## 各論④：研修会社の選定

### 基本方針

- たとえ同一テーマの研修でも、理想的には毎回ゼロベースで選定し直す
- 研修会社も日々コンテンツをブラッシュアップしているため

### 選定プロセス

1. 大手研修会社（2〜3社）のWebサイト・資料で情報収集
2. 見積もりを取り、面談を実施
3. 評価フォーマットにまとめ、比較検討

**評価軸**
- 価格（コストパフォーマンス）
- コンテンツの広さ・深さ
- 講師の質・実績
- カスタマイズの柔軟性
- アフターフォローの充実度

## プロセス④：実施計画の設計

### 実施計画の必要性

研修体系は「一時点での必要な研修」を示すが、時間軸が含まれていない。
いつ・どのような順序で実施するかをまとめるのが実施計画。

### 計画期間

- 一般社員向け: 年度計画（毎年繰り返し）
- マネージャー・次世代リーダー向け: **3年間の中期計画**

### 実施の優先順位（重要）

> **原則: 上の階層から取り組む**

部下が研修で学んだ新しい概念・スキルを実践しようとしても、上司が研修未受講で理解できないと、部下の成長意欲を摘んでしまう。

**実施計画立案フロー**
```
Year 1: 役員・部長層 → 管理職層の研修実施
Year 2: 管理職層 → 中堅・チームリーダー層の研修実施
Year 3: 若手・新人層の研修実施、全体フォローアップ
```

> **AskUserQuestion**: 実施計画の詳細（年度・予算・参加人数）については
> 会社の状況に応じてユーザーに確認が必要。

## 研修体系の運用ポイント

### 定期的なチェック・更新

研修体系の更新は、以下を契機に実施：
- 人事制度全体の見直し
- 経営トップ交代・部門再編

ただし、大きなきっかけがなくても定期的に検討すべき。

### KPI設定による効果測定

| KPI指標 | 確認方法 |
|--------|---------|
| 研修満足度 | アンケート・インタビュー・組織サーベイ |
| 受講者数・受講率 | LMS管理データ |
| 研修コスト（教育投資額） | 経費管理 |
| 研修・学習時間 | LMS管理データ |
| スキル・資格の保有者数 | スキルマップ・資格管理システム |
| 育成計画の達成度 | 経営幹部候補者数等 |

> 人的資本開示の流れから、KPIを設定し情報公開する動きが増加。

# プログラム設計・カリキュラム構築ガイド

研修プログラムの目的設定からカリキュラム構築・タイムテーブル設計・コンテンツ構成まで、研修を設計する際の実践的な指針。

---

## 1. 研修プログラム設計の全体像

研修プログラムは「教育施策」の中核をなす。目的・目標・コンテンツ・手法・時間配分を整合させることで、初めて受講者の行動変容が生まれる。

### 設計の4ステップ

```
STEP 1: 目的・目標設定
         ↓（目的から逆算）
STEP 2: カリキュラム設計（学習内容・順序）
         ↓（コンテンツを決める）
STEP 3: 手法・形式選定（座学・ワーク・フィールド）
         ↓（手法を決める）
STEP 4: タイムテーブル設計（時間配分）
```

設計は必ず「目的から逆算」で行う。タイムテーブルは最後に決める。

---

## 2. 研修の目的・目標設定

### 研修に期待できる3つの効果

| 効果の種類 | 内容 | 具体例 |
|-----------|------|--------|
| **学習効果** | 知識・概念の習得 | ロジカルシンキング習得、業界知識理解 |
| **スキル習得効果** | 行動できるレベルへの到達 | プレゼンテーションができる、課題設定ができる |
| **行動変容効果** | 研修後の実務での行動変化 | 顧客との向き合い方が変わる、フィードバックを積極的に行う |

> **重要**: 人材育成の観点から最も価値が高いのは「行動変容効果」。研修は非日常の場であり、その非日常性を活かして受講者の気づきを醸成することが求められる。

### 行動変容型目標の書き方

良い目標は「研修終了後に〇〇ができるようになる」という形で書かれる。

**NG例**: 「イノベーションについて理解する」
**OK例**: 「顧客の不満・不便・不安（「不」）を探索し、事業案のフレームに落とし込める」

3つの観点で目標を確認する:
1. **何を学習するか** → クローズドスキル（プログラミング等、ルールが一つ）かオープンスキル（マネジメント等、状況適応型）かを区別する
2. **何を効果とするか** → 受講者本人・所属組織・社会への効果を整理する
3. **どのように効果を高めるか** → 受講前の状況把握・プログラム最適化・研修後の実践支援体制を設計する

## 3. 研修タイプの分類と選定

### 目的別の研修タイプ

| 研修タイプ | 主な目的 | 対象者 | 特徴 |
|-----------|---------|--------|------|
| **マネジメント強化** | 組織力最大化・部下育成 | マネジャー層 | 基礎から自律型まで段階あり |
| **若手・中堅強化** | 主体性・行動変容促進 | 若手〜中堅 | 対象数が多く組織力底上げに効果大 |
| **営業強化** | スキルアップ・顧客価値向上 | 営業・販売職 | 現場直結、即効性高い |
| **キャリア開発** | 自律的なキャリアデザイン支援 | 全層 | 個人の「やりたい」を軸にする |
| **グローバル人材開発** | 異文化理解・グローバル対応力 | 海外関連部門 | 言語 + 異文化コミュニケーション |
| **次世代リーダー開発** | 将来の経営者・幹部育成 | 選抜メンバー | 長期・ヘビー級が多い |

### 対象者による設計の違い

```
新入社員    → 基礎知識習得・ビジネスマインド形成・業務慣れ
若手（〜4年目）→ スキル強化・主体性醸成・チームワーク
中堅（5〜10年）→ 課題解決力・リーダーシップ萌芽
管理職      → マネジメント原則・部下育成・組織横断連携
部長・経営幹部→ 経営視点・戦略立案・次世代リーダー育成
```

### 受講者の選抜方式

| 方式 | 特徴 | 向いている場面 |
|------|------|--------------|
| **全員受講型** | 新入社員研修等、階層全員対象 | 基礎知識・コンプライアンス・方針徹底 |
| **選抜型** | 評価や推薦で受講者を決定 | 次世代リーダー・イノベーション人材発掘 |
| **挙手型（エントリー制）** | 受講希望者が自ら申し込む | キャリア開発・専門スキル向上 |

## 4. 研修形式の選定

### 提供形式と特徴

| 形式 | 内容 | 長所 | 短所 |
|------|------|------|------|
| **対面集合** | 同一会場で実施 | グループダイナミクス最大化・非日常体験 | 交通・会場コスト大 |
| **オンライン（同期型）** | Web会議で実施 | 距離不問・低コスト | 発言しにくい・集中力低下リスク |
| **オンライン（非同期型）** | eラーニング・動画学習 | 自分のペースで学習 | 双方向性なし |
| **ハイブリッド** | 対面＋オンライン混在 | 柔軟性高い | ファシリテーション難易度高い |
| **ブレンディッドラーニング** | 複数形式の組み合わせ | 深い学習・行動変容促進 | 設計・運営コスト大 |
| **アクションラーニング** | 研修とフィールドワークを交互に実施 | 実践直結・高い行動変容 | 期間が長い・運営負荷大 |

### 研修形式選択のフロー

```
目的が「知識習得・情報共有」のみ → オンライン非同期（eラーニング）
       ↓
目的が「スキル習得・演習必要」 → 対面集合 or オンライン同期
       ↓
目的が「行動変容・深い気づき」 → 対面集合 + ブレンディッド要素
       ↓
目的が「実際の事業成果・イノベーション」→ アクションラーニング型
```

### 手法・メソッド一覧

| 手法 | 内容 | 効果 |
|------|------|------|
| **講義** | 講師からのインプット | 知識習得・概念理解 |
| **セルフワーク（内省）** | 個人で考える時間 | 気づき・自己理解 |
| **グループワーク** | 複数人でのディスカッション | 認知ギャップ・グループダイナミクス活用 |
| **フィールドワーク** | 現場・顧客への直接アプローチ | リアルな課題発見・行動変容 |
| **ロールプレイ** | 場面を模擬的に体験 | スキルの実践練習 |
| **ケーススタディ** | 事例を分析・討議 | 思考力・判断力向上 |
| **プレゼンテーション** | 受講者による発表 | 言語化・アウトプット力 |

## 5. カリキュラム設計の手順

### カリキュラム設計の基本構造

```
① 目標設定（学習者が研修後に何ができるようになるか）
       ↓
② 学習内容の洗い出し（目標達成に必要な知識・スキル・態度）
       ↓
③ 学習順序の設計（前提となる知識 → 応用へ）
       ↓
④ 手法・形式の割り当て（各コンテンツに適した手法を選ぶ）
       ↓
⑤ 時間配分の設計（全体時間 → 各パートへの配分）
```

### コンテンツ構成の原則：導入→展開→まとめ

| フェーズ | 目的 | 配分目安 | 実施内容 |
|---------|------|---------|---------|
| **導入** | 研修の目的・ゴール共有、モチベーション喚起 | 10〜15% | アイスブレイク、ゴール共有、現状確認 |
| **展開** | コア学習・演習・内省 | 70〜75% | 講義、グループワーク、演習、フィードバック |
| **まとめ** | 学習の定着・行動宣言 | 15〜20% | 振り返り、アクションプラン作成、共有 |

> **重要**: まとめフェーズを省略しがちだが、研修後の行動変容には「自分は何を変えるか」を宣言する時間が必須。

### 学習の流れを設計するときの注意点

- 「知識 → 考える → 試す → 振り返る」の経験学習サイクルを組み込む
- グループワークと個人ワークを交互に配置する（集中力維持）
- フィードバックの機会（受講者同士・ファシリテーターから）を複数設ける
- 研修冒頭で「なぜこの研修をするのか」を必ず共有する

## 6. タイムテーブル設計

### 半日研修モデル（3〜4時間）

| 時間 | 内容 | 手法 |
|------|------|------|
| 0:00〜0:20 | 開会・研修の目的共有・アイスブレイク | 講義 + グループワーク |
| 0:20〜1:00 | コアインプット①（概念・フレームワーク） | 講義 |
| 1:00〜1:40 | 演習①（個人ワーク → グループ共有） | セルフワーク + グループワーク |
| 1:40〜2:20 | コアインプット②（深掘り・応用） | 講義 + ディスカッション |
| 2:20〜3:00 | 演習②（グループワーク → 発表） | グループワーク + プレゼン |
| 3:00〜3:30 | 振り返り・アクションプラン作成・共有 | セルフワーク + 全体共有 |

### 1日研修モデル（7〜8時間）

| 時間 | 内容 | 手法 |
|------|------|------|
| 9:00〜9:30 | 開会・目的共有・アイスブレイク | 講義 + グループワーク |
| 9:30〜11:00 | セッション1（基礎概念・現状認識） | 講義 + 演習 |
| 11:00〜12:00 | セッション2（応用・深掘り） | グループワーク |
| 12:00〜13:00 | 昼食休憩 | — |
| 13:00〜14:30 | セッション3（実践演習・フィードバック） | ロールプレイ or ケーススタディ |
| 14:30〜16:00 | セッション4（まとめ演習・発表） | グループワーク + プレゼン |
| 16:00〜17:00 | 振り返り・アクションプラン・クロージング | セルフワーク + 全体共有 |

### 複数日・プロジェクト型研修モデル（数ヶ月）

アクションラーニング形式。研修セッションとフィールドワーク期間を交互に組み合わせる。

```
[Session 1] 目的共有・テーマ設定・インプット（終日）
      ↓ フィールドワーク期間（2〜4週間）
[Session 2] フィールドワーク報告・深掘り・中間整理（終日）
      ↓ フィールドワーク期間（2〜4週間）
[Session 3] 中間プレゼン・アドバイス（終日）
      ↓ フィールドワーク期間（2〜4週間）
[Session 4] 事業案・提案の磨き合い（終日）
      ↓ 最終準備期間（1〜2週間）
[Session 5] 最終プレゼン・経営者へ提案・表彰（終日）
```

## 7. Off-JTとOJTのバランス設計

### 70:20:10の法則

人の成長の割合:
- **70%**: 直接経験（業務での成功・失敗）← OJT
- **20%**: フィードバック（上司・同僚からのアドバイス）← OJT
- **10%**: 研修・座学（Off-JT）

> **重要**: 研修（Off-JT）は成長の10%にすぎない。研修「だけ」では人材は育たない。OJT（実践・内省）との連動設計が必須。

### 経験学習サイクル（コルブのモデル）

```
a. 具体的な経験（新しい取り組み）
       ↓
b. 内省的な観察（様々な視点で振り返る）
       ↓
c. 抽象的な概念化（法則・教訓を見出す）
       ↓
d. 能動的な実験（次の経験に適用）
       ↓（繰り返し）
```

研修（Off-JT）は「c. 抽象的概念化」のサポート。フィードバックは「b. 内省的観察」のサポート。

### 研修と実務の連動設計

```
研修（Off-JT）:     概念・フレームワーク習得
       ↓
現場での実践（OJT）: 研修で学んだことを実務に適用
       ↓
フィードバック（OJT）: 上司・同僚からのアドバイス・内省
       ↓
次の研修（Off-JT）:  新たな段階の概念習得
```

## 8. 研修タイプ別の判断テーブル

### どの研修形式を選ぶか

```
【判断質問1】参加者は同じ場所に集まれるか？
  → Yes: 対面集合を検討
  → No:  オンライン形式を検討

【判断質問2】目的は「知識習得」だけか、「行動変容」まで求めるか？
  → 知識習得のみ: eラーニング・単発研修
  → 行動変容:    対面集合 or アクションラーニング

【判断質問3】研修に使える期間はどのくらいか？
  → 半日〜2日:    単発研修（座学＋演習）
  → 1〜3ヶ月:    ブレンディッドラーニング
  → 3〜6ヶ月以上: アクションラーニング型

【判断質問4】研修の成果として「実際の事業提案・成果物」を求めるか？
  → Yes: アクションラーニング型・プロジェクト型
  → No:  通常の研修形式でよい
```

## 9. プログラム設計チェックリスト

### 設計前

- [ ] 研修の最終目的（行動変容後の組織・事業への貢献）が明確か
- [ ] 「求める人材像」との整合性が確認されているか
- [ ] 対象者（階層・職種・選抜方式）が決定されているか
- [ ] 研修形式（対面・オンライン・ブレンディッド等）が選定されているか
- [ ] 研修期間・日数・時間数が確定しているか
- [ ] 経営者や現場とのすり合わせが完了しているか

### 設計中

- [ ] 目標が「行動変容型」で記述されているか（〇〇できるようになる）
- [ ] 導入→展開→まとめの3フェーズが設計されているか
- [ ] 経験学習サイクル（経験→内省→概念化→実験）が組み込まれているか
- [ ] グループワーク・個人ワーク・全体共有がバランスよく配置されているか
- [ ] アクションプラン作成の時間が確保されているか
- [ ] OJTとの連動（研修後の現場実践・フィードバック計画）が設計されているか

### 設計後・実施前

- [ ] ファシリテーターへのブリーフィングが完了しているか
- [ ] 受講者への事前案内（目的・期待すること）が送付されているか
- [ ] 会場・ツール・資料の準備が完了しているか

## 10. AskUserQuestion の使用タイミング

以下の場面では、ユーザーに確認してから設計を進める:

**研修期間の確認**
```
「研修に割ける期間はどれくらいですか？
 1. 半日〜1日（単発研修）
 2. 数日〜1週間（集中研修）
 3. 1〜3ヶ月（ブレンディッドラーニング）
 4. 3〜6ヶ月以上（プロジェクト型・アクションラーニング）」
```

**優先する効果の確認**
```
「この研修で最も重視する効果はどれですか？
 1. 知識・概念の習得
 2. 特定スキルの習得
 3. 行動・意識の変容
 4. 実際の事業成果・イノベーション創出」
```

**対象者レベルの確認**
```
「受講対象者の主な層はどれですか？
 1. 新入社員・若手（〜4年目）
 2. 中堅社員（5〜10年目）
 3. 管理職・マネジャー層
 4. 部長・経営幹部層
 5. 特定の職種・部門（例：営業、研究開発）」
```

## KSAフレームワーク

### KSAとは

教える内容を **Knowledge（知識）/ Skills（スキル）/ Attitudes（態度・姿勢）** の3つに分類するフレームワーク。「何を教えるか」を明確化し、適切な教授法を選択するための基盤となる。

| 要素 | 内容 | 教え方 |
|-----|------|-------|
| **K（Knowledge）** | 概念・情報・理論の理解 | 講義・テキスト・説明 |
| **S（Skills）** | 実践的なスキル・技術 | 演習・ロールプレイ・実習 |
| **A（Attitudes）** | 価値観・姿勢・マインドセット | 体験・内省・ストーリー・ケース |

### A（態度・姿勢）を明文化することの重要性

多くの研修設計者がKとSには注力するが、**Aを明文化しないまま設計に入る**という落とし穴がある。しかし、行動変容には態度の変化が不可欠である。

**Aを明文化する具体的な手順**:
1. 「この研修終了後、参加者にどんな気持ちで業務に向かってほしいか？」を言語化する
2. 「研修前と後で、何に対する見方が変わっていてほしいか？」を問う
3. 態度目標を行動言語で記述する（例：「〜することに前向きな姿勢を持つ」）

### KSAのメリット

- 教える内容の抜け漏れを防ぐ
- 研修の目的・ゴールを関係者と共有しやすくなる
- 評価方法（何をもって成功とするか）が明確になる
- 適切な研修形式・手法の選択が容易になる

### KSAに基づく目標記述の例

```
テーマ: 「質問力向上研修」

K（知識）目標:
→ オープンクエスチョン・クローズドクエスチョンの違いと使い分けを説明できる

S（スキル）目標:
→ ロールプレイで適切な質問を5回以上実践できる

A（態度）目標:
→ 相手の話を引き出すことの価値を認識し、質問することに積極的な姿勢を持つ
```

## 4領域の目的設定

研修の目的は1つの領域に限定されない。インストラクショナルデザインでは**4つの領域**から目的を捉え、バランスよく設定することが重要である。

### 4つの領域

| 領域 | 内容 | 目標の例 |
|-----|------|---------|
| **①認知領域**（知識・理解） | 知識・概念の習得・理解 | 「〜を説明できる」「〜の違いを理解する」 |
| **②感情領域**（態度・価値観） | 感情・姿勢・価値観の変容 | 「〜することに意欲を持つ」「〜の重要性を認識する」 |
| **③行動領域**（スキル・実践） | 具体的な行動・スキルの習得 | 「〜を実践できる」「〜を使いこなす」 |
| **④対人関係領域**（コミュニケーション） | 他者との関係・協働の改善 | 「〜の場面で適切にフィードバックできる」「チームで〜を実践する」 |

### 「成果」から逆にデザインする

目標設定は**研修修了後の理想の状態（成果）から逆算**して設計する。

```
Step 1: 成果のイメージ
「この研修の3ヶ月後、受講者はどんな行動をとっているか？」

Step 2: 4領域への分解
「そのために、何を知り（認知）、どう感じ（感情）、
 何ができ（行動）、誰とどう関われる（対人）必要があるか？」

Step 3: 優先順位の決定
「4つの中で、この研修で最も重視する領域はどれか？」

Step 4: 測定可能な目標記述
「研修終了時点で何を確認できれば成功か？」
```

### 4領域のバランスチェック

```
□ 知識だけ教えて終わっていないか？（認知領域偏重）
□ 参加者の感情・態度変容を意識した設計になっているか？（感情領域）
□ 実際に「やってみる」時間が確保されているか？（行動領域）
□ 他者との関わり・フィードバックが組み込まれているか？（対人関係領域）
```

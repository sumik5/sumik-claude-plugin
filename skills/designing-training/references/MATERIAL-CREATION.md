# 研修資料作成ガイド

研修で使用する各種資料（スライド・ワークシート・配付資料・ファシリテーターガイド）の
設計・作成方法と、オンライン研修における注意点を提供するリファレンス。

---

## 1. 研修資料の種類と役割

研修資料は「誰が・いつ・何のために使うか」によって明確に役割が異なる。

| 資料の種類 | 使用者 | タイミング | 目的 |
|-----------|-------|----------|------|
| **スライド（講義用）** | 講師（参加者も閲覧） | 研修中 | コンテンツの視覚的提示 |
| **ワークシート（演習用）** | 参加者 | 研修中のアクティビティ | 考える・書く・実践する |
| **配付資料（参照用）** | 参加者 | 研修後 | 研修内容の復習・参照 |
| **ファシリテーターガイド（講師用）** | 講師 | 研修中 | 進行・タイミング・話し方の指示 |

> **AskUserQuestion（実装時の判断）：** どの資料が必要かは研修の形式・目的によって異なる。以下の判断テーブルを参照。

### 資料の必要性 判断テーブル

| 研修の特徴 | 推奨資料 |
|-----------|---------|
| 知識インプット中心 | スライド + 配付資料 |
| スキル演習・ワーク中心 | スライド + ワークシート |
| 対話・ディスカッション中心 | スライド（最小限）+ ワークシート |
| オンライン研修 | デジタルワークシート（印刷不要） |
| 複数回にわたる研修 | ワークブック（全回分を1冊に統合） |
| 講師が複数いる研修 | ファシリテーターガイド（必須） |

## 2. スライド（講義用）の作成ガイド

### 2.1 スライドの役割を正しく理解する

スライドは講師の「台本」ではなく、参加者の理解を助けるための「視覚補助ツール」。

**スライドに書くべきこと：**
- キーワード・短い文章（箇条書き）
- 図・グラフ・表（視覚的に説明できるもの）
- ポイントを強調する言葉
- アクティビティの指示文

**スライドに書かないこと：**
- 講師が話す内容をすべて文章で書く（読み上げる必要がない）
- 長い文章（参加者が読み切れない）
- 1枚に多くの情報を詰め込む

### 2.2 情報量の適正化

**1スライドの情報量の目安：**

| 要素 | 推奨 |
|------|------|
| 1スライドのメッセージ | 1つに絞る |
| 箇条書きの数 | 3〜5点まで |
| 文字量 | 参加者が10秒以内に読める量 |
| フォントサイズ | 最小24pt以上（会場後方・小画面から見えること） |

**スライドの情報量が多すぎるサイン：**
- 参加者がスライドを読んでいて講師の話を聞いていない
- スライドの切り替えを急いでしまう
- 「ここに書いてある通りです」と言わなければならない

### 2.3 ビジュアルの活用

**効果的なビジュアル活用：**

| ビジュアル | 使用場面 |
|-----------|---------|
| 図・フロー図 | 手順・プロセスを示す |
| 表・マトリクス | 比較・分類を示す |
| グラフ | 数値の変化・比較を示す |
| 写真・イラスト | 感情・雰囲気・具体的なイメージを伝える |
| アイコン | 箇条書きをわかりやすくする |

**ビジュアルを使う際の注意点：**
- 飾りのためのビジュアルは使わない（目的のないビジュアルは逆効果）
- カラーは3〜4色以内に絞る（メインカラー・アクセントカラー・背景）
- 白黒印刷でも読めるコントラストを確保する

### 2.4 参加者主体に合わせたスライド構成

参加者主体の研修では、スライドを「講師が説明するための資料」から「参加者が考えるきっかけとなるもの」にシフトする。

**講義型スライドの例：**
```
❌ 「コミュニケーションの3つの要素：
    1. 言語（7%）
    2. 声のトーン（38%）
    3. 表情・ジェスチャー（55%）」
```

**参加者主体型スライドの例：**
```
✅ 「相手に与える印象の7%・38%・55%は何だと思いますか？」
   → 参加者にチャットや投票で予測させてから答えを出す
```

### 2.5 EAT構成とスライドの対応

| EATの段階 | スライドの内容 |
|---------|--------------|
| E（経験を引き出す） | 「〇〇の経験を思い出してください」「A・B・Cのどれだと思いますか？」 |
| A（気づきを深める） | グループワークの指示、問いかけの文章 |
| T（理論を提示する） | フレームワーク・概念図・まとめ |

## 3. ワークシート（演習用）の作成ガイド

### 3.1 ワークシートの目的

- 参加者が「考える・書く・整理する」ための記録媒体
- 研修後の参照・実践の記録としても使える
- 講師が一人ひとりと対話できないオンライン研修では特に重要

### 3.2 個人ワーク用ワークシートの設計

**設計のポイント：**
- 何を記入するかを明確にする（曖昧な問いは参加者が困惑する）
- 記入スペースは十分に確保する
- 時間内で書ける量に収める

**記入スペースの目安：**

| アクティビティの時間 | 書き込む量の目安 |
|------------------|---------------|
| 1分 | 1〜2文（30〜50字） |
| 3分 | 3〜5点の箇条書き |
| 5分 | 段落形式で1〜2段落 |

**効果的なワークシートの問いかけ例：**

```
個人振り返りシートの例：
─────────────────────────────
1. 今日の研修で最も印象に残ったことは何ですか？
   （1〜2つ、具体的に）
   →

2. 職場に戻ってすぐに実践したいことは何ですか？
   （具体的な行動として）
   →

3. 実践する際に予想される障壁は？ どう乗り越えますか？
   →
─────────────────────────────
```

### 3.3 グループワーク用ワークシートの設計

**グループワークシートに必須の要素：**
- グループ名・参加者名を記入する欄
- 課題・問いかけの文章（インストラクション）
- 各自が記入するスペース（個人→全体の流れが多い）
- グループとしてのアウトプットをまとめるスペース
- 発表準備のための記入欄（発表する場合）

**グループワークシートの例：**
```
グループ：___________
メンバー：___________

【課題】 このケースにおいて、あなたならどう対応しますか？

個人で考える（2分）→ メンバーと共有（3分）→ グループの意見をまとめる（2分）

■ 個人の意見（各自記入）
→

■ グループとしての意見（まとめる）
→

■ 発表用ポイント（3点以内）
1.
2.
3.
```

### 3.4 振り返りシートの設計

研修終了時・研修後のフォローアップ用の振り返りシート。

**振り返りシートに含める要素：**

| 要素 | 内容例 |
|------|-------|
| 学んだこと | 「今日の研修で新しく学んだこと・再確認したことは？」 |
| 気づき | 「これまでの自分の行動を振り返って気づいたことは？」 |
| アクションプラン | 「研修後、具体的に行動することは？（期限を含めて）」 |
| 実践の宣言 | 「誰に・何を・いつまでに実践するか」 |
| 振り返り確認 | 「〇日後に実践状況を確認する」 |

## 4. 配付資料（参照用）の作成ガイド

### 4.1 配付資料の役割

研修で学んだことを、後から参照・活用するための資料。スライドとは別に設計することが多い。

**配付資料の種類：**

| 種類 | 内容 | 使用タイミング |
|------|------|--------------|
| 要約資料 | 研修のキーポイントをコンパクトにまとめたもの | 研修後の復習 |
| チェックリスト | 行動変容のための確認リスト | 職場での実践時 |
| テンプレート | 習得したスキルを使うための型 | 職場での実践時 |
| 参考資料リスト | さらに深く学ぶための書籍・URL等 | 自主学習時 |

### 4.2 配付資料の設計原則

- **スライドをそのまま配付しない：** スライドは視覚補助ツールなので、文字が少なく、配付資料として使えない場合が多い
- **自立した資料にする：** スライドを見なくても内容が理解できる資料にする
- **実践に使える形式にする：** チェックリスト・テンプレートなど、職場で使える形式にする

### 4.3 オンライン研修での配付方法

| 方法 | 特徴 |
|------|------|
| 事前メールで送付 | 研修当日に手元にある。印刷して参加してもらえる |
| チャットでリンク共有 | 研修中に配布。URLを共有して即時アクセス |
| クラウドストレージ共有 | Googleドライブ等で共有。ファイル管理が容易 |
| デジタルワークシート | PDF/スプレッドシートなど。デジタルで記入可能 |

## 5. ファシリテーターガイドの作成方法

### 5.1 ファシリテーターガイドとは

講師（ファシリテーター）が研修を進行するための詳細な手引き書。複数の講師が同じ研修を行う場合や、講師が自分以外の人が作成した研修を担当する場合に特に重要。

### 5.2 ファシリテーターガイドに含める要素

**必須セクション：**

```
1. 研修の概要
   - 研修目的・目標
   - 対象者・前提条件
   - 所要時間
   - 必要な準備物

2. タイムテーブル
   - 時刻（開始からの経過時間）
   - 各セクションの内容
   - 使用するスライド番号
   - ツール・機能

3. 各セクションの詳細指示
   - 話し方メモ（キーワード・言い回し）
   - アクティビティの指示文（完全版）
   - 参加者の反応例と対応方法
   - 注意事項・トラブルシューティング

4. アクティビティのインストラクション
   - 8ステップの手順
   - グループ分けの方法
   - 時間設定

5. よくある質問と回答例
```

### 5.3 タイムテーブルの書き方

| 時間（経過） | 時刻 | セクション | 内容 | スライド | ツール | 備考 |
|------------|------|----------|------|---------|-------|------|
| 0:00 | 13:00 | ソフトオープニング | クイズをスライドに表示 | #1 | チャット | 3分前から開始 |
| 0:03 | 13:03 | オープニング | クイズの答えを発表 | #2-3 | 投票 | 全員が参加できるよう確認 |
| 0:08 | 13:08 | セクション1（C） | 〇〇の概念説明 | #4-7 | - | 4分で説明 |
| 0:12 | 13:12 | セクション1（S） | グループディスカッション | #8 | ブレイクアウト | 3-4人 / 3分 |
| 0:15 | 13:15 | セクション1（R） | リビジット | #9 | チャット | 1分 |

### 5.4 話し方メモの書き方

研修で話す内容を完全な台本として書くのではなく、キーワードと流れを書き留める。

**話し方メモの例：**
```
【スライド #4：〇〇の3つのポイント】
・「では、〇〇について3つのポイントをご紹介します」
・ポイント1を示す → 「まず1つ目は〇〇です。これは...」
・事例を1つ紹介（△△の事例）
・参加者への問いかけ：「みなさんの職場ではいかがでしょうか？チャットに入力してください」
・回答を2〜3つ紹介してから次へ
```

## 7. オンライン研修資料の特有の注意点

### 7.1 スライドのオンライン最適化

オンライン研修では「画面共有」でスライドが表示される。以下の点に注意。

| 課題 | 対策 |
|------|------|
| 文字が小さくて見えない | フォントサイズを最小28pt以上にする |
| 画面の一部しか見えない | 重要な要素を中央寄りに配置する |
| コントラストが低い | 背景と文字のコントラストを高める（白背景に黒文字が最も安全） |
| 複数の画面（スライド + カメラ映像）が混在 | 情報を整理し、シンプルにする |

### 7.2 デジタルワークシートの設計

オンライン研修では、印刷したワークシートを配布できない。以下の代替手段を設計する。

| 手段 | 特徴 | 使用場面 |
|------|------|---------|
| チャットへの記入 | 手軽。全員が同時に入力できる | 短い回答・1〜2文 |
| オンラインホワイトボード（Miro等） | 付箋・図を使った自由な表現 | ブレインストーミング・整理 |
| スプレッドシート（Google Sheets等） | 共同編集。構造化された記入が可能 | グループワーク・まとめ |
| PDFワークシート | 事前送付して記入してもらう | 個人ワーク・振り返り |
| 投票・アンケート機能 | 選択形式。集計が自動 | 理解度確認・意見収集 |

### 7.3 チャットを活用した「書く」活動

チャットは最もシンプルで参加ハードルが低いツール。効果的に活用する方法：

```
● アクティビティ中は「送信ボタンを押すのを待つ」指示を出す
  → 全員が同時にチャットに入力して一斉送信することで、一人の回答に引きずられない

● チャットに書いた回答を画面に映しながら紹介する
  → 多くの回答を可視化して「自分の意見が伝わった」実感を与える

● 「感想はチャットに、質問はチャットの頭に【質問】とつけて」などのルールを決める
  → 内容の分類をしやすくする
```

## 8. 資料作成の品質チェックリスト

### スライドチェックリスト
```
□ 1スライド1メッセージになっているか
□ フォントサイズは28pt以上か（オンライン・見やすさ）
□ 1スライドの情報量が多すぎないか（5点以内の箇条書き等）
□ 不要なビジュアル・装飾がないか
□ 参加者が「考える・反応する」きっかけになるスライドが含まれているか
□ スライドの番号が振られているか（「スライド5を見てください」と言えるか）
```

### ワークシートチェックリスト
```
□ 何を記入するかが明確か
□ 記入スペースは十分か
□ 時間内に記入できる量か
□ 研修後に「見返したくなる」内容か
□ アクションプランに結びつく問いかけがあるか
□ デジタル対応（オンライン研修の場合）ができているか
```

### ファシリテーターガイドチェックリスト
```
□ 研修の全体の流れが一目でわかるか
□ 各アクティビティのインストラクションが完全に記載されているか
□ タイムテーブルが詳細に記載されているか
□ よくある質問と回答が含まれているか
□ オンライン研修の場合、使用ツール・操作手順が記載されているか
```

## 研修資料作成の実践ワークフロー

### なぜワークフローが重要か

多くの研修担当者が「スライドを作ることから始める」罠に陥る。正しいワークフローはコンテンツ設計が確定してから資料作成に入ることで、手戻りを防ぎ、質の高い資料を効率的に作成できる。

### 資料作成の推奨フロー

```
Step 1: 研修設計の確定（資料作成前に完了させる）
   ├─ 研修目的・ゴール（KSA目標）の明確化
   ├─ 対象参加者・人数・レベルの把握
   ├─ カリキュラム・タイムテーブルの確定
   └─ 各セッションのEATフロー設計

Step 2: コンテンツの整理
   ├─ 各セッションで「何を教えるか」の洗い出し（KSAで整理）
   ├─ 情報の優先度付け（Must → Should → Could）
   └─ 7±2の法則に基づく情報量のチェック

Step 3: ラフスケッチ（スライドを開く前に）
   ├─ 紙またはホワイトボードでスライドの構成を手書きスケッチ
   ├─ 各スライドに「このスライドで伝えたいこと1つ」を明記
   └─ アクティビティ・演習スライドの位置を確認

Step 4: スライド作成
   ├─ テンプレートに基づいてスライドを作成
   ├─ 1スライド1メッセージの原則を守る
   └─ テキスト量は最小限（読むより話す）

Step 5: ワークシート・配付資料の作成
   ├─ スライドと連動した記入欄・演習課題の設計
   └─ ファシリテーターガイドへの転記・整合性確認

Step 6: 全体レビュー
   ├─ タイムテーブルとの照合（時間に収まるか）
   ├─ 参加者目線でのチェック（わかりやすいか・使いやすいか）
   └─ 試作ファシリテーションでの修正
```

### 資料作成の時間管理

| 工程 | 目安 |
|-----|------|
| コンテンツ整理・ラフスケッチ | 全体の30% |
| スライド作成 | 全体の30% |
| ワークシート・ガイド作成 | 全体の25% |
| レビュー・修正 | 全体の15% |

「スライドを綺麗にすること」に時間をかけすぎず、**コンテンツ設計とアクティビティ設計に最も時間を投資する**。

## AV機器・会場設営ガイド

### 研修前の機器確認チェック

**プロジェクター・スクリーン**
```
□ プロジェクターとPCの接続確認（HDMI/VGA/無線）
□ スクリーンの位置と参加者の視認性確認
□ 解像度・明るさの調整
□ スライドショーモードでの表示確認
```

**音響設備**
```
□ マイクの種類確認（ハンドマイク/ラベリアマイク/ヘッドセット）
□ スピーカーとの音量バランス確認
□ 動画・音声を使う場合のPCスピーカーとの切り替え確認
□ ハウリング防止の確認（マイクとスピーカーの距離）
```

**PC・備品**
```
□ 電源コード・延長コードの確認
□ スライドリモコン（レーザーポインター付）の準備
□ タイマー（スクリーン表示用）の準備
□ ホワイトボード・フリップチャートとマーカーの確認
□ 付箋・ペンなどワーク用備品の準備
```

**会場セッティング**
```
□ 参加人数に対した椅子・机の配置（島型/教室型/シアター型）
□ 空調・換気の調整（暑すぎ・寒すぎは集中力に影響）
□ 休憩スペース・荷物置き場の確認
□ 緊急時の避難経路確認
```

### セッティングパターン別のメリット・デメリット

| セッティング | メリット | デメリット | 適用場面 |
|-----------|---------|----------|---------|
| **島型（グループ配置）** | グループ対話・協働に最適 | 前方への注意が散漫になりやすい | グループワーク中心の研修 |
| **教室型（前向き縦列）** | 全員が前を向き、集中しやすい | 参加者同士の対話がしにくい | 知識習得中心の研修 |
| **U字型** | 全員が顔を見合わせられる | 参加者が多いと成立しない | 少人数（20名以下）の対話型研修 |
| **シアター型（椅子のみ）** | 多人数収容可能 | 書く・動くアクティビティが困難 | 大人数講演型・キックオフ |

### トラブル時の対応

```
プロジェクターが映らない場合:
1. PCの外部ディスプレイ出力設定を確認（Windows: Win+P、Mac: ⌘+F1）
2. HDMIケーブルの抜き差し
3. プロジェクターの入力切替ボタンを確認
4. 予備のケーブル・アダプターを常備する

音が出ない場合:
1. PCの音量設定確認（ミュートになっていないか）
2. スピーカーの電源確認
3. PCとスピーカーの接続確認
4. 動画の場合はプレーヤーの音量設定も確認

Wi-Fi・ネット接続トラブル:
1. オフライン版を必ず用意しておく（スライド・動画のローカル保存）
2. スマートフォンのテザリングを準備
```

## 視覚的教材設計原則

### スライドデザインの基本4原則（CARP）

| 原則 | 英語 | 内容 |
|-----|------|------|
| **近接** | Contrast | 関連する情報を視覚的にグループ化する |
| **整列** | Alignment | すべての要素を意図的に整列させる（見えないラインを意識する） |
| **繰り返し** | Repetition | デザイン要素を繰り返してまとまりと一貫性を作る |
| **コントラスト** | Proximity | 重要な要素を強調するためにコントラストを作る |

### 視覚的ヒエラルキーの設計

情報の重要度を視覚的に表現することで、参加者が一瞬で「何が重要か」を把握できる。

```
最重要（見出し・キーメッセージ）:
→ 大きいフォント（28〜36pt）、太字、色強調

重要（本文・説明）:
→ 標準フォント（20〜24pt）、通常または太字

補足（注記・出典）:
→ 小さいフォント（14〜16pt）、グレー系
```

### カラーデザインの原則

```
色数: 基本3色まで（多色使いは混乱を招く）
→ メインカラー: 1色（ブランドカラー or テーマカラー）
→ アクセントカラー: 1色（重要箇所の強調）
→ テキストカラー: 黒またはダークグレー

背景色と文字色のコントラスト:
→ 白背景 + 黒文字: 最も読みやすく疲れにくい
→ 暗い背景 + 白文字: 映像・プレゼン向き（プリント不向き）

色のユニバーサルデザイン:
→ 赤・緑の組み合わせは色覚多様性の参加者に配慮が必要
→ 色だけでなく形・記号・位置でも情報を伝える
```

### 図解・ビジュアルの活用

| ビジュアルタイプ | 適した内容 |
|----------------|----------|
| **フローチャート** | プロセス・手順・意思決定の流れ |
| **マトリクス（2×2表）** | 2つの軸での分類・比較 |
| **ピラミッド図** | 階層・優先度・量的関係 |
| **ベン図** | 概念の重なり・包含関係 |
| **タイムライン** | 時系列の変化・歴史・プロジェクト計画 |
| **グラフ（棒・折れ線・円）** | 数値データの比較・推移 |

**図解化のルール**:
```
□ 文字だけで説明できることでも、図解できないか検討する
□ 図解は「一目で意味がわかる」シンプルさを優先する
□ 図解後に必ずテキストで補足説明を入れる
□ 複雑な図よりシンプルな図 + 口頭説明の組み合わせを選ぶ
```

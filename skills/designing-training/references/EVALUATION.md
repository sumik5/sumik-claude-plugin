# 研修効果測定・評価ガイド（カークパトリック4段階評価）

研修効果の測定・評価手法と、研修体系の継続的改善のためのPDCAサイクルを提供するリファレンス。
カークパトリック4段階評価モデルを中心に、実践的な評価設計・実施・フィードバック方法をカバーする。

---

## 1. なぜ研修効果を測定するのか

### 研修の本来の目的

研修は「学びの場を提供すること」が目的ではなく、「ビジネスにプラスの影響をもたらすこと」が目的。

- **誤った考え方：** 研修を実施したこと自体が目的
- **正しい考え方：** 研修による参加者の行動変容 → ビジネス成果につながること

### 評価が必要な理由

| 理由 | 内容 |
|------|------|
| 研修の効果を確認する | 「やりっぱなし」を防ぎ、投資対効果を測定する |
| 改善点を特定する | 何がうまくいっていないかを把握して改善につなげる |
| 次回の設計に活かす | データに基づいたインストラクショナルデザインの改善 |
| 利害関係者への説明 | 研修の価値を経営陣・管理職に示す |

### Level 4 からさかのぼって設計する原則

カークパトリック評価では、**設計時は逆順（Level 4 → 1）で考え、評価時は順番（Level 1 → 4）で行う**。

```
【設計フロー（逆順）】
Level 4（結果）: どのビジネス指標を改善したいか？
    ↓
Level 3（行動）: そのために、参加者にどう行動を変えてほしいか？
    ↓
Level 2（学習）: その行動を取るために、どんな知識・スキルが必要か？
    ↓
Level 1（反応）: 参加者が研修に前向きに取り組める環境を設計する
```

## 2. カークパトリック4段階評価モデル

### 4段階の概要

| レベル | 名称 | 評価内容 | 難易度 |
|-------|------|---------|-------|
| **Level 1** | 反応 | 研修への満足度・印象 | 低（容易） |
| **Level 2** | 学習 | 知識・スキルの習得度 | 中 |
| **Level 3** | 行動 | 職場での行動変容 | 高 |
| **Level 4** | 結果 | ビジネス指標への影響 | 最高（困難） |

---

## 3. Level 1（反応）：満足度評価

### 何を評価するか

研修に対する参加者の反応・満足度を測定する。「研修を受けてどう感じたか」を把握する。

**評価の観点：**
- 研修内容の有益性・関連性（「職場で役立てられると思うか」）
- 研修の進め方・ファシリテーションの満足度
- 研修環境（会場・オンライン環境）の快適さ
- 総合満足度

### アンケート設計のポイント

**Level 1 アンケート項目例：**

```
■ 研修内容について
Q1. 今日学んだことは、業務に役立てられると思いますか？
   ①強くそう思う ②そう思う ③どちらでもない ④そう思わない ⑤全くそう思わない

Q2. 研修の内容は、あなたのニーズに合っていましたか？
   ①強くそう思う ②そう思う ③どちらでもない ④そう思わない ⑤全くそう思わない

■ 研修の進め方について
Q3. 研修の進め方（ファシリテーション）は効果的でしたか？
   ①強くそう思う ②そう思う ③どちらでもない ④そう思わない ⑤全くそう思わない

■ 自由記述
Q4. 研修で特に印象に残ったことは何ですか？
   （自由記述）

Q5. 研修への要望・改善点があれば教えてください。
   （自由記述）
```

**アンケート設計の注意点：**
- 研修終了直後に実施する（記憶が鮮明なうちに）
- 選択肢は4〜5段階評価が一般的
- 自由記述欄を設ける（定量だけでは改善点がわかりにくい）
- 質問数は10問以内に絞る（多すぎると回答の質が下がる）

### Level 1 評価の限界

Level 1（満足度）が高いからといって、学習・行動変容・ビジネス成果が出るとは限らない。

> **重要：** 「楽しかった研修」と「成果が出る研修」は必ずしも同じではない。Level 1 だけで評価を終えないこと。

## 4. Level 2（学習）：習得度評価

### 何を評価するか

研修を通じて参加者が知識・スキルを実際に習得したかを測定する。

**評価の観点：**
- 知識習得（知っているか）
- 理解度（正しく理解しているか）
- スキル習得（できるか）
- 態度変容（やろうとする意欲があるか）

### テスト設計の方法

**テスト形式の選択：**

| 形式 | 特徴 | 適した場面 |
|------|------|-----------|
| 選択問題（4択等） | 採点が容易。知識確認に適する | 知識習得の確認 |
| 記述・論述 | 理解の深さを確認できる | 概念理解の確認 |
| ロールプレイ・実技 | スキルを実際に評価できる | スキル習得の確認 |
| シミュレーション | 実際の業務に近い状況で評価 | 応用力の確認 |

**事前テスト（Pre-test）と事後テスト（Post-test）の活用：**

```
研修前にPre-testを実施
    ↓
研修実施
    ↓
研修後にPost-testを実施
    ↓
差分（向上度）を測定 → 研修による学習効果を測定
```

**利点：** 研修前後の差分で「研修の貢献」を測定できる。事前知識のばらつきを考慮した評価が可能。

### Level 2 評価の実施タイミング

| タイミング | 目的 |
|----------|------|
| 研修直後 | 習得度の確認（短期記憶） |
| 研修後1〜2週間 | 定着度の確認（長期記憶に移行したか） |
| 研修後1ヶ月 | 実践を経た後の理解深化の確認 |

## 5. Level 3（行動）：行動変容評価

### 何を評価するか

研修での学びを参加者が実際に職場で活用し、行動が変容したかを測定する。

**評価の観点：**
- 研修で学んだことを職場で実践しているか
- アクションプランが実行されているか
- 研修前後で行動パターンが変化しているか

### フォローアップ手法

#### 方法1：アクションプランのトラッキング

研修終了時に参加者が立てたアクションプランを一定期間後に確認する。

**実施フロー：**
```
研修終了時: アクションプラン作成（「いつまでに・何を・どうやって」）
    ↓
1〜2週間後: 進捗確認メール・オンラインアンケート
    ↓
1ヶ月後: フォローアップセッション or 上長確認
    ↓
3ヶ月後: 最終確認・成果報告
```

#### 方法2：360度フィードバック

上司・同僚・部下など複数の視点から行動変容を評価する。

| 評価者 | 評価内容 |
|-------|---------|
| 本人 | 自己評価（研修前後の変化） |
| 上長 | 職場での行動変化を観察・評価 |
| 同僚 | 日常の働き方の変化を評価 |
| 部下 | マネジメント変化を評価（管理職研修の場合） |

#### 方法3：観察・面談

| 手法 | 内容 |
|------|------|
| 上長観察 | 研修後の行動を上長が定期的に観察・記録 |
| 1on1面談 | 研修の実践状況を上長が定期確認 |
| 現場視察 | 研修担当者が現場の様子を観察（現場がある場合） |

### Level 3 評価の実施タイミング

| タイミング | 目的 |
|----------|------|
| 研修後1〜2週間 | 初期の実践確認（忘れる前に） |
| 研修後1ヶ月 | 定着の確認 |
| 研修後3〜6ヶ月 | 継続実践・習慣化の確認 |

### Level 3 評価を高めるための設計

**行動変容を促す研修設計のポイント：**
- 研修内でアクションプランを作成させる
- 上長を研修に巻き込む（研修前後の面談を設定させる）
- フォローアップセッションを研修計画に組み込む
- 実践報告の場を設ける（コミュニティ・SNS等）

## 6. Level 4（結果）：ビジネス成果評価

### 何を評価するか

研修による行動変容が、最終的にビジネス指標（ROI）にどう影響したかを測定する。

**評価指標の例：**

| カテゴリ | 指標例 |
|---------|-------|
| 売上・業績 | 売上高の変化、顧客満足度スコア |
| 生産性 | 処理件数、エラー率、残業時間 |
| 人材 | 離職率、昇進率、エンゲージメントスコア |
| コスト | コスト削減額、問題発生件数の減少 |

### ROI（投資対効果）の計算

```
ROI（%）= [(研修による利益 - 研修コスト) / 研修コスト] × 100
```

**研修コスト（分母）に含める要素：**
- 研修費用（講師費・会場費・教材費等）
- 参加者の時間コスト（参加者の時給 × 研修時間 × 人数）
- 研修担当者の工数コスト

### Level 4 評価の難しさと対処

Level 4 の評価は、「研修の貢献」を切り出すことが難しい（他の要因も影響するため）。

**現実的な対応：**
- 完全な因果関係の証明は難しいと認識したうえで、できる範囲で測定する
- 「研修グループ」と「非研修グループ」の比較（コントロールグループ設計）
- 研修前後の数値変化を記録し、研修の貢献を推計する
- 定量評価が難しい場合は定性的な事例収集（成功事例・インタビュー）

## 7. 評価レベル選択の判断基準

> **AskUserQuestion（実装時の判断）：** どのレベルまで評価するかは、研修の目的・予算・リソースによって異なる。以下のテーブルを参照。

| 研修の特徴 | 推奨評価レベル |
|-----------|-------------|
| 全研修共通（最低限） | Level 1（満足度アンケート） |
| 知識・資格取得が目的 | Level 1 + Level 2（テスト） |
| 行動変容が目的（管理職研修等） | Level 1 + 2 + Level 3 |
| 重要な経営課題への対応研修 | Level 1 〜 4（全段階） |
| 研修投資の効果を経営に示す必要がある | Level 4（ROI計算） |

## 8. 評価結果のフィードバックと改善サイクル

### PDCAサイクルの実践

```
【Plan】 研修設計
 ・目的・目標の明確化（Level 4からさかのぼって設定）
 ・評価方法の決定
 ・成功基準の設定
    ↓
【Do】 研修実施・評価データ収集
 ・研修実施
 ・Level 1: アンケート実施
 ・Level 2: テスト実施
 ・Level 3: フォローアップ
 ・Level 4: ビジネス指標確認
    ↓
【Check】 評価分析
 ・各レベルのデータを分析
 ・成功基準との比較
 ・課題・改善点の特定
    ↓
【Act】 改善実施
 ・研修設計の改善
 ・ファシリテーションの改善
 ・事前・事後フォローの改善
 ・次回研修への反映
    ↓
（次のPlanへ）
```

### 評価結果のフィードバック先

| フィードバック先 | 内容 | タイミング |
|---------------|------|----------|
| 研修担当者（自己） | 設計・ファシリテーションの改善点 | 研修直後 |
| 講師（外部講師の場合） | 満足度・改善点の共有 | 研修後1週間以内 |
| 参加者の上長 | Level 3 の結果・フォロー依頼 | 研修後1〜2週間 |
| 人事・研修企画部門 | 体系的な改善提案 | 四半期・年次 |
| 経営層 | ROI・ビジネス成果報告 | 半期・年次 |

### 改善サイクルのポイント

- **Level 1 の不満足が多い場合：** 研修の進め方・環境・コンテンツの問題
- **Level 2 の習得率が低い場合：** 学習設計（インストラクショナルデザイン）の問題。難易度・時間配分・教授法を見直す
- **Level 3 の行動変容が起きない場合：** 研修だけの問題ではなく、職場環境・上長のサポートが不足している可能性
- **Level 4 の成果が出ない場合：** そもそも研修で解決できる問題かを再検討する

## 9. 研修体系における評価の位置づけ

### 研修体系の全体設計との連動

個々の研修の評価だけでなく、研修体系全体として評価・改善のサイクルを設計することが重要。

```
求める人材像
    ↓
研修体系設計（必要なスキル・研修一覧）
    ↓
スキルマップによる現状把握
    ↓
各研修の実施・評価（カークパトリック）
    ↓
研修体系の見直し・更新
    ↓（次のサイクルへ）
```

### スキルマップと評価の連動

スキルマップを活用することで、研修効果（Level 3：行動変容）を体系的に測定できる。

**スキルマップを使った評価フロー：**

```
研修前: スキルマップによる現状スキルレベルを評価（自己評価・上長評価）
    ↓
研修実施
    ↓
研修後（3〜6ヶ月後）: 同じスキルマップで再評価
    ↓
スキルレベルの変化を測定 → 研修効果の可視化
```

**スキルマップ活用のポイント：**
- スキルマップの評価基準を明確にする（「レベル1: 知っている」「レベル2: できる」「レベル3: 教えられる」等）
- 自己評価と上長評価の両方を実施し、乖離を確認する
- 評価結果を次の研修ニーズ分析に活かす

### 研修体系の運用ポイント

**研修体系の継続的改善：**

| 観点 | 確認内容 | 実施頻度 |
|------|---------|---------|
| 網羅性 | 求める人材像に必要なスキルが体系に含まれているか | 年次 |
| 適時性 | 環境変化に合わせて内容が更新されているか | 半期・年次 |
| 効果性 | 各研修が実際に効果を上げているか（評価結果） | 各研修後 |
| 連続性 | 研修同士が連携・発展しているか（ステップアップ設計） | 年次 |

## 10. 評価の実践チェックリスト

### 研修設計段階
```
□ Level 4の成功指標（ビジネスKPI）を定義したか
□ Level 3の行動変容目標（「研修後に何をするか」）を設定したか
□ Level 2の学習目標（「何を習得するか」）を設定したか
□ 各レベルの評価方法・タイミングを決定したか
□ 評価データの収集方法・担当を決定したか
□ アクションプランのフォローアップ計画を立てたか
```

### 研修実施後（直後）
```
□ Level 1 アンケートを実施したか
□ Level 2 テスト・確認を実施したか
□ アクションプランを参加者に作成させたか
□ 上長へのフォローアップ依頼を送ったか
```

### フォローアップ段階
```
□ 研修後1〜2週間: Level 3 初期確認を実施したか
□ 研修後1ヶ月: Level 3 定着確認を実施したか
□ 研修後3ヶ月: Level 3 最終確認・Level 4 指標確認を実施したか
□ 評価結果をフィードバック・次回設計に活かしたか
```

# イノベーション研修・プロジェクト型研修ガイド

イノベーション研修の定義・6つの効果・5つの設計手法・プロジェクト型研修の設計・運営・協働型の展開可能性を網羅したリファレンス。

---

## 1. イノベーション研修とは

### 定義と位置づけ

イノベーション研修とは、「**アクションラーニング形式で、イノベーション（経済成果をもたらす革新）の生成プロセスを再現する研修プログラム**」である。

- 研修セッション（数回・終日）とフィールドワーク期間（各数週間）を交互に組み合わせる
- 受講者が「不（不便・不満・不安・不足・不利・不合理・不具合）」を探索・深耕し、事業案を磨き上げる
- 最終的に経営者への提案を行い、資源動員の意思決定を得ることを目指す

### 他の研修との根本的な違い

| 比較項目 | 通常の研修 | イノベーション研修 |
|---------|-----------|-----------------|
| **目標** | 知識・スキル習得 | 実際の事業案創出・経営者への提言 |
| **期間** | 半日〜数日 | 3〜6ヶ月（セッション+フィールドワーク） |
| **手法** | 座学・演習・ケーススタディ | プロジェクト型・フィールドワーク中心 |
| **扱う課題** | 自社の構造改革案（経営者に却下されがち） | イノベーション（経営者が本気で受け止める） |
| **成果物** | レポート・スキル向上 | 事業案・経営者からの承認・資源動員 |

### イノベーション研修が機能する理由

イノベーションの生成プロセス（個人の問題意識 → チームでの深耕 → 組織への提言 → 社会への普及）と、アクションラーニング型研修の構造が本質的に一致している。

```
イノベーション生成プロセス       研修での対応
個人の問題意識 ──────────────→ 「不」の探索（フィールドワーク）
チーム・組織内での議論 ─────────→ セッションでの深掘り・ブラッシュアップ
事業案の形成 ──────────────────→ 事業案フレームの作成
経営者への提案・資源動員 ────────→ 最終プレゼン・意思決定
```

## 2. イノベーション研修の6つの効果

### 効果①：人材開発

**イノベーションリーダーの育成**。管理・維持型ではなく、不確実な状況で自らイノベーションを生み出せる人材を開発する。

研修で養われる能力:
- 変化対応力・課題設定力
- 概念化する力・表現する力・具現化する力
- 諦めずに最後までやり切るスタンス（圧倒的な当事者意識）

> 自立的・自律的に目的を定め、圧倒的な当事者意識で行動し切ることが、イノベーション研修では強く求められる。

行動変容の流れ:
```
受講者が「不」を探索 → 顧客・市場と向き合う → 組織内部にも目を向ける
→ 刺激・内省・対話を繰り返す → 行動が変わる → 研修後の日常業務にも波及
```

### 効果②：人材発掘

**受講者の隠れた才能の可視化**。通常業務では見えにくい資質が研修のプロセスで表面化する。

- 研修プロセスを通じて受講者の行動・コミュニケーション・思考力が顕在化
- 最終プレゼンで「事業案と発表者の本質」が経営者に見えてくる
- 「よそ者・バカ者・若者」が特に活躍する傾向がある

発掘されやすいタイプ:
- 物事を深く考えること・厳しい議論を厭わない人
- 社会課題を解決したいという強い使命感を持つ人
- 傍流・関係会社・若手社員などの「よそ者」的ポジションにいる人

### 効果③：事業開発

**本気の事業開発プラットフォームとしての活用**。

- イノベーション研修を新規事業提案制度にビルトインしている企業もある
- 次のステップに進んだ事業案は：テストマーケティング、プロジェクト発足、新規事業部門への異動、別法人設立、外部投資家への打診など多様
- 事業案フレームは既存事業の計画推進・経営会議での共通フォーマットとしても活用できる

> 全社員が本気でイノベーションを考えることで、創出確率は高まる。イノベーションは「多産多死」が前提。

### 効果④：組織風土改革

**研修を通じたイノベーティブな風土の醸成**。

組織風土改革を実現する4つのレバー:
1. **トップのコミットメント** — 経営者の関与なしにイノベーション文化は生まれない
2. **コミュニケーション** — 研修の場 + 現場でのイノベーションをめぐる対話
3. **伝説・エピソードの継承** — 自社のイノベーション物語を語り継ぐ
4. **仕組み・制度の整備** — 人事評価にイノベーション項目を設ける（例：NVC）

組織風土改革のための実践施策:
- 研修プロセスと結果を社内に逐次共有・広報する
- 優れた事業案を称賛し、シンボリックな人事異動を行う
- イノベーション研修卒業生による社内セミナーを開催する
- 社外のイノベーターを招いて対話の場を設ける

### 効果⑤：危機感の醸成

**フィールドワークを通じて「今そこにある危機」に気づく**。

- 自社に対する危機感: 「新しい事業を起こさないと、うちの会社が衰退する」
- 社会に対する危機感: 社会課題を直視し、解決責任を感じる主体意識の形成

フィールドワークが危機感醸成の鍵。顧客・市場の現実と直接向き合うことで、抽象的な危機意識が「自分ごとの危機感」に変わる。

### 効果⑥：経営者の原点回帰

**「裸の王様」状態の経営者が顧客のリアリティを取り戻す**。

経営者は組織の階層構造とコンプライアンス体制の中で現場から遠ざかりがち。イノベーション研修では受講者が顧客・市場と組織を行き来することで、経営者も自社の原点を再認識できる。

## 3. イノベーション研修の成功要因（5つの設計手法）

### 手法①：「不」の探索・深耕を促し、顧客・市場に入り込む

**「不」とは何か**: 不便・不満・不安・不足・不利・不合理・不具合など、顧客・市場が抱える課題や不快感の総称。

「不」には2つのレベルがある:

| レベル | 特徴 | 問題点 |
|-------|------|--------|
| **マクロの「不」** | 統計・調査から見えるトレンド（少子高齢化等） | 確からしいが「手触り感」がない。事業提案では弱い |
| **ミクロの「不」** | 特定の人・場面の生々しい困りごと | 手触り感があり説得力が高い。フィールドワークで獲得する |

「不」に手触り感を持たせる方法:
- 「不」を持つ人の一人称体験をする（現場に行き、当事者の立場で体験する）
- 「5W1H」で「不」を解剖する（誰が・いつ・どこで・何が・なぜ・どのように不便なのか）
- 表面の「不」の背後にある根本原因まで掘り下げる
- 「不」に共鳴する人の数・頻度・影響の大きさを検証する

**「不」の探索のための問い**:
```
・この「不」を感じている人は他に誰がいるか？
・この「不」の背後にある根本的な原因は何か？
・この「不」は制度・法律・慣習によるものか？
・この「不」を解消しようとしている既存プレイヤーはいるか？
・この「不」のどの部分に自社が最も貢献できるか？
```

### 手法②：組織との「新結合」を狙う

イノベーションの普及には資源動員（ヒト・モノ・カネ・情報・技術・時間）が必要。自社の強みと「不」を結合させる「新結合」の視点が重要。

CFTフレームによる自社資源の把握:
- **C**ustomer（顧客資産）: 顧客基盤・チャネル・信頼関係
- **F**unction（機能・技術）: 特許・技術・専門知識
- **T**rust（信頼・ブランド）: ブランド力・市場での評判

> 社内相談を積極的に行うことで、自社のどの部門が協力できるかを把握できる。

### 手法③：事業案フレームで磨く

経営者を納得させる事業案には「6つの武器（本気度を示す要素）」が必要:

| 武器 | 内容 |
|-----|------|
| **「不」の手触り感** | ミクロの生々しい現場体験に基づく課題設定 |
| **専門家** | 外部専門家の巻き込み・意見収集 |
| **ステークホルダーの把握** | 経営者・関係部門が何を重視しているかの把握 |
| **自社資源の把握** | 使える自社のヒト・モノ・カネ・技術・顧客の整理 |
| **プロトタイピング** | 試作・PoC・ユーザーテストの実施 |
| **ビジョンの表明** | この事業で何を実現したいか（個人の使命感との接続） |

事業案フレームの構成（7要素）:
1. 誰の（ターゲット顧客）
2. どんな「不」を（課題の明確化）
3. どのように解消するか（解決策）
4. なぜ自社がやるか（自社優位性）
5. どのくらいの市場があるか（市場規模）
6. どう収益化するか（ビジネスモデル）
7. 次にどのようなステップを踏むか（アクションプラン）

### 手法④：プロジェクト型研修として実施する

アクションラーニング形式でフィールドワークを中心に設計する。

```
[研修セッション] ← → [フィールドワーク] ← → [研修セッション]
      ↑                      ↑                       ↑
  概念・フレーム           現場での探索             深掘り・提案
  インプット               顧客インタビュー          ブラッシュアップ
  振り返り                 資源調査
```

ケーススタディとの根本的な違い:
- ケーススタディ: 他者の事例を分析・学習する（安全だが当事者意識が生まれにくい）
- プロジェクト型: 自分の事業案を実際に動かし、経営者の承認を取り付ける（リスクあるが圧倒的な行動変容）

ファシリテーターの役割:
- 「慣行の外」にいる人物（その企業の常識に囚われない視点を持てる）
- 経営者の期待領域を理解した上でファシリテートする
- 受講者に無邪気な共感を繰り返し、当事者意識を育てる問いを投げかける

### 手法⑤：経営者のコミットメントを確保する

事前のすり合わせが成功を左右する:

```
事前準備チェックリスト
□ 経営者が研修に何を求めるかを確認した
□ イノベーションの「期待領域」を経営者と共有した（アンゾフのマトリクスで確認）
□ 研修初日に経営者から受講者へ期待領域を直接表明してもらう予定
□ 最終日に経営者が資源動員の意思決定をできる環境を整えた
□ 研修期間中のオーナー交代リスクを確認した
```

## 4. プロジェクト型研修の設計・運営

### 標準プログラム構成（5セッション + 4フィールドワーク）

```
[Session 1] イノベーションを体感させる（終日）
  Step 1: 研修の目的・ゴール共有（経営者からの期待表明）
  Step 2: イノベーターのインプット（社外専門家・イノベーターの話を聞く）
  Step 3: 事業案フレームのインプット
  Step 4: 「不」の掘り下げ演習
  Step 5: 次回までの課題説明（フィールドワークへの送り出し）
  Step 6: 目標設定

  ↓ フィールドワーク①（1ヶ月程度：「不」の探索）

[Session 2] 「不」を発表・深掘りする（終日）
  Step 1: 「不」の内容発表
  Step 2: 全員で「不」を深掘り（メタ認知・問い直し）
  Step 3: アクションプラン発表

  ↓ フィールドワーク②（1ヶ月程度：「不」の深耕・事業案の骨格形成）

[Session 3] 中間プレゼン（終日）
  Step 1: プレゼン練習
  Step 2: 中間プレゼン（経営者・関係者への中間報告）
  Step 3: アクションプランの発表・フィードバック

  ↓ フィールドワーク③（1ヶ月程度：事業案のブラッシュアップ・資源調査）

[Session 4] 事業案の磨き合い（終日）
  Step 1: 模擬プレゼンと評価
  Step 2: プレゼンのアドバイス
  Step 3: アクションプランの発表

  ↓ フィールドワーク④（1〜2週間：最終調整・プレゼン資料完成）

[Session 5] 最終プレゼン（終日）
  Step 1: プレゼン練習・資料修正
  Step 2: 最終プレゼン（経営者への提言・資源動員の意思決定）
  Step 3: 研修全体の振り返り
  Step 4: 表彰式・懇親会
```

### アフターフォローの設計

最終プレゼン後も「伴走」が必要:

1. **研修結果を社内に共有** — プロセス・成果・事業案を社内広報する
2. **資源動員されるまで見守る** — 事業案が承認された後も、実際に資源が動くまでフォロー
3. **事業立ち上げに伴走する** — 事務局がインキュベーターとして機能することも

次のステップとして起こり得ること:
- テストマーケティングの実施
- プロジェクト発足・チーム組成
- 新規事業部門への異動
- 別法人の設立
- 外部専門家・投資家との連携

## 5. 「場」の設計

### 非日常の「開かれた共同体」

- 普段の仕事場所とは別の場所で実施する（オフサイト）
- 物理的・心理的に「非日常」を感じられる環境（遠隔地、フューチャーセンター等）
- 上司・部下の関係を切り離す（ウィークタイズの状態にする）

**ウィークタイズ（弱いつながり）の原則**:
```
ストロングタイズ（強いつながり）= 上司・部下・直属チーム
  → 役割・指揮命令系統が固定 → 新しい発想が生まれにくい

ウィークタイズ（弱いつながり）= 異部署・異世代・社外の人との接点
  → 「新結合」が起きやすい → イノベーションが生まれやすい
```

実践上のポイント:
- 受講者選定時に上司・部下の組み合わせを避ける
- 異業種・異部署の受講者を意図的に組み合わせる
- フィールドワークで普段接触しない人（顧客・専門家・社外の人）との接点を作る

## 6. 協働型イノベーション研修

通常の社内研修を超えた「協働型」の展開形態。

### 社会課題軸

当該社会課題領域の専門家（NPO、行政、研究者等）と議論しながら事業案を磨く。受講者の所属企業を越えた「協働」が、より大きな「不」の探索を可能にする。

- 被災地・地域再生テーマでの実施（複数企業から受講者が参加）
- ヘルスケア・高齢化・教育など社会課題領域での実施

### 技術軸

最新技術（AI・IoT・VR・素材等）を持つ専門家と組み合わせ、技術で解消できる「不」を探索する。

- 技術者が「不」から解消策を考える（技術先行から逆転）
- 研究開発部門のイノベーション文化醸成に有効

### 地域軸

地域企業・地方自治体・地域コミュニティと組んで、地域固有の「不」を解消する事業案を作る。

- 地域の課題は地域で解決する（地元のリソースを最大活用）
- インターンシップ・学生と企業の協働形態も可能

### 全員参加のイノベーション体制

```
イノベーション部門だけが考える ← 非効率・責任集中
       ↓
全社員がイノベーションを日常的に考える ← 創出確率の最大化
       ↓
必要なもの:
  1. イノベーション研修の定期実施（受講機会の整備）
  2. 新規事業提案制度（エントリー型・全社公募型）
  3. 事業案フレームの全社共通言語化
  4. 評価制度にイノベーション項目を組み込む
  5. 社内での成功事例の共有・称賛の文化
```

## 7. AskUserQuestion の使用タイミング

以下の場面では、ユーザーに確認してから設計を進める:

**イノベーション研修の主目的の確認**
```
「この研修で最も重視したい効果はどれですか？
 1. イノベーションリーダーの人材開発
 2. 次世代人材の発掘・見極め
 3. 実際の新規事業案の創出（事業開発）
 4. 組織風土のイノベーション文化への転換
 5. 全社的な危機感の醸成
 6. 経営者の原点回帰」
```

**研修期間・規模の確認**
```
「イノベーション研修の形式はどのようなものを想定していますか？
 1. 社内単独型（自社の受講者のみ）
 2. 協働型・社会課題軸（社外の専門家・他企業と協働）
 3. 協働型・技術軸（最新技術の専門家と協働）
 4. 協働型・地域軸（地域企業・自治体と協働）
 5. インターンシップ型（学生と企業の協働）」
```

**経営者の関与度の確認**
```
「経営者はどの程度この研修に関与できますか？
 1. 全セッション参加・最終プレゼンで意思決定できる
 2. 最終プレゼンのみ参加・意思決定は後日
 3. 担当役員が代理で参加
 4. 経営者の関与が難しい（別の成果設計が必要）」
```

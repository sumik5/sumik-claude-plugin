## 1. 使用タイミング

以下の場面でこのスキルを使用する:

- プレゼン草稿・アウトラインのレビューと改善
- 既存スライドデッキの構成見直し
- 話したい内容をプレゼン形式に整理
- プレゼンスクリプトの明確化・インパクト向上
- プレゼン本番前のデリバリー準備

**このスキルの対象外（別スキルへ）:**
- HTMLスライドの技術的生成 → `slidekit-create`
- Google Slides APIによる出力 → `generating-google-slides`
- 文章レベルの推敲・AI臭除去 → `writing-effective-prose`

## 2. プレゼン改善の5領域

プレゼンの質は5つの独立した領域で評価・改善できる。

### 2.1 マインドセット・準備

**核心原則**: プレゼンターの「なぜこれを伝えたいのか」という情熱が、聴衆への伝播力を決定する。

| チェックポイント | 問い |
|----------------|------|
| 情熱の明確化（譲れない想い） | 自分がこのテーマを本当に重要だと思う理由は何か？ |
| ゴール設定（3つのゴール） | ①聴衆がハッピーになる ②行動を引き出す ③言いふらしたくなる |
| 思考整理（核の設計） | 1フレーズ・20文字以内に凝縮できるか？（戦略＝「略す」こと） |
| 聴衆分析 | 誰が意思決定者か？ 何を知り、何を知らないか？ |

**思考整理フレームワーク（3つの問い）:**
```
What（何を）: このプレゼンで伝える核心メッセージ
Why（なぜ）: 聴衆にとってなぜ重要か
How（どう）: 行動や次のステップ
```

**聴衆分析（3軸 + 重視タイプ）:**
- 立場: 意思決定者 / 情報収集者 / 現場実行者
- 知識レベル: 専門家 / 中程度 / 初心者
- スタンス: 賛成寄り / 中立 / 反対寄り
- 重視タイプ: 結果重視 / 革新性重視 / 確実性重視 / 関係性重視（→ アプローチ・訴求軸を変える）

**準備の時間配分（目安）:** ストーリー設計 40% ／ リハーサル 30% ／ 資料作成 30%

### 2.2 ストーリー・構成

**核心原則**: ストーリーは話し手と聞き手の脳を同期させる（神経科学的根拠）。箇条書きではなくストーリーが記憶に残る。

**ストーリーテリング3類型:**

| 類型 | 内容 | 効果 |
|------|------|------|
| 個人ストーリー | 自分自身の体験・失敗・転換点 | 信頼感・共感の構築 |
| 第三者ストーリー | 顧客・ユーザー・事例 | 社会的証明・具体化 |
| ブランドストーリー | 組織の理念・使命・変革の軌跡 | 大義・文脈の提供 |

**ロジカル構成（ピラミッド構造）:**

```
メインメッセージ（1つ）
    ├── サポートポイント1（理由・根拠）
    ├── サポートポイント2
    └── サポートポイント3（最大3つまで）
        └── 各ポイントに「結論→理由」の順序
```

**論理の型と聴衆スタンスの対応:**

| 聴衆スタンス | 論理の型 | 構成順序 |
|------------|---------|---------|
| 単刀直入を好む | 帰納法 | 結論→根拠1,2,3 |
| じっくり説得 | 関係論証法 | 状況→課題→解決→結果 |
| 厳格な論理を望む | 演繹法 | 大前提→小前提→結論 |
| 三幕で動かす | 1:2:1 三幕構成 | 導入(1)→本体(2)→結び(1) ※詳細 → [LOGICAL-STRUCTURE.md](references/LOGICAL-STRUCTURE.md) |
| 関心→行動を促す | AIDMA | Attention→Interest→Desire→Memory→Action |
| 事実→行動を促す | 空・雨・傘型 | 事実（空）→解釈（雨）→行動提案（傘） |

**フック（冒頭の引き）の技法:**
- 謎・問いかけ: 「なぜ〇〇は△△なのか？」
- 驚きの事実: 意外性のある数字・事例から開始
- 物語の途中から: クライマックスから始め、経緯を後で語る
- 「バスの行き先」理論: 最初の1分で「このプレゼンは自分に関係ある」と示す
- エピソード＋問題提起＋結論: 体験談で感情を掴み、問題を示し、解決策で締める

### 2.3 スライド・資料デザイン

**核心原則**: スライドは「話を補助するもの」。スライドが主役になった瞬間、プレゼンは失敗する。

**2種類の品質基準:**

| 基準 | 内容 | チェック方法 |
|------|------|------------|
| リスクフリー | ミスを防ぐ（誤字・事実誤認・著作権） | 印刷して目視確認 |
| ストレスフリー | 読みやすさ最適化（視覚的快適さ） | 3秒ルール（3秒で要点が分かるか） |

**KISS原則（Keep It Simple & Straight）:**
- 1スライド = 1メッセージ
- 文字は最小限（タイトル + キーワード程度）
- フォント: ゴシック系、24pt以上
- 背景: 白または単色
- アニメーション: 原則使用しない
- 1〜3分の話に対してスライド1枚

**チャート選択ガイド:**

| 伝えたい内容 | 使うチャート |
|------------|------------|
| テキスト（論点・結論） | テキストチャート（KISS形式） |
| 概念・関係性 | コンセプトチャート（図形・矢印） |
| 定量比較・傾向 | データチャート（棒・折線・円） |

**余白の重要性**: 余白には心理的余裕を生む効果がある。情報を詰め込むほど伝わらない。

### 2.4 デリバリー・話し方

**核心原則**: 聴衆は「あなたがどう話すか」で判断の60〜90%を決める。内容だけでは伝わらない。

**説得の3要素（ロゴス・パトス・エトス）:**

| 要素 | 内容 | 理想配分の目安 |
|------|------|-------------|
| エトス（信頼感） | 実績・立場・誠実さの提示 | 10〜15% |
| ロゴス（論理） | データ・根拠・エビデンス | 25〜30% |
| パトス（感情） | ストーリー・情熱・共感 | 55〜65% |

> 最も説得力のあるプレゼンは、論理だけでなく感情への訴えが大きな割合を占める。

**話し方のベストプラクティス:**
- 会話するように話す（原稿を読まない）
- 「間」を3種類使い分ける: 理解の間（句読点・1〜2秒）/ 強調の間（重要事項の前後・2〜3秒）/ 集中の間（問いかけ後・3〜5秒）
- 声の抑揚: 強調したい箇所でゆっくり・大きく
- 視線: 聴衆全体を1〜2秒ずつ見渡す（ワンセンテンス・アイコンタクト）
- ボディランゲージ: 開いた姿勢、身体の動きを意識的に使う
- 自分らしくある（オーセンティシティ）: 信念に基づく自然な感情表現

**スライド操作の3T原則（Touch→Turn→Talk）:**
1. Touch: スライドの説明箇所をタッチ（または口頭で場所を指定）
2. Turn: 聴衆の方へ向き直る
3. Talk: 聴衆を見ながら説明する（スライドを背に読まない）

**18分ルール（時間管理）:**
- 人間の集中力は15〜18分が限界（認知負荷の科学的根拠）
- 18分を超えるプレゼンには「区切り」と「リセット」を設ける
- 区切りのたびに「ここまでのまとめ」を入れる

**リハーサルと緊張管理:**
- 本番前に最低でも3回フル通し練習
- 自分の話を録音・録画して客観視
- 緊張管理: 本番2分前にパワーポーズ（大きな姿勢）でホルモンバランスを調整
- 「緊張トリセツ」を事前に作成: 症状ごとの対策を書き出してパターン化する（→ 詳細 [DELIVERY-TECHNIQUES.md](references/DELIVERY-TECHNIQUES.md)）

**Q&Aへの備え:**
- 想定Q&Aを事前に5〜10問準備する
- 「ちゃぶ台返し」「揚げ足取り」など質問タイプ別に対応を準備する（→ 詳細 [DELIVERY-TECHNIQUES.md](references/DELIVERY-TECHNIQUES.md)）
- 回答に詰まったら「確認して後ほど回答します」が誠実な対応

### 2.5 聴衆エンゲージメント

**核心原則**: 人は受動的に聞くよりも、能動的に参加することで理解・記憶する（ラーニングピラミッド原則）。聞き手主体の「聞く」プレゼンへの転換が現代のスタンダード。

**「聞く」パラダイム（3層）:**
- 自分に聞く: 自分が本当に伝えたい「動機」を明確にする
- 相手に聞く: 相手の目的・モチベーションタイプを把握する
- 場に聞く: 場の空気・参加者の反応を読み、柔軟に対応する

**エンゲージメント技法一覧:**

| 技法 | 内容 | 適用場面 |
|------|------|---------|
| 驚きの瞬間（Holy Shit Moment） | 意図的に「え？」という瞬間を演出 | プレゼン中盤で注意を引き直す |
| 五感刺激 | 視覚・聴覚・触覚的な体験要素 | 製品デモ・実物提示 |
| 問いかけ | 聴衆に考えさせる質問 | 導入・セクション転換 |
| ビューポイント提示 | 「自分はこう思う」という明確な立場表明 | 差別化・記憶定着 |
| 双方向性 | Q&A・ワーク・ディスカッション | 長めのプレゼン |
| ユーモア | 笑いを取らず、軽快さを演出 | 緊張緩和・親近感醸成 |
| PUNCH原則 | Personal / Unexpected / Novel / Challenging / Humorous | 冒頭90秒のつかみ |
| インクルーシブ言語 | 主語を「私」から「皆さん」「あなた」に変えて聴衆を包摂 | 全体向け発信 |

**記憶定着の鉄則:**
- 繰り返し: 重要なメッセージは3回言及（冒頭・中盤・末尾）
- ストーリーに乗せる: 事実よりも物語で聞き手の脳に刻む
- ビジュアルとセット: 言葉と視覚の組み合わせで記憶効率が上がる

## 3. 改善ワークフロー（5ステップ）

### Step 1: 入力受付と目的確認

ユーザーから以下のいずれかを受け取る:

| 入力形式 | 対応 |
|---------|------|
| プレゼン草稿（テキスト） | そのまま分析開始 |
| スライドアウトライン | 構成分析を実施 |
| 話したい内容のメモ | 整理→構成化を支援 |
| 既存スライドの説明 | 構成・デザイン改善を提案 |

目的が不明な場合は **AskUserQuestion** でプレゼンの種類・目的・聴衆を確認する（Section 6参照）。

### Step 2: 現状分析（プレゼン種類・聴衆分析）

**プレゼン種類の判定:**

| 種類 | 目的 | 主な聴衆 | 重点領域 |
|------|------|---------|---------|
| 提案型 | 意思決定を促す | 上司・クライアント・投資家 | 論理構成・ROI・リスク |
| 報告型 | 情報を正確に伝える | 上司・チーム・ステークホルダー | 明確性・完全性 |
| 啓発型 | 気づきを与える・考えを変える | 一般聴衆・学習者 | ストーリー・感情・記憶 |
| ピッチ型 | 投資・支持・協力を得る | 投資家・経営陣・パートナー | 情熱・差別化・実現可能性 |

**聴衆分析チェックリスト:**
- [ ] 意思決定者は誰か（DMU: Decision Making Unit）
- [ ] テーマへの知識レベルはどの程度か
- [ ] 提案に賛成/反対/中立か
- [ ] 合理訴求派か感情訴求派か
- [ ] 持ち時間は何分か

### Step 3: 改善領域の特定

全領域を同時に改善しようとするとオーバーキルになる。ユーザーに重点領域を確認し、優先順位を決める。

| 状況 | 優先領域 |
|------|---------|
| 「内容は決まっているが伝わりにくい」 | 2.2 ストーリー・構成 |
| 「スライドが見づらい・詰め込みすぎ」 | 2.3 スライドデザイン |
| 「本番で緊張する・話し方が硬い」 | 2.4 デリバリー |
| 「聴衆の反応が薄い」 | 2.5 エンゲージメント |
| 「そもそも何を伝えるか整理できていない」 | 2.1 マインドセット・準備 |

### Step 4: 改善提案の実行

特定した領域の詳細ガイド（references/）を参照しながら、具体的な改善案を提示する:

1. **構成の再設計**: 現状の構成→改善後の構成を並列提示
2. **メッセージのシャープ化**: 「言いたいことが多すぎる場合」の1メッセージ化
3. **スライド改善案**: 各スライドのビフォー/アフター
4. **デリバリーアドバイス**: 話し方・リハーサル方法

### Step 5: 出力と次のアクション

改善提案の出力後、必要に応じて技術的出力スキルへ橋渡しする:

```
コンテンツ改善完了
    ├── HTMLスライド化 → slidekit-create
    ├── Google Slides化 → generating-google-slides
    └── スクリプト文章化 → writing-effective-prose
```

## 4. プレゼン種類別ガイド

### 4.1 提案型プレゼン

**目標**: 聴衆に特定の行動・意思決定をしてもらう

**推奨構成（SCQ + ピラミッド）:**
```
1. 導入（Situation・Complication・Question）
   - 現状（Situation）: 誰もが合意できる前提事実
   - 課題（Complication）: 現状に生じた問題・変化
   - 問い（Question）: それに対してどうすべきか？

2. 提案（Answer = メインメッセージ）
   - 「〇〇すべきです」という明確な結論

3. 根拠（Why × 3）
   - 帰納法で根拠を3つ提示
   - 各根拠にデータ・事例を添える

4. 実行計画（How）
   - タイムライン・リソース・リスク対策

5. 結び（感情訴求）
   - 行動しない場合のリスク or 行動した場合のベネフィット
```

**重要チェックポイント:**
- [ ] キーパーソン（意思決定者）のゴールと一致しているか
- [ ] Win-Winの落としどころを把握しているか
- [ ] 反対意見への回答を準備しているか

### 4.2 報告型プレゼン

**目標**: 情報を正確かつ効率的に伝える

**推奨構成:**
```
1. 結論（最初に言う）
2. 現状・背景
3. 詳細データ・事実
4. 次のアクション（あれば）
```

**重要チェックポイント:**
- [ ] 過剰一般化をしていないか（「〜が多い」ではなく数値で示す）
- [ ] 情報は中立的に提示されているか
- [ ] スライドは3秒で要点が分かるか（ストレスフリー原則）

### 4.3 啓発型プレゼン

**目標**: 聴衆に新しい視点・気づきを与える

**推奨構成（TED型）:**
```
1. フック（聴衆の心をつかむ冒頭）
2. 核心アイデア（明確に1つ）
3. ストーリー + 驚きの瞬間
4. 実践的示唆（聴衆が「明日できること」）
5. 印象的な締めくくり（元のフックに戻る）
```

**重要チェックポイント:**
- [ ] 「1つの大きなアイデア」が明確か
- [ ] 最低1つの「驚きの瞬間」があるか
- [ ] ストーリーがプレゼンの65%以上を占めているか
- [ ] 18分以内に収まるか

### 4.4 ピッチ型プレゼン

**目標**: 投資・支持・リソースを獲得する

**推奨構成:**
```
1. 問題提示（聴衆が「確かに」と感じる課題）
2. ソリューション（自分たちの提案）
3. 差別化（なぜ自分たちか）
4. 実績・証拠（小さくても具体的に）
5. 要求（何を求めているか明確に）
```

**重要チェックポイント:**
- [ ] 情熱・オーセンティシティが伝わっているか
- [ ] 「なぜあなたたちか」の答えが明確か
- [ ] 要求（Ask）が明確か

### 4.5 社内プレゼン型（報告＋提案ハイブリッド）

**目標**: 上司・関係部門の承認・理解を得て、スムーズに意思決定を進める

**QCD期待値コントロール:**
- Q（Quality）: 何を達成するか（品質・成果基準）を明示
- C（Cost）: 予算・リソースの根拠を示す
- D（Delivery）: タイムラインと進捗管理方法を明確にする

**推奨構成:**
```
1. 結論（承認してほしいこと・判断を求める内容）
2. 現状・課題（なぜ今やるか）
3. 提案詳細とQCD（Q/C/D それぞれ明示）
4. リスクと対策
5. 次のアクション（承認後の初手）
```

**重要チェックポイント:**
- [ ] 意思決定者の「重視タイプ」（結果/革新/確実/関係）に合わせた訴求か
- [ ] 反対意見・懸念への先回り回答があるか

### 改善チェックリスト（全種類共通）

**コンテンツ:**
- [ ] メインメッセージが1つに絞られているか
- [ ] 聴衆分析（Who・What・Why）が完了しているか
- [ ] ストーリーが論理を補完しているか

**スライド:**
- [ ] 1スライド = 1メッセージになっているか
- [ ] テキスト量が最小限か（KISS原則）
- [ ] 視覚的階層が明確か

**デリバリー:**
- [ ] 18分ルールを意識した時間配分か
- [ ] 冒頭と末尾が印象的か

## 6. ユーザー確認の原則（AskUserQuestion）

**基本方針**: 推測で進めず、判断分岐のある場面では必ず確認する。

### 確認すべき場面（必須）

```python
AskUserQuestion(
    questions=[
        {
            "question": "プレゼンの種類を教えてください",
            "header": "種類",
            "options": [
                {"label": "提案型", "description": "意思決定・承認を得るプレゼン"},
                {"label": "報告型", "description": "情報・状況を正確に伝えるプレゼン"},
                {"label": "啓発型", "description": "気づきを与える・考えを変えるプレゼン"},
                {"label": "ピッチ型", "description": "投資・支持・協力を獲得するプレゼン"},
            ],
            "multiSelect": False
        },
        {
            "question": "主な聴衆はどのような方ですか？",
            "header": "聴衆",
            "options": [
                {"label": "意思決定者（上司・クライアント・経営陣）"},
                {"label": "現場実行者・チームメンバー"},
                {"label": "一般聴衆・参加者"},
                {"label": "投資家・パートナー"},
            ],
            "multiSelect": False
        }
    ]
)
```

| 確認場面 | 理由 |
|---------|------|
| プレゼン種類の判定 | 構成アプローチが完全に異なるため |
| 聴衆の特定 | メッセージの粒度・トーンが変わるため |
| 重点改善領域 | 全領域フル適用するとオーバーキルになるため |
| 時間制約（持ち時間） | 18分ルール適用と構成密度が変わるため |

### 確認不要な場面

| 場面 | 理由 |
|------|------|
| 1メッセージ原則の適用 | 普遍的ベストプラクティス |
| ストーリーテリングの推奨 | 神経科学的根拠により確立 |
| 余白・シンプル化の推奨 | 資料設計の基本原則 |
| 18分ルールの言及 | 認知負荷の科学的根拠あり |

## 9. アンチパターン集

### マインドセットのアンチパターン（7大勘違い）

| 勘違い | 現実 |
|--------|------|
| 「プレゼン＝自分が主役」 | 主役は聴衆。伝えたいことではなく聴衆に届けたいことを考える |
| 「話し上手はアドリブが得意」 | 準備・練習が多いほど余裕が生まれる。アドリブは準備の産物 |
| 「論理的に話せばいい」 | 感情（パトス）が記憶・行動の鍵。論理だけでは動かない |
| 「資料作成に一番時間をかける」 | ストーリー設計に最大の時間を使う（40%ルール） |
| 「緊張はダメなこと」 | 緊張は集中力を高めるエネルギー。コントロールすれば武器になる |

### コンテンツのアンチパターン

| アンチパターン | 問題 | 改善策 |
|-------------|------|-------|
| 「伝えたいことが全部ある」 | メッセージが分散し何も残らない | 1メッセージに絞り込む（核を設計する）。他は補足資料へ |
| ノイズの放置 | 核に関係ない情報・言葉・動きがすべてノイズになる | 「これを外したら核が伝わらないか？」で判断する |
| データ羅列 | 論理的に見えるが感情的共感ゼロ | データの前後にストーリーを配置する |
| 結論を最後に言う | 聴衆が途中で迷子になる | 結論→根拠の順（帰納法）を基本とする |
| 専門用語の多用 | 聴衆を置いてきぼりにする | 聴衆の知識レベルに合わせた言葉に変換 |

### スライドのアンチパターン

| アンチパターン | 問題 | 改善策 |
|-------------|------|-------|
| 文字ぎっしりスライド | 読まれない・話し手が不要になる | 1スライド = 1メッセージ + キーワードのみ |
| アニメーション過多 | 注意が分散・テクニカルトラブルリスク | アニメーションは原則使用しない |
| 小さいフォント | 後方から読めない | 最低24pt以上 |
| カラフルすぎる | 視覚的疲労・焦点が定まらない | 1〜2色に統一 |

### デリバリーのアンチパターン

| アンチパターン | 問題 | 改善策 |
|-------------|------|-------|
| 原稿の棒読み | 信頼感・情熱がゼロになる | キーワードだけメモし、会話するように話す |
| スライドを背に向いて読む | 聴衆との接続が切れる | スライドは聴衆に見せるもの。自分はノートPCを見る |
| 単調な話し方 | 睡眠効果をもたらす | 抑揚・間・速度変化を意識的に使う |
| 時間超過 | 聴衆の集中が切れる | 18分を目安に必ずリハーサルで計測 |

### エンゲージメントのアンチパターン

| アンチパターン | 問題 | 改善策 |
|-------------|------|-------|
| 「ご質問はありますか？」だけ | 質問が出ない沈黙 | 具体的な問いかけを準備しておく |
| 一方通行のプレゼン | 聴衆が受動的になる | 問いかけ・ワーク・ペアワークを組み込む |
| 感情なし・情熱なし | 何も残らない | 「なぜ自分がこれを伝えたいか」を前面に出す |

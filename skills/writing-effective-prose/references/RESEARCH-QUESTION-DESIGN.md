## 1. テーマと「問い」の違い

多くの書き手が「テーマ」を決めたつもりで「問い」を立てていない。

| 項目 | テーマ | 問い |
|------|--------|------|
| **性質** | 研究・論述の領域・対象 | 論文が答えるべき具体的な疑問 |
| **形式** | 名詞句・キーワード | 疑問文（「〜か」「〜なぜ」「〜どのように」） |
| **完結性** | 論文を書いても「答え」が出ない | 論文を通して「答え」が出る |
| **例** | 「動物の権利」 | 「動物に法的権利を認めるべきか」 |

### Before/After の例

| Before（テーマ止まり） | After（問いに変換） |
|----------------------|-------------------|
| 「企業倫理について」 | 「なぜ日本企業における不祥事は繰り返されるのか」 |
| 「SNSと若者」 | 「SNSの長時間利用は大学生の学業成績にどのような影響を与えるか」 |
| 「気候変動問題」 | 「日本の企業はカーボンニュートラル目標の達成に向けどのような取り組みが有効か」 |
| 「ヘーゲル哲学」 | 「ヘーゲルの「相互承認」概念は現代の職場における権威関係にどう適用できるか」 |

## 2. 良い問いの条件

### 2-1. 必須条件（これが満たされないと論文にならない）

| 条件 | 説明 | 問題のある例 |
|------|------|-------------|
| **答えが自明でない** | 「明らかにそうだ」「当然だ」が答えにならない | 「殺人は悪いか」 |
| **答えが検証可能** | 何らかの証拠・調査によって答えが導ける | 「神は存在するか」（純粋な信仰問題として） |
| **範囲が適切** | 与えられた紙幅・期間内で論じられる | 「近代とはどのような時代か」 |
| **論文に結びつく** | 資料・データ・先行研究にアクセスできる | 「ミソ煮込みうどんはいかにして始まったか」 |

### 2-2. 良い問いの追加条件

- **新規性がある**: 既に明確に答えられていない問い（先行研究でまだ議論が決着していない）
- **重要性がある**: 答えを知ることに意味がある（学術的・社会的価値）
- **具体性がある**: 言葉の意味が明確で、何を調べれば答えが出るかがわかる

### 問いの自己チェック

答えを出すために何をすればよいか、具体的な手順が浮かぶか？ 浮かばなければ問いがまだ曖昧すぎる。

## 3. 問いの型：What/Why/How/Should

問いのタイプを意識すると、論文の方向性と必要な証拠がはっきりする。

| 問いの型 | 日本語表現 | 求められるもの | 主な証拠の種類 |
|---------|-----------|--------------|--------------|
| **What型** | 「〜とは何か」「〜にはどのような特徴があるか」 | 定義・分類・記述 | 文献分析・調査データ |
| **Why型** | 「なぜ〜なのか」「〜の原因は何か」 | 因果・メカニズムの説明 | 事例分析・統計・先行研究 |
| **How型** | 「どのように〜が起きているか」「〜はどのように機能するか」 | プロセス・メカニズムの記述 | 事例分析・インタビュー・観察 |
| **Should型** | 「〜すべきか」「〜は許されるか」 | 価値判断・規範的結論 | 先行研究・論理的推論・事例 |

### 型ごとの論文構成の違い

**Why型の例（なぜ起きたか）**:
```
問い → 仮説（原因の候補） → 証拠の検討 → 最もよい説明の選択 → 結論
```

**Should型の例（すべきか否か）**:
```
問い → 判断基準の提示 → 賛否双方の論拠 → 評価 → 結論（付帯条件付き）
```

## 4. テーマから問いを絞り込む手順

### 手順の全体像

```
漠然とした関心（興味・疑問・問題意識）
    ↓
Step 1：入門書・概説書でその分野の全体像を把握
    ↓
Step 2：その分野でどんな問いが議論されているかを特定
    ↓
Step 3：自分が「これは面白い、これは答えが出ていない」と感じる問いを選ぶ
    ↓
Step 4：問いを絞り込む（大きすぎる問いを小さくする）
    ↓
Step 5：問いを定式化する（疑問文で書く）
    ↓
Step 6：問いが論文に結びつくか確認（資料・先行研究へのアクセス）
```

### Step 4 の絞り込み方

大きすぎる問いを小さくするには、問いを「分解」する。

| 大きな問い | 分解の視点 | 絞り込んだ問い |
|----------|-----------|--------------|
| 「動物実験は許されるか」 | 動物の種類・実験の目的で限定 | 「化粧品開発を目的とした哺乳類への毒性試験は倫理的に正当化できるか」 |
| 「SNSと社会問題」 | 特定のSNS・特定の問題に限定 | 「Twitterにおけるヘイトスピーチは匿名性の廃止によって減少するか」 |
| 「遠隔授業の効果」 | 効果の種類・対象者で限定 | 「大学初年次生の授業満足度に対して、遠隔授業と対面授業はどのように異なる影響を与えるか」 |

## 5. 先行研究からの問い発見

既存の研究を読むことで、答えるべき問いが見えてくる。

### 先行研究から問いを見つけるアプローチ

| アプローチ | 問いの形式 | 例 |
|-----------|-----------|-----|
| **空白を埋める** | 「〜についてはまだ研究されていない」 | A地域での調査はあるが、B地域では未研究 |
| **矛盾を解く** | 「AとBの研究が互いに矛盾している」 | 研究者Xは賛成、Yは反対。なぜか |
| **条件を問い直す** | 「Xが成立する条件は何か」 | 従来の研究はXを前提にしているが本当に成立するのか |
| **文脈を変える** | 「別の文脈でも同じことが言えるか」 | 欧米での研究結果は日本にも当てはまるか |
| **反論を立てる** | 「通説・定説Xに疑問がある」 | 通説Xでは説明できない事例がある |

### 先行研究の読み方（問い発見のために）

1. アブストラクト（要約）で問いと結論を確認する
2. 結論の末尾の「今後の課題（limitations）」セクションを読む
3. 論文内で「〜は今後の研究課題である」「〜については検討していない」と書かれている箇所を探す

## 6. 「問い」の修正と発展

問いは書き始める前に確定する必要はない。書きながら問いを磨いていくことが重要。

### 問いが変化する典型パターン

| 状況 | 対応 |
|------|------|
| 調査・読書を進めると問いが答えにくいとわかった | 問いの範囲を限定する |
| 最初の問いに答えるためには別の問いへの答えが先に必要だとわかった | 問いを分解して小問いのセットにする |
| 問いへの答えが調べる前から明らかだとわかった | 問いを言い直す（答えがより不確実な問いに修正） |
| 予想外のデータ・事実が出てきた | 問いを修正して新しいデータを取り込む |

### 問いの精度確認テスト

問いを他者に見せて次を確認する:
- 「この問いの答えは何か、見当がつくか？」（自明でないかのチェック）
- 「この問いに答えるためには何を調べればよいか？」（検証可能性のチェック）
- 「この問いに答えるのにどのくらいかかるか？」（範囲の適切さのチェック）

## 7. ブレインストーミングとマインドマップ

問いを発見する段階では、論理的整合性よりも発散思考が重要。

### ブレインストーミングのルール

1. **評価しない**: 「くだらない」「難しすぎる」と思っても書き出す
2. **質より量**: 思いついたことをすべて書き出す
3. **連想する**: ある問いから連想できる問いを芋づる式に広げる
4. **時間を区切る**: 10〜15分集中して書き出す

### マインドマップの使い方（問いの整理）

```
中心：テーマ（例：「遠隔授業」）
├─ 賛否：メリット → 学習効率/アクセシビリティ/コスト
│         デメリット → 孤立感/設備の不均等/実技不可
├─ 対象：大学生/社会人/小中学生/教員
├─ 文脈：コロナ禍/地方/障害のある学生
└─ 問いの候補：「誰にとって効果的か」「どんな科目に向くか」
```

マインドマップは問いを発散させる段階に有効。絞り込む段階では線形のアウトライン（Step 1〜6の手順）に移行する。

## 8. テーマ設定でよくある失敗パターン

### 失敗パターン一覧と対処法

| 失敗パターン | 問題の内容 | 修正策 |
|-------------|----------|--------|
| **大きすぎる問い** | 一生かけても答えが出ない | 時期・地域・対象・側面を限定 |
| **答えが自明な問い** | 誰もが同じ答えを知っている | 「なぜ」「どのように」に変換するか別の問いに変える |
| **資料のない問い** | 手がかりも調査方法もない | 先行研究・データが存在するかを確認 |
| **起源を問う問い** | 「いつ始まったか」は困難 | 「現在どうなっているか」「なぜ変化したか」に変換 |
| **コンサルティング型の問い** | 「どうすれば売れるか」「最適化するには」 | 分析的な問い（なぜ〜か、何が〜に影響するか）に変換 |
| **言葉の意味だけを問う問い** | 「権利とは何か」（辞書で調べればよい） | 「なぜ動物に権利を認めるべきか（認めるべきでないか）」に変換 |
| **テーマを並べただけ** | 「動物・権利・幸福」（問いになっていない） | 疑問文で書き直す |
| **広すぎる時期・地域指定** | 「戦後サブカルチャーの展開」 | 1980年代・特定ジャンル等で絞る |

### 問いの良し悪し判断フロー

```
1. 疑問文で書かれているか？
   → No → 疑問文に変換する

2. 答えが自明でないか？
   → No（自明） → 別の問いに変える

3. 与えられた紙幅・期間で答えられるか？
   → No（大きすぎ） → 問いを分解・限定する

4. 資料・先行研究にアクセスできるか？
   → No → 問いを修正するか資料を探す

5. 答えが出たとして、意味がある（重要性がある）か？
   → No → 問いの意義を再考する

→ すべてYesなら、論文の問いとして機能する可能性が高い
```

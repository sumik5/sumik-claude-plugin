## 1. アウトラインとは何か

アウトラインとは論文の「設計図」または「骨格」である。

> **論文はアウトラインを太らせたものだ。**

論文を書くことは「書き下ろす」ことではなく、まずアウトラインを作り、それに肉付けしていくプロセスである。アウトラインさえできていれば、目次もアブストラクト（要旨）も本文も、すべてそこから派生させることができる。

### アウトラインと目次の違い

| 項目 | アウトライン | 目次 |
|------|-----------|------|
| **誰のため** | 書く本人のため | 読者のため |
| **役割** | 論文の設計・思考ツール | 読み手の案内 |
| **タイミング** | 書き始める前から継続的に更新 | 論文完成後に確定 |
| **詳しさ** | 細かい論点・証拠まで含む | 見出し名だけ |

### アウトラインがないと起きること

- 「次に何を書けばよいかわからない」状態が続く
- 書いているうちに論旨がずれていく
- 書き終わって読み返すと構成がバラバラ
- 重複や抜けが後から発覚する

## 2. トップダウン型とボトムアップ型

アウトラインを作るアプローチには2種類ある。問いの定式化ができているかどうかで選ぶ。

### 2-1. トップダウン型（問いが定まっている場合）

問いと大まかな結論が見えている状態から始め、論証構造を組み立てていく。

```
Step 1：問い（リサーチクエスチョン）を確定する
    ↓
Step 2：その問いへの自分の答え（主張・仮説）を仮置きする
    ↓
Step 3：主張を支える論証ポイントを列挙する
    ↓
Step 4：各論証ポイントを支える証拠・根拠を書き出す
    ↓
Step 5：序論・本論・結論の各セクションに配置する
    ↓
Step 6：論証の順序を整える
```

**適した場面**: 論争型論文、短いレポート、問いが与えられた課題

### 2-2. ボトムアップ型（材料から構成を組み立てる）

資料・データを集めた後、そこから帰納的に構成を組み立てる。

```
Step 1：集めた資料・データ・アイデアをカードや箇条書きで書き出す
    ↓
Step 2：関連するものをグループ化する（カテゴリー分け）
    ↓
Step 3：各グループに名前（見出し）をつける
    ↓
Step 4：グループの論理的順序を決める
    ↓
Step 5：この構成が問いへの答えになっているか確認する
    ↓
Step 6：序論・結論を追加して完成させる
```

**適した場面**: 実証型論文、卒業論文・修士論文、先行研究が豊富な分野

## 3. 階層構造の作り方

アウトラインは階層構造（大見出し→中見出し→小見出し）で作る。

### 3-1. 基本的な3階層構造

```
【大見出し（章レベル）】
├─【中見出し（節レベル）】
│  ├─【小見出し（項レベル）】
│  │   └─ 具体的な論点・証拠（段落レベル）
│  └─【小見出し】
│      └─ 具体的な論点・証拠
└─【中見出し】
   └─ ...
```

### 3-2. カテゴリーの階層化の考え方

階層構造は「上位カテゴリー（概念）→下位カテゴリー（具体）」の関係で作る。

| 階層 | 役割 | 例 |
|------|------|-----|
| 大見出し（章） | 論文全体の大きな論証ステップ | 「チャレンジャー号事故の概要」 |
| 中見出し（節） | その章で扱う主要な論点 | 「打ち上げ前日の意思決定過程」 |
| 小見出し（項） | 論点を支える具体的な内容 | 「エンジニアの懸念と管理職の判断」 |

### 3-3. 階層ごとの詳細度の目安

短いレポート（4,000字以内）は2階層で十分。卒業論文は3〜4階層。

| 論文の種類 | 推奨階層数 | 章数の目安 |
|-----------|-----------|----------|
| 短いレポート（〜4,000字） | 2階層 | 3〜4節 |
| 通常のレポート（〜8,000字） | 2〜3階層 | 4〜5章 |
| 卒業論文（〜20,000字） | 3〜4階層 | 5〜7章 |
| 修士論文（〜60,000字） | 4階層 | 6〜8章 |

## 4. MECEの原則

アウトラインの各見出し・セクションは、MECEの原則に従うと構成が堅牢になる。

> **MECE（Mutually Exclusive, Collectively Exhaustive）**: 漏れなく・ダブりなく

### 4-1. 漏れのチェック（CE）

各セクションを合わせると、問いへの答えが完全に示されているか。

```
問い：「なぜ企業不祥事は繰り返されるか」

章立て（漏れあり）：
  ①個人の倫理意識の低さ
  ②組織文化の問題
  ③法規制の不十分さ

検討：競争圧力・情報開示の問題なども重要な要因では？
→ ④外部競争プレッシャーを追加
```

### 4-2. ダブりのチェック（ME）

異なるセクションで同じ内容を論じていないか。

```
ダブりの例：
  ①個人の倫理意識の低さ → 「幹部の判断ミス」
  ②組織文化の問題 → 「経営層の倫理意識の欠如」

→ ①と②は重複している。「個人の倫理意識」を②組織文化の一部として整理する
```

### 4-3. ズレのチェック（論点の整合性）

各セクションが問い・主張と関係しているか（関係ない情報を入れていないか）。

```
問い：「動物実験を制限すべきか」に対して

ズレているセクション：
  「③世界各国の動物実験の歴史」

→ 歴史は文脈として必要な範囲で言及するが、独立した章にする必要はない
```

## 5. アウトラインの推敲

アウトラインは書き始める前に一度完成させてからでなく、書きながら継続的に見直す。

### 5-1. 構成の見直し基準

| 問い | 確認内容 |
|------|---------|
| 問いへの答えになっているか | 序論と結論が対応しているか |
| 論証の順序は適切か | 前のセクションの内容が次のセクションの前提になっているか |
| 各セクションの分量は均等か | 極端に短い/長いセクションがないか |
| 重複はないか | 同じ内容を複数箇所で述べていないか |
| 抜けはないか | 反論・例外・限界についての言及があるか |

### 5-2. アウトラインの成長過程

論文作成における典型的なアウトラインの変化。

```
初期アウトライン（問いと大枠のみ）：
  Ⅰ はじめに
  Ⅱ 動物実験の現状
  Ⅲ 賛否の議論
  Ⅳ 結論

     ↓ 先行研究を読む中で変化

中間アウトライン（論点が具体化）：
  Ⅰ はじめに（問いと論文の構成）
  Ⅱ 動物実験の定義と現状（規模・目的別）
  Ⅲ 動物実験支持論の検討（医学的必要性論・代替手段論への反論）
  Ⅳ 動物実験批判論の検討（動物の権利論・功利主義的批判）
  Ⅴ 評価と結論（条件付き制限の提案）

     ↓ 執筆しながらさらに変化

最終アウトライン（証拠・例まで含む）：
  （各章に具体的な引用・データの配置が決まる）
```

## 6. 各セクションの分量配分

### 6-1. 標準的な分量配分

| セクション | 比率 | 内容 |
|-----------|------|------|
| **序論（はじめに）** | 10〜15% | 問いの提示・背景・論文の構成 |
| **本論** | 70〜80% | 論証の展開 |
| **結論（おわりに）** | 10〜15% | 主張の確認・含意・今後の課題 |

### 6-2. 本論内の分量配分

本論の中でも、論証のキモになる部分（問いへの直接的な答えを論証する部分）に最も多くの分量を割く。

```
【本論の構成例：論争型論文（8,000字）】

Ⅱ 背景・先行研究の整理（約1,500字）
  → 議論の文脈を示す

Ⅲ 反対意見の検討（約1,500字）
  → 想定される反論を整理・検討

Ⅳ 主張の論証（約2,500字）
  → 論文の核心：最も力を入れる

Ⅴ 例外・限界の検討（約1,000字）
  → 主張の適用範囲を明確にする
```

### 6-3. 分量のバランス崩れのサイン

| 問題 | 原因 | 対処 |
|------|------|------|
| 序論が長すぎる | 背景説明に深入りしすぎ | 本論に必要な情報のみに絞る |
| 結論が短すぎる | 「以上のとおり」で終わっている | 含意・今後の課題・自己評価を追加 |
| 特定の章だけ極端に長い | 特定テーマに深入りしすぎ | 他章と比較してバランスを取る |
| 先行研究紹介が本論より長い | 自分の論証が薄い | 先行研究は必要最小限に絞り、自分の論証を増やす |

## 7. アウトラインから本文への展開

アウトラインの各項目は、本文の「パラグラフ」に変換する。

### 7-1. アウトライン→パラグラフの変換ルール

| アウトラインの要素 | 本文での展開 |
|-----------------|------------|
| 中見出し（節） | パラグラフのグループ（3〜5段落程度） |
| 小見出し（項） | 1つのパラグラフ |
| 論点・証拠のメモ | パラグラフの根拠文（サポートセンテンス） |

### 7-2. パラグラフ化の手順

```
アウトラインの項目：
「エンジニアは打ち上げに反対していた（低温テストデータ）」

　↓

パラグラフに展開：
【トピック文】サ社のエンジニアたちは、打ち上げ当日の気温でのOリング
の機能について深刻な懸念を抱いていた。
【根拠文】打ち上げ前夜の電話会議で、エンジニアたちは低温データを
示し、気温が32°F（0°C）以下での打ち上げに明確に反対した
（○○、1987、p.XX）。
【展開文】しかし翌朝の直前会議で、マネジャーたちはエンジニアの
懸念を覆し、打ち上げを承認した。
【まとめ文・次への接続】この逆転の理由こそが、集団思考仮説の
核心的な問いとなる。
```

### 7-3. アウトライン段階で確認すべきこと

本文を書き始める前に、アウトラインの段階で：

1. **各節のトピック文**を一文で書けるか確認する
2. **節と節のつながり**が論理的かを確認する（前の節が次の節の前提になっているか）
3. **証拠の配置**が適切かを確認する（主張の直後に根拠が来ているか）

## 8. 実例：レポートと卒論のアウトライン

### 8-1. レポートのアウトライン例（4,000字程度）

**テーマ**: 新型コロナウイルス収束後も遠隔授業を継続すべきか

```
Ⅰ はじめに（問いの提示・論文の構成）       約400字
  1-1 問いの背景
  1-2 本論文の主張と構成

Ⅱ 遠隔授業の定義と現状                    約600字
  2-1 遠隔授業の種類
  2-2 コロナ禍での普及状況

Ⅲ 継続に賛成する根拠                      約1,000字
  3-1 アクセシビリティの改善（地方・障害者）
  3-2 時間的効率性
  3-3 アンケートによる満足度データ（○○調査、2021）

Ⅳ 継続に反対する根拠と検討                約1,000字
  4-1 対面授業が優れている点（コミュニティ形成）
  4-2 設備格差の問題
  4-3 科目別の適不適

Ⅴ 結論（条件付き継続の提案）              約600字
  5-1 主張の再確認：一部科目での継続を提案
  5-2 条件：設備支援・科目選定基準の整備
  5-3 今後の課題
```

### 8-2. 卒業論文のアウトライン例（20,000字程度）

**テーマ**: なぜスペースシャトル・チャレンジャー号事故は防げなかったか

```
Ⅰ はじめに                                約1,500字
  1-1 事故の概要と問いの提起
  1-2 本論文の主張（集団思考仮説）
  1-3 論文の構成

Ⅱ 事故の経緯                              約3,000字
  2-1 打ち上げまでの決定過程
  2-2 打ち上げ前夜の電話会議
  2-3 当日の最終決定と打ち上げ

Ⅲ エンジニアによる警告と無視の実態       約3,000字
  3-1 Oリングの技術的懸念
  3-2 エンジニアの反対意見の記録
  3-3 管理職側の判断の根拠

Ⅳ 集団思考仮説による説明                 約5,000字
  4-1 集団思考とは何か（Janis, 1972）
  4-2 チャレンジャー事故への適用
  4-3 集団思考仮説の支持証拠
  4-4 代替仮説との比較検討

Ⅴ 事故防止への示唆                        約2,500字
  5-1 組織的意思決定プロセスへの提言
  5-2 エンジニア倫理の役割
  5-3 他の事例への応用可能性

Ⅵ 結論                                   約1,500字
  6-1 主要な発見の要約
  6-2 本論文の限界と今後の課題
```

### 8-3. アウトラインの良し悪し比較

**Before（問いへの答えが見えない）**:
```
1. はじめに
2. 動物の権利とは何か
3. 動物実験について
4. 様々な立場から見た議論
5. おわりに
```

**After（論証の構造が見える）**:
```
1. はじめに（化粧品目的の動物実験は倫理的に許されるかを問う）
2. 現在の動物実験の実態（規模・目的・苦痛の程度）
3. 動物実験を支持する論拠の検討（医学的必要性）
4. 代替手段の存在と倫理的問題（なぜ苦痛を与える必要があるか）
5. 化粧品目的への限定適用（本論文の核心的論証）
6. 結論（化粧品目的の動物実験は代替手段があり倫理的に正当化できない）
```

## アウトライン作成のクイックチェックリスト

### 作成前
- [ ] 問いが疑問文で明確に書かれているか
- [ ] その問いへの仮の答え（主張）が一文で書けるか

### 作成中
- [ ] 大見出し（章）が3〜7個程度に収まっているか
- [ ] 各章に具体的な論点・証拠の候補が書き出されているか
- [ ] 序論・本論・結論の三部構成になっているか

### 見直し段階
- [ ] 章を順に読んで、論証の流れが通じるか（前の章が次の章の前提になっているか）
- [ ] MECE：漏れはないか（問いへの答えが不完全でないか）
- [ ] MECE：ダブりはないか（同じ内容が複数の章にないか）
- [ ] 序論の「問い」と結論の「答え」が対応しているか
- [ ] 分量配分：本論が全体の70〜80%を占めているか
- [ ] 反論・例外・限界への言及が含まれているか

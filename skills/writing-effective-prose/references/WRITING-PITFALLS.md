## 1. 論文の禁句集

### 1.1 「思う」系の表現

感想文では「思う」が自然だが、論文では分析・主張を根拠とともに示す必要がある。

| 禁句 | 問題点 | 修正例 |
|-----|-------|-------|
| `〜と思う` | 個人的な印象に過ぎない | `〜と考えられる` / `〜と言える` |
| `〜だと思います` | 主観的な感想 | `〜が明らかになった` / `〜が示されている` |
| `〜と思いました` | 読書感想文の表現 | `〜が確認できた` / `〜と結論づけられる` |
| `〜ではないかと思う` | 二重の曖昧さ | `〜という可能性がある` / 根拠を示した上で`〜と考える` |
| `個人的には〜だと思う` | 「個人的には」自体が論文不要 | 主語を変えて根拠とともに述べる |

**注意**: 「〜と考える」も多用すると主観的印象を与える。可能な限り「〜と言える」「〜が示されている」など根拠に基づく表現を使う。

### 1.2 曖昧な数量表現

| 禁句 | 問題点 | 修正例 |
|-----|-------|-------|
| `たくさんの研究がある` | 「たくさん」の基準が不明 | `多数の研究がある（例: Smith 2019; Jones 2021）` |
| `多くの人が〜と言っている` | 根拠なしの一般化 | `〇〇の調査によれば、回答者の△%が〜と述べた` |
| `ほとんどの〜` | 「ほとんど」の定義が不明 | `N件中M件（約X%）の〜` |
| `いくつかの問題がある` | 「いくつか」が何件か不明 | `以下の三点において問題がある。第一に〜` |

### 1.3 強調・副詞の過剰使用

| 禁句 | 問題点 | 修正例 |
|-----|-------|-------|
| `すごく重要` | 口語的、「すごく」に根拠がない | `特に重要` / `〜の点で重要` |
| `とても大切` | 口語的 | `〜において不可欠` / `〜の核心をなす` |
| `非常に深刻` | 「非常に」が抽象的 | 具体的な数値・事例で深刻さを示す |
| `かなり多い` | 「かなり」の基準が不明 | `平均を約X%上回る` |

### 1.4 ヘタ夫的フレーズ（論文頻出のNG表現）

これらは論文の論理的流れを断ち切る、あるいは書き手の曖昧さを隠す表現。

| 禁句 | 問題点 | 修正例 |
|-----|-------|-------|
| `ここで〜してみる` / `ここで〜見ていく` | 論理のつながりが不明。「見てみる」では何をするかわからない | 「〜を批判的に検討する」「〜の変遷を分析する」等、具体的な動詞を使う |
| `では、〜について見てみよう` | 「では」は話題転換の接続詞。論文の論理展開では使いにくい | 前の段落との論理的な接続を示した上で本題に入る |
| `そんな中、〜` | 「どんな中」かを言えていない言い逃げ | 前の内容を一文でまとめてから「このような状況のもとで、〜」 |
| `〜について検証する` | 「検証」は仮説の真偽を実験・調査で確かめること。単に「詳しく調べる」意味で使うのは誤用 | 「〜を詳しく分析する」「〜を考察する」「〜を検討する」 |
| `（幼稚な指摘かもしれないが）` | 自分で書いた内容を自己否定。見苦しい | 自信を持って書く。幼稚なら書かない |
| `倫理感・価値感` | 誤字。「感」ではなく「観」 | `倫理観・価値観` |
| `何か〜かどうか` / `どちらが〜かどうか` | 疑問詞と「かどうか」の二重使用 | `何を〜か` / `どちらが〜か` |

### 1.5 感想文・作文の表現

| 禁句 | 論文での問題 | 修正の方向 |
|-----|------------|----------|
| `この研究は私にとって非常に勉強になりました` | 論文の内外の話が混同されている | 研究の学術的意義を述べる |
| `最後に、先生への感謝を述べたい` | 論文本文に謝辞は不要（別紙で） | 謝辞は別ページに移す |
| `この問題は大変難しいと感じた` | 感想。難しさの構造を分析すべき | 「この問題が難しい理由は〜にある」 |
| `〜について調べてみた結果、〜だとわかった` | 「調べてみた」は研究プロセスの記述。不要 | 結果だけを述べる：「〜が明らかになった」 |

## 2. 構成上のよくある間違い

### 2.1 序論（はじめに）の間違い

| 間違い | 問題点 | 修正方向 |
|-------|-------|---------|
| **序論で結論を詳しく書く** | 論文を読む動機がなくなる | 序論では「問い」と「方法」のみを示す |
| **個人的な動機だけを書く** | 学術的な問いの重要性が示されない | 動機は1〜2文で済ませ、研究の意義を中心に書く |
| **定義の羅列から始める** | 読者を退屈させる | 問いを先に示し、必要な定義は本論中で行う |
| **「昔から〜と言われてきた」で始める** | 根拠のない一般論。陳腐な書き出し | 具体的な問題状況・データから入る |

### 2.2 本論の間違い

| 間違い | 問題点 | 修正方向 |
|-------|-------|---------|
| **各章が問いと無関係** | 論文の一貫性がなくなる | 「この章は問いのどの部分に答えるか」を常に確認 |
| **調べたことを羅列するだけ** | 論証がなく感想文レベルになる | 各節に「この情報が何を意味するか」の分析を加える |
| **先行研究の紹介で終わる** | 自分の立場・主張がない | 先行研究の「何が足りないか」「自分はどう発展させるか」を示す |
| **章のつながりが不明** | 読者が論文の論理を追えない | 各章の末尾に「次章では〜を検討する」等の橋渡しを入れる |

### 2.3 結論（おわりに）の間違い

| 間違い | 問題点 | 修正方向 |
|-------|-------|---------|
| **結論で新しい論点・データを持ち出す** | 本論で論証していない主張をしていることになる | 結論で述べることは本論の内容に限定する |
| **まとめを書くだけで主張がない** | 「本論文では〜について述べました」では結論にならない | 問いへの直接的な答えを一文で示す |
| **感謝・お断り・言い訳を書く** | 論文本文に不要な要素が混入する | 謝辞は別ページ、限界・課題は適切に但し書きで示す |
| **「今後の課題」で埋める** | 「今回はできませんでした」の列挙になる | 今後の課題は1〜2点に絞り、理由と方向性を示す |

## 3. 文章上のよくある間違い

### 3.1 主語と述語の不一致

主語と述語が対応していない「ねじれ文」は日本語学術文章で最も多い間違いの一つ。

**Before（ねじれ文）**:
```
「本研究の目的は、近年の少子化問題の要因について
 統計データをもとに分析した。」
（「目的は〜分析した」は主述がねじれている）
```

**After（修正済み）**:
```
「本研究の目的は、近年の少子化問題の要因を
 統計データをもとに分析することである。」
または:
「本研究は、近年の少子化問題の要因を
 統計データをもとに分析した。」
```

### 3.2 「の」の三連鎖以上

「〜の〜の〜の〜」という連鎖は修飾関係が曖昧になる。

| 修正前 | 問題 | 修正後 |
|-------|-----|-------|
| `日本語の学習者の語彙の習得` | 「誰の語彙か」「何の習得か」が不明確 | `日本語学習者が語彙を習得する過程` |
| `現政権のマスコミ批判` | 「現政権がマスコミを批判」か「マスコミが現政権を批判」か不明 | `現政権によるマスコミ批判` または `マスコミによる現政権批判` |
| `国内経済の低迷の原因の分析` | 「の」が3つ連続 | `国内経済が低迷した原因の分析` |

### 3.3 指示語の曖昧な使用

指示語（これ・それ・その・この）が何を指すか不明確な場合。

**Before（あいまい）**:
```
「Aは〜を主張した。Bはこれに反対した。これは、
 両者の立場の根本的な違いを示している。」
（後半の「これ」が何を指すか不明）
```

**After（明確）**:
```
「Aは〜を主張した。Bはこれに反対した。このA・B間の
 対立は、両者の立場の根本的な違いを示している。」
```

**指示語の使い分け原則**:
- 直前の文・名詞を指す → `この〜` / `その〜`
- 遠い内容や既知の事柄を指す → `当該〜` / 名詞を繰り返す
- 曖昧になりそうな場合 → 名詞を明示する

### 3.4 長すぎる一文

一文が長すぎると主語・述語の対応が崩れ、読み手の理解を妨げる。

**目安**: 日本語論文では1文50〜80字程度。80字を超えたら分割を検討。

**Before（長すぎる一文）**:
```
「この論文で分析した、国内の主要企業20社を対象とした
 2020〜2022年のアンケート調査の結果から、リモートワーク
 導入後の生産性変化について、職種別・年齢別の比較検討を
 行った結果、管理職においては生産性が低下する傾向がある
 一方で、専門職においては向上する傾向が見られた。」
```

**After（適切な長さに分割）**:
```
「本論文は、国内の主要企業20社を対象として2020〜2022年に
 アンケート調査を実施した。この調査から、リモートワーク
 導入後の生産性変化を職種別・年齢別に比較した。結果として、
 管理職では生産性が低下する傾向がある一方、専門職では
 向上する傾向が明らかになった。」
```

### 3.5 受動態の過剰使用

受動態の多用は文章をわかりにくく、責任の所在を曖昧にする。

| 修正前（過剰な受動態） | 修正後（能動態） |
|-------------------|--------------|
| `〜が分析された` | `〜を分析した` / `著者は〜を分析した` |
| `インタビューが行われた` | `研究者はインタビューを実施した` |
| `以下のことが明らかにされた` | `以下のことが明らかになった` |

**ただし**: 行為者が不明・不要な場合は受動態でよい。
- 「〜と定義されている」（定義の発信者を問わない場合）
- 「データは国土交通省から取得された」（行為者が自明の場合）

## 4. 書式上のよくある間違い

### 4.1 表紙・基本情報

| 確認項目 | よくある間違い |
|---------|-------------|
| 所属・学部・学年 | 学部名を略称のみにする |
| 学籍番号 | 番号の桁数ミス |
| 氏名 | 名前の誤字（自分の名前の漢字ミス） |
| タイトル | 論文の内容と一致していない |
| 提出日 | 期日ではなく実際の提出日 |
| ページ番号 | 表紙に番号をつけてしまう |

### 4.2 参考文献リストの不備

これは最もよくある書式上の失点。

**よくある不備**:
- 本文で引用したが参考文献リストに掲載していない
- 参考文献リストにあるが本文で引用していない
- 著者・タイトル・出版社・年のうちどれかが欠けている
- 直接引用なのにページ数が記されていない
- 文献の並び順が不統一（五十音順・アルファベット順が混在）
- ウェブサイトのURLのみで著者・タイトルが不明

**参考文献記載の基本形式（日本語）**:
```
著者名（出版年）「論文・章タイトル」『書名/雑誌名』巻・号, ページ数, 出版社.

例:
小林一郎（2020）「日本語論文における接続詞の使用傾向」
  『日本語研究』45(2), pp. 123-145, 日本語研究学会.

田中花子（2022）『論文の書き方入門』東京: 学習出版社.
```

### 4.3 章・節番号の管理

| よくある間違い | 確認方法 |
|-------------|---------|
| 第2章が2つある（1→2→2→3） | 目次と本文を照合 |
| 図表の番号が飛んでいる（図1→図3） | 図表一覧を作成して確認 |
| 脚注番号の連続性が失われている | 本文全体を通して確認 |
| 目次と本文の章タイトルが不一致 | 最後に再度照合 |

### 4.4 フォント・余白の不統一

| 問題 | 対処 |
|-----|------|
| 本文中のフォントがコピペで変わっている | 全選択→書式リセット→再統一 |
| 引用部分のフォントが異なる | 引用の書式は統一（インデントか引用符で区別） |
| 章タイトルのサイズが不統一 | スタイルシートを設定する |
| 行間が部分的に異なる | 段落設定を確認 |

## 5. AI時代の新しい注意点

### 5.1 生成AIの出力をそのまま使うリスク

ChatGPT・Claude・Gemini等の生成AIは高品質な文章を生成できるが、論文でそのまま使用することは複数の問題を引き起こす。

| リスク | 内容 |
|-------|------|
| **ハルシネーション** | 存在しない文献・統計・人物を「確信を持って」生成する。引用文献を必ず原典で確認する |
| **最新情報の欠如** | 学習データに時間的な限界がある。2022年以降のデータは不完全 |
| **剽窃とみなされるリスク** | 多くの大学がAI生成文章の無断使用を不正行為と定めつつある |
| **自分の理解の欠如** | AIが書いた論文は自分で理解していないため、口頭試問で答えられない |
| **平均的な文章** | AIは「それっぽい文章」を生成するが、独自の視点・分析はない |

### 5.2 AIを適切に使う指針

| 使い方 | 適切度 | 注意点 |
|-------|-------|-------|
| 構成案のブレインストーミング | ○ | あくまで出発点として使う |
| 文法・表現の確認 | ○ | 内容の正確性は自分で判断する |
| 要約の確認 | △ | 重要な箇所は原典を必ず確認 |
| 引用文献の検索 | △ | AIが示す文献は必ず実在を確認する |
| 論文本文の生成 | ✕ | 学術的誠実性の問題 |
| データ・統計の生成 | ✕ | 事実誤認・ハルシネーションのリスク大 |

### 5.3 剽窃（盗作）に関する重要注意

剽窃はAIの普及以前から大学の不正行為として厳しく対処される。

**大学が剽窃を検出する方法**:
- iThenticate 等の剽窃検出ソフトウェア（多くの大学が導入済み）
- 教員が「学生の書きそうな言い回しかどうか」を直感的に感知
- 複数の学生が同じウェブページ・AI出力を使うと重複が発見される

**剽窃を避けるための原則**:
1. 他者のアイデアを使う時は必ず出典を示す（パラフレーズした場合も）
2. 直接引用は引用符をつけ、出典とページ数を明示する
3. AI生成文章を自分の文章として提出しない（大学の規定を確認）

## 6. 提出前最終チェックリスト（30項目）

### A. 全体構成（6項目）
- [ ] 序論に問いが明示されているか
- [ ] 結論で問いに正面から答えているか
- [ ] 各章が問いへの答えに貢献しているか
- [ ] 結論で新しい論点・データを持ち出していないか
- [ ] 章・節の番号に欠番・重複がないか
- [ ] 目次と本文の章タイトルが一致しているか

### B. 論証の質（6項目）
- [ ] すべての主張に根拠（データ・文献・論理）がついているか
- [ ] 「〜と思う」「〜ではないか」を排除したか
- [ ] 「たくさん」「すごく」「とても」等の曖昧表現を修正したか
- [ ] 根拠から結論が論理的に導けるか確認したか
- [ ] 想定される反論への対処が含まれているか
- [ ] 先行研究との関係（一致・相違・発展）が示されているか

### C. 引用・参考文献（5項目）
- [ ] すべての引用に出典が示されているか
- [ ] 本文引用と参考文献リストが一致しているか
- [ ] 参考文献リストの形式が統一されているか
- [ ] 直接引用にページ番号が記されているか
- [ ] ウェブサイト引用にアクセス日が記されているか

### D. 文章表現（7項目）
- [ ] 主語と述語が対応しているか（ねじれ文がないか）
- [ ] 「の」が3つ以上連続している箇所がないか
- [ ] 指示語（これ・それ・その）が何を指すか明確か
- [ ] 1文が80字を超えている箇所を分割したか
- [ ] 「ここで〜を見てみる」「では〜について」等のNG表現を排除したか
- [ ] 「倫理感・価値感」等の誤字がないか
- [ ] 「何か〜かどうか」等の疑問詞と「かどうか」の二重使用がないか

### E. 書式（6項目）
- [ ] 表紙に所属・学籍番号・氏名・タイトル・日付があるか
- [ ] ページ番号が付いているか（表紙は除く）
- [ ] フォント・余白が統一されているか
- [ ] 図表に番号とキャプションが付いているか
- [ ] 誤字・脱字がないか（声に出して読んで確認）
- [ ] 提出形式（電子ファイル/印刷物、ファイル名のルール）に従っているか

## 付録: よくある間違い パターン早見表

```
Q: 「この分野についてたくさんの研究がある」
A: 具体的な研究名・文献を挙げる。「多数の研究がある（Smith 2019; Jones 2021等）」

Q: 「〜について検証する」
A: 仮説を実験・調査で確かめる意味でない限り「分析する」「考察する」

Q: 「非常に」「とても」「すごく」
A: 削除するか、程度を具体的に示す（「平均を20%上回る」等）

Q: 「〜と思います」
A: 根拠を示した上で「〜と考えられる」「〜と言える」

Q: 「では、〜について見ていきます」
A: 「以下では、〜を検討する」（前の論点との論理的な接続を示す）

Q: 「この問題は難しい」
A: 「この問題が複雑な理由は〜にある。具体的には〜」

Q: 参考文献が「URL」のみ
A: 著者名・サイト名・公開日・アクセス日を記録

Q: 「調べてみたら〜だとわかった」
A: 「〜が明らかになった」（プロセスではなく結果を述べる）

Q: ChatGPTの出力をそのまま使った
A: 引用文献を原典確認、内容を自分で検証・再執筆
```

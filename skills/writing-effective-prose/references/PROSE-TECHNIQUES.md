## 1. 「書く」ことは「考える」こと

### 1.1 書くことで思考が明確になる

「論理が整理されてから書く」のではなく、「書くことで論理が整理される」——これが文章修業の核心。

資料を読み、ノートをとり、たくさんの情報を集めたとしても、それは「材料」に過ぎない。論文という「機械」を組み立てるためには、材料を取捨選択し、並べ直す作業が必要だ。そして、その取捨選択・並べ直しは、実際に書き始めてみないとできない。

> 「書くことは考えること、考えを明確にすること。書き進むうちに、空で考えたときの論理のアラが見えてくる。」

この認識が大切。書けない・詰まる、というのは考えが整理されていない証拠。詰まったら、手を止めて考えるのではなく、「なぜ詰まるか」を紙に書き出してみる。

### 1.2 「捨てる」技術

良い文章は「書き足す」より「削る」ことで磨かれる。

収集した情報の大半は最終的に使わない。使わない情報を抱えたまま書こうとするから行き詰まる。「捨てきった後に残るもの」こそが書くべきことだという認識が大切。

```
情報収集段階: たくさん集める（量優先）
整理段階    : 問いに直結するものだけを残す（質優先）
執筆段階    : 残したものを論理的に配列する
```

## 2. 要約の技術

### 2.1 要約とは何か

要約は「短くまとめること」ではなく、**文献の核心的な主張を自分の理解で再表現すること**。

| 悪い要約 | 良い要約 |
|---------|---------|
| 本文の文章を短く切り貼り | 著者の主張を自分の言葉で再構成 |
| 重要な部分が不明確 | 「著者は〜を主張している」が明確 |
| 情報の羅列 | 主張の論理関係が示されている |

### 2.2 要約の手順

| ステップ | 作業内容 |
|---------|---------|
| **1. 全体を読む** | 飛ばし読みして全体像をつかむ |
| **2. 主張を探す** | 著者が最も言いたいことは何かを特定する |
| **3. 論拠を確認** | 主張を支える主な根拠を3点以内で把握する |
| **4. 構造を図示** | 「主張→根拠1・根拠2・根拠3」を箇条書きにする |
| **5. 自分の言葉で書く** | 本を閉じて、ノートに再表現する |
| **6. 照合する** | 原文と照合し、意味が変わっていないか確認する |

### 2.3 要約の長さの目安

| 元の文章の長さ | 要約の目安 |
|------------|---------|
| 1冊の本（200〜300ページ） | 400〜800字（原稿用紙1〜2枚） |
| 論文（20〜50ページ） | 200〜400字 |
| 論文の1章（5〜10ページ） | 100〜200字 |
| 1パラグラフ | 1〜2文 |

### 2.4 要約のBefore/After

**Before（悪い要約）**:
```
木下本は、日本語の文章における段落の重要性について述べており、
段落は文章の単位であって、一つの段落には一つの主題があるべきで、
それによって文章が読みやすくなると書かれている。また、接続詞の
使い方についても論じており、様々な接続詞の用法を説明している。
```

**After（良い要約）**:
```
木下（1981）は、日本語文章の論理的構成において、欧米の
パラグラフ・ライティングの概念を導入することを提唱している。
各段落に一つの主題文を置き、接続詞で段落間の論理関係を
明示することが、読みやすい論文文章の基本だと主張する。
```

## 3. パラフレーズの技法

### 3.1 パラフレーズとは何か・何のためにするか

パラフレーズとは、**他者の考えを自分の言葉に書き換えること**。

レポート・論文では直接引用より積極的に使うべき技法。理由:
- 自分が本当に理解しているかが試される
- 文章の流れが自然になる
- 自分の分析・解釈を加えやすくなる

### 3.2 パラフレーズの方法

| 方法 | 説明 | 例 |
|-----|------|---|
| **語彙の置き換え** | 同義語・類似表現に換える | 「研究する」→「調査する」「考察する」 |
| **構文の変換** | 能動態↔受動態、名詞化↔動詞化 | 「AはBを促進した」→「BはAによって促進された」 |
| **順序の変更** | 情報の提示順を変える | 結論→根拠を、根拠→結論に変える |
| **視点の転換** | 別の切り口で同じ内容を表す | 「減少した」→「低下した」「縮小した」 |

### 3.3 パラフレーズのBefore/After

**原文**:
```
「論文とは、明確な問いを立て、それに対する答えを主張し、
その主張を論理的に証明する文章である。」
```

**パラフレーズ例A（語彙の置き換え）**:
```
学術的な論文は、具体的な問題を設定し、それへの回答を提示して、
その回答の正当性を論拠をもって示す文章形式である。
```

**パラフレーズ例B（視点の転換）**:
```
問いを持たない文章や、答えを示さない文章、あるいは答えを
根拠なしに主張する文章は、いずれも論文とは呼べない。
```

### 3.4 パラフレーズ後も出典表示は必須

パラフレーズしたとしても、他者のアイデアを使っている場合は出典を示す義務がある。

```
正しい形式:
木下（1981）によれば、論文は問い・主張・論証の三要素から成る。

または:
論文は問い・主張・論証の三要素から成る（木下, 1981）。
```

## 4. 接続表現の体系的な使い方

### 4.1 接続表現の役割

接続表現には二つの重要な役割がある:

1. **論理を示す**: 文と文のつながり方（原因→結果、対比、補足など）を明示する
2. **次を予測させる**: 読者が次の文の「向き（ポジティブかネガティブか）」を予測できるようにする

接続表現が適切でないと、読者は文章の論理を取り違える。

### 4.2 接続表現の分類と代表例

#### 順接（前の文が根拠・原因で、後の文が結論・結果）

| 表現 | 使い方のポイント |
|-----|--------------|
| したがって | 論理的な帰結。ほぼ常に使える標準形 |
| ゆえに | 「したがって」よりやや格式が高い |
| そのため | 原因→結果の関係を示す |
| だから | 口語的。レポートでは「したがって」を推奨 |
| よって | 「したがって」と同義。法律・公文書的なニュアンス |
| その結果 | 行為や出来事の後に続く結果を示す |

#### 逆接（前の文と後の文の主張の向きが逆、後の文に重みがある）

| 表現 | 使い方のポイント |
|-----|--------------|
| しかし | 最も汎用的な逆接。後の文が重要 |
| ところが | 予想に反する展開を示す。しかしより驚きが強い |
| にもかかわらず | 前の条件がありながら、それに反することを示す |
| しかしながら | 「しかし」のやや格式高い形 |
| 〜が / 〜けれども | 文末につけて逆接を示す。多義的なので多用注意 |

**注意**: 「〜が」は逆接以外（補足・付加）にも使われる多義的な表現。「〜が」が多い文章は論理が不明確になりやすい。

#### 補足（前の文と後の文の主張の向きが逆、前の文に重みがある）

| 表現 | 使い方のポイント |
|-----|--------------|
| ただし | 前の主張に例外・限定条件を付ける |
| もっとも | 前の主張を認めた上で留保を加える |
| なお | 前の内容に補足情報を加える |

**逆接と補足の違い**:
```
逆接（後の文が重要）:
「この政策は経済成長を促進した。しかし、環境への影響は深刻だった。」
→ 環境への影響が主張のポイント

補足（前の文が重要）:
「この政策は経済成長を促進した。ただし、一部の地域では効果が限定的だった。」
→ 経済成長の促進が主張のポイント、地域差は限定的な補足
```

#### 付加（同じ方向の主張を並べる）

| 表現 | 使い方のポイント |
|-----|--------------|
| また | 同等の情報を追加する。並列関係を示す |
| さらに | 前の内容を強化・発展させる情報を追加 |
| しかも | 前の主張を強める内容を追加（「に加えて」「その上」） |
| 加えて | やや格式高い「また」。論文向き |
| そして | 時間的な継起または付加。「だから」と混同注意 |

**「そして」と「だから」の違い**:
```
「気温が下がった。そして、積雪量が増えた。」
（前後が並列・継起、因果関係の有無は不明）

「気温が下がった。だから、積雪量が増えた。」
（前が原因、後が結果という因果関係を主張）
```

#### 換言（同じ内容を別の言い方で表す）

| 表現 | 使い方のポイント |
|-----|--------------|
| つまり | 前の内容を簡潔にまとめて言い換える |
| すなわち | 「つまり」のやや格式高い形 |
| 言い換えれば | 読者にわかりやすい表現に換える |
| 要するに | 複数の内容をまとめて核心を述べる |

#### 例示（前の主張の具体例を示す）

| 表現 | 使い方のポイント |
|-----|--------------|
| たとえば | 最も汎用的な例示 |
| 具体的には | 例示というより、具体的な詳細を展開するとき |
| 〜など | 後ろにつけて例を挙げる |

#### 対比（二つの事柄の違いを並べる）

| 表現 | 使い方のポイント |
|-----|--------------|
| 一方 / 一方で | 二つの事柄を並べて対照させる |
| これに対して | 前述の内容と異なる立場・状況を示す |
| 他方 | 「一方」と同義 |

### 4.3 接続表現の適切な使用密度

論文では接続表現が多すぎても少なすぎてもよくない。

- **多すぎる**: くどい印象、論理が逆に見えにくくなる
- **少なすぎる**: 読者が論理をつなぎ直す負担を強いられる

**目安**: 2〜3文に1回程度。特に重要な論理の転換点（逆接・順接）には必ず入れる。

### 4.4 接続表現を使わなくてよい場合

- 前の文の内容が明らかに次の文の根拠になっている
- 時系列の流れが明確な記述
- 例示が自明の場合（「たとえば」なしでも例とわかる場合）

## 5. 推敲の方法

### 5.1 推敲の三段階

| 段階 | 焦点 | 方法 |
|-----|------|------|
| **マクロ推敲** | 全体の構成・論理の流れ | 目次を見直す、各章の要旨を書き出す |
| **ミクロ推敲** | パラグラフ・文章レベル | 各パラグラフの主張が明確か確認 |
| **校正** | 表記・書式 | 誤字・脱字・書式の統一 |

### 5.2 声に出して読む

「読み上げ法」は最も効果的な推敲技法の一つ。

- **なぜ有効か**: 目で読むと脳が「補完」するため、本当に書いてある文を読んでいない。声に出すと実際に書いてある文を一字一句確認できる。
- 読んでいて「詰まる」部分は、論理的におかしいか文章が複雑すぎる
- 一文が長すぎて息が続かない場合は分割を検討する

### 5.3 時間を置いてから読む

書いた直後は「脳内に原稿が残っている」ため、紙に書いてある文と記憶の中の文を区別できない。少なくとも1日、できれば3日以上置いてから読み直す。

時間を置くことで:
- 「自分が書いた文」ではなく「他人が書いた文」として読める
- 文脈の補完なしに読めるようになり、不明確な部分が浮かび上がる
- 構成上の問題（重複、欠落、順序の不適切）に気づきやすい

### 5.4 他者に読んでもらう

書き手にとって「当然の前提」が、読み手には伝わっていないことが多い。

**効果的な他者レビューの依頼方法**:
```
「この文章を読んで、以下を教えてください:
  1. 何がこの論文の問いだと思いましたか？
  2. 論旨はわかりましたか？わからなかった部分はどこですか？
  3. 違和感を覚えた表現・箇所はありましたか？」
```

「全体的な感想」を聞くより、具体的な質問をした方が有益なフィードバックが得られる。

### 5.5 パラグラフの最初の文だけを読む

各パラグラフの最初の文（トピックセンテンス）だけを順番に読んでいく。論文の論理の骨格が見えてくる。

```
チェックポイント:
- 各パラグラフのトピックセンテンスが問いへの答えに向けて
  一本の線を作っているか
- 話題が突然変わっている箇所はないか
- 結論に至る道筋が見えるか
```

## 6. 短文修業から始める文章力の鍛え方

### 6.1 短文修業の思想

優れた文章家の多くは、長い文章を「短文の積み重ね」と捉えている。

> 「短い文章は長い文章の絶対の前提。短文が長文の基礎・要素になっている。短文という部分品を作っておいて、それを組み立てることによって長い文章という機械を作り上げる。」

一気に長い文章を書こうとするのではなく、まず短い、完結した文を書けるようになることが先決。

### 6.2 短文修業の具体的な方法

#### ステップ1: 一文一意識
「一文には一つの主張だけを入れる」練習。

```
修正前（一文に複数の主張）:
「日本の少子化は深刻であり、政府はさまざまな対策を講じているが、
 その効果は限定的で、根本的な解決には至っておらず、今後も
 人口減少が続くと予測されている。」

修正後（一文一主張）:
「日本の少子化は深刻な状況にある。政府はさまざまな対策を講じているが、
 その効果は限定的だ。したがって、今後も人口減少が続くと予測される。」
```

#### ステップ2: 要約訓練（1000字→400字→100字→1文）
読んだ論文・書籍を段階的に圧縮する。この作業によって「何が最も重要か」を見極める力が養われる。

#### ステップ3: 接続詞なしで書いてから接続詞を加える
まず接続詞なしに主張だけを書き並べ、後から各文の関係を確認しながら接続詞を追加する。論理が先、接続詞は後。

### 6.3 書く習慣の作り方

| 方法 | 効果 |
|-----|------|
| **毎日100〜200字書く** | 文章との距離を縮める、「書く筋肉」を維持する |
| **読んだ本を1文で要約する** | 要約力・判断力の向上 |
| **自分の論文の問いを毎日1文で書く** | 問いが明確になっているかを確認できる |
| **「話す→書く」練習** | 話すように書くことで、書き出しの詰まりを解消 |

## 7. 文章を磨く具体的テクニック

### 7.1 主語と述語を近づける

主語と述語が離れると、読者が主語を記憶したまま読み続けなければならず疲れる。

```
修正前:
「この論文で採用した、先行研究を批判的に検討したうえで新たな
 分析枠組みを提示するというアプローチは、〜を明らかにした。」

修正後:
「本論文は批判的な先行研究検討と新たな分析枠組みの提示という
 アプローチを採用した。このアプローチによって、〜を明らかにした。」
```

### 7.2 「〜の〜の〜の〜」を避ける

「の」が3つ以上連続すると曖昧になる。

```
修正前:
「日本語の論文の評価の基準」（「の」3個）

修正後A:
「日本語論文の評価基準」（名詞を連結）

修正後B:
「日本語で書かれた論文を評価するための基準」（関係節で説明）
```

### 7.3 指示語（これ・それ・あれ）の明確化

指示語が何を指すか不明確な場合は、名詞に置き換えるか、再度名詞を繰り返す。

```
修正前:
「Aは〜を主張した。Bはこれに反論した。これは〜を示している。」
（最後の「これ」が何を指すか不明）

修正後:
「Aは〜を主張した。Bはこれに反論した。このA・B間の論争は、
 〜という問題の複雑さを示している。」
```

### 7.4 受動態・能動態の使い分け

| 使い分けの指針 | 例 |
|-------------|---|
| **行為者を強調したい**: 能動態 | 「田中（2020）は〜を実証した。」 |
| **行為の対象を前面に出したい**: 受動態 | 「〜は2020年代以降に急速に広まった。」 |
| **行為者が不明・重要でない**: 受動態 | 「〜と定義されている。」 |

論文では過度な受動態（「〜が行われた」「〜が示された」の連続）は能動態に変える方が明確で読みやすい。

### 7.5 一文の長さの目安

| 目安 | 説明 |
|-----|------|
| **日本語**: 50〜80字程度 | 読点1〜2個で読める長さ |
| **80字超**: 分割を検討 | 接続を使って2文に分ける |
| **120字超**: 必ず分割 | 主語と述語の対応が失われやすい |

**例外**: 並列関係を示す場合、比較を示す場合は長くなってもよい。長い文が必要な場合は「〜（A）。一方で、〜（B）。」のように接続でつなぐ。

### 7.6 「先行オーガナイザー」の活用

長い説明の前に「これから何を述べるか」を一文で示す。

```
先行オーガナイザーあり:
「以下では、三つの観点から〜を検討する。
 第一に〜。第二に〜。第三に〜。」

先行オーガナイザーなし:
「〜について述べる。まず〜。次に〜。そして〜。」
（読者はどこへ向かうのかわからない）
```

## 付録: 文体・表現の使い分け対照表

| 場面 | 推奨表現 | 避ける表現 |
|-----|---------|----------|
| 主張を示す | 「〜と言える」「〜と考えられる」「本稿は〜を主張する」 | 「〜と思う」「〜ではないだろうか」 |
| 数量を示す | 「多くの〜」「N件の〜」「約N%の〜」 | 「たくさんの〜」「すごく多い」 |

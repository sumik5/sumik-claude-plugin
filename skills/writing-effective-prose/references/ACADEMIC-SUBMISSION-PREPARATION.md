## 1. 提出前チェックリスト

### 1.1 内容確認

| チェック項目 | 確認内容 |
|-------------|----------|
| **課題への適合** | 設問・タスクに直接的・包括的・適切に答えているか |
| **関連性** | 内容がすべて関連性を持っているか |
| **根拠の提示** | 主張・アイデアに信頼できるソース・エビデンスを示しているか |
| **参照の完全性** | 参照文献が完全・正確・一貫しているか |
| **ガイドライン遵守** | 内容・参照・フォーマット・プレゼンテーションの全ガイドラインに従っているか |

### 1.2 ワードカウント

**ワードカウントが設定される理由**:
1. **公平性**: すべての学生に同じ制限を課す
2. **詳細度**: 十分な詳細や実質的な議論を可能にする
3. **レベル分け**: 学年に応じた期待値を設定
4. **選択能力**: 最重要トピックを選ぶ判断力を評価
5. **冗長性の排除**: 不要な繰り返しを避ける訓練
6. **簡潔性**: 言葉数を抑える能力を養う

**一般的な許容範囲**:
- ±10%（例: 2,500語課題 → 2,250～2,750語が許容範囲）

**推定方法**:
- A4用紙1ページ（標準フォント、ダブルスペース）≈ 250-300語
- 11ページ ≈ 2,750-3,300語

**注意**: 通常、参考文献リストはワードカウントに含まれない（事前確認推奨）

### 1.3 提出プロセスの確認

| 確認事項 | 詳細 |
|---------|------|
| **提出方法** | ハードコピー / オンライン / 両方 |
| **ハードコピー形式** | 部数、用紙サイズ（A4）、片面/両面印刷、製本方法 |
| **カバーシート** | 標準カバーシート必要か / 独自作成か |
| **提出場所** | 事務室 / 担当者オフィス等 |
| **オンライン手順** | VLE経由のアップロード方法を事前確認・練習 |
| **匿名性** | 氏名を除外し、学生番号のみ記載（通常必須） |
| **締切時刻** | 正確な締切（例: 4:00pmは4:01pmにはアウト） |

## 2. 編集と校正

### 2.1 最終編集 vs 校正の違い

| プロセス | 目的 | フォーカス |
|---------|------|-----------|
| **最終編集（Final Editing）** | 意味と流れの確認 | 内容・構成・意味・論理性 |
| **校正（Proofreading）** | 表面的エラーの修正 | スペリング・文法・句読点・タイポ・フォーマット |

**重要**: 両方のプロセスが必要。多くの学生は校正のみに集中しがちだが、最終編集も不可欠。

### 2.2 最終編集のベストプラクティス

| テクニック | 効果 |
|-----------|------|
| **音読** | 読者の立場に立ち、流れと自然さを確認 |
| **時間を空ける** | 読む → 休憩 → 再度読む（新鮮な視点で確認） |
| **複数回読み通す** | 意味・論理性・読みやすさを多角的に確認 |

### 2.3 用語とアクロニムの体系的使用

#### ルール
1. **初出時**: フルネーム + (アクロニム)
2. **以降**: アクロニムのみ使用

**例**:
> The 1988 Education Reform Act (ERA) threatened... Local Education Authorities (LEA) and teacher unions. The Assessment of Performance Unit (APU) from 1975 provided... LEAs often led the way in supporting schools' response to ERA.

#### 略語（Abbreviations）との違い
- アクロニム: 頭字語（SATs, KS, MFL）
- 略語: 単語の短縮形（approx, etc）

**注意**: 略語の末尾のピリオドは省略可能だが、一貫性を保つ

### 2.4 テキスト強調の体系的使用

| 強調方法 | 用途 | 例 |
|---------|------|-----|
| **太字（Bold）** | 重要用語の初出 | **homophone** such as *sew*, *so*, and *sow* |
| **イタリック（Italics）** | 例示語・外来語・専門用語 | the ending *-ee* in words such as *fiancée*, *divorcee* |
| **引用符（Quotation marks）** | 引用・特殊な用法 | 'nonsense' words, "Twas brillig..." |

**重要**: 一貫したシステムを維持する（シングル/ダブル引用符は引用専用に統一等）

### 2.5 校正（Proofreading）

#### 校正の範囲
- ✅ スペリング、文法、句読点の表面的エラー修正
- ✅ タイポ（typing errors）の発見
- ✅ フォーマットの不整合修正
- ❌ 意味や構成の変更（これは編集の範囲）

#### 校正のアプローチ

| 方法 | メリット | 注意点 |
|------|---------|--------|
| **セルフ校正** | 自分の傾向を把握し、改善できる | 過去のフィードバックをもとに重点分野を特定 |
| **スペルチェックツール** | 明白なミスを素早く発見 | コンピューターは文脈やニュアンスを理解できない。提案を慎重に評価 |
| **ピアレビュー** | 新鮮な視点で問題を発見 | 協働活動として有益。盗作（collusion）とは異なる |
| **プロ校正者** | 専門的視点での修正 | 表面的エラーのみ。意味や構成は変更しない |

#### 校正の倫理的境界

| 許容される | 許容されない |
|-----------|-------------|
| スペリング・文法・句読点の修正 | 構成や意味の変更 |
| タイポの指摘 | 学生の声やメッセージの歪曲 |
| 提案の提示（最終判断は学生） | 学生に代わって書く |

**原則**: 校正者は*提案*するのみ。受け入れ・却下の判断と最終責任は学生自身にある。

## 3. フォーマットとプレゼンテーション

### 3.1 行間隔（Line Spacing）

| 設定 | 用途 | 推奨度 |
|------|------|--------|
| **シングルスペース** | - | ❌ 読みにくい・コメント追加不可 |
| **1.5スペース** | 学術論文 | ✅ 推奨 |
| **ダブルスペース** | 学術論文 | ✅ 推奨 |

**理由**:
- 読みやすさの向上
- 評価者がコメント・修正を記入するスペースを確保

### 3.2 段落フォーマット

| 方法 | 説明 | 推奨度 |
|------|------|--------|
| **インデント（字下げ）なし、段落間の空白なし** | 段落境界が不明瞭 | ❌ |
| **段落間に1行空白** | 段落を明確に分離 | ✅ |
| **各段落の最初にインデント** | 段落の開始を明示 | ✅ |
| **空白 + インデント** | 段落境界を最も明確化 | ✅ 推奨 |

**目的**: 段落の結束性（1段落=1つの統一されたアイデア）を視覚的に強調

### 3.3 表と図のフォーマット

#### ラベリング規則
| 要素 | ラベル位置 | 例 |
|------|-----------|-----|
| **表（Table）** | 上部 | **Table 9.4: Schools' Involvement In Collaboration** |
| **図（Figure）** | 下部 | **Figure 2:** Percentage of schools in England... |

#### 参照方法
- As Table 1 shows/illustrates, ...
- As can be seen in Table 1, ...
- Figure 2 shows/illustrates ...
- The RDA increases according to age (Table 1).
- Waiting times continue to rise (Figure 2).

#### 出典表示
表・図を他のソースから引用・改変した場合は明確に参照を記載する。

## 4. タイトルページとヘッダー

### 4.1 タイトルページの必須要素

| 項目 | 内容 | 注意点 |
|------|------|--------|
| **学生番号** | Student ID Number | 匿名提出の場合は氏名を含めない |
| **所属機関** | 例: University of Towncaster School of Education | 所属を明記 |
| **課題タイトル** | Assignment title | 正確に記載 |
| **コース/モジュール情報** | コース名・モジュール名・開始日 | 詳細を明記 |
| **提出日** | 完成日または提出日 | ガイドラインに従う |
| **担当教員名** | 担当教員または学術アドバイザー名 | 必要に応じて記載 |

**注意**: 標準カバーシートが要求される場合でも、独自のタイトルページを追加可能な場合が多い。

### 4.2 ワードカウント表示

- 文書の末尾に明確に記載
- Microsoft Word: 画面左下にワードカウント表示

### 4.3 ページ番号

- ページ番号を必ず追加（印刷時のページ順序維持のため）
- Microsoft Word: 「挿入」ツールで追加
- 位置: ページ上部・下部・中央・左右コーナー等（一貫性を保つ）

### 4.4 フォント選択

#### 推奨フォント
- **Times New Roman**
- **Arial**

#### 避けるべきフォント
- ❌ **Comic Sans**（子供っぽく見える）

**理由**: 読みやすさと適切な「真面目さ」のバランス

### 4.5 フォントサイズ

| サイズ | 用途 | 評価 |
|--------|------|------|
| 10pt未満 | - | ❌ 小さすぎて読めない |
| **12pt** | 本文 | ✅ 理想的 |
| 14pt超 | - | ❌ 子供っぽく見える |
| 14pt以上 | 見出し・サブ見出し | ✅ 許容 |

### 4.6 印刷ガイドライン

| 項目 | 確認事項 |
|------|---------|
| **印刷面** | 両面 / 片面（ガイドライン確認） |
| **ポートフォリオ** | ハードコピーが必要な場合の準備 |

## まとめ: 提出前の最終チェックリスト

### 内容
- [ ] 課題に直接回答している
- [ ] すべての内容が関連性を持っている
- [ ] 主張に適切なエビデンスを提示している
- [ ] 参照文献が完全・正確・一貫している

### フォーマット
- [ ] ワードカウントが許容範囲内（±10%）
- [ ] 行間隔が適切（1.5またはダブルスペース）
- [ ] 段落が明確に分離されている
- [ ] 表・図が適切にラベル付け・参照されている

### プレゼンテーション
- [ ] タイトルページを作成
- [ ] ワードカウントを記載
- [ ] ページ番号を付与
- [ ] 適切なフォント・サイズを使用（12pt Times New Roman/Arial）
- [ ] 装飾を排除し、プロフェッショナルな外観を維持

### 編集・校正
- [ ] 最終編集（意味・流れ・論理性の確認）
- [ ] 校正（スペリング・文法・句読点・タイポの修正）
- [ ] 用語・アクロニムの体系的使用を確認
- [ ] テキスト強調（太字・イタリック・引用符）の一貫性を確認

### 提出
- [ ] 提出方法・形式を確認（オンライン/ハードコピー/両方）
- [ ] 匿名性要件を確認（氏名除外、学生番号記載）
- [ ] 締切時刻を正確に把握
- [ ] カバーシートを準備（必要な場合）

## 学術支援の活用

多くの大学が提供する支援:
- **アカデミックライティングクラス**: 文章作成・クリティカルシンキング等
- **ライティングコンサルテーション**: 専門家との1対1セッション（30分～1時間）
  - 既提出課題のフィードバック理解
  - 進行中の課題への助言

**注意**:
- 校正サービスではなく、形成的（formative）コンサルテーション
- 専門外の視点は、説明の明確性をテストする利点がある
- 提出前に時間的余裕を持って予約（数週間前推奨）

---

**最終原則**: プロフェッショナルで明確、ナビゲートしやすい文書を作成することが、内容の質を最大限に伝える鍵である。

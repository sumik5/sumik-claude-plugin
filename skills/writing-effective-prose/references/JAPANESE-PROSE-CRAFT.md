# 日本語作文の古典的原則

## 概要

本多勝一『〈新版〉日本語の作文技術』（朝日文庫）および
『〈新版〉実戦・日本語の作文技術』（朝日文庫）から抽出したリファレンス。

感覚論・印象批評ではなく構文論（統辞論）に基づく体系的原則を確立した「原典」。
「わかりやすい文章を書くこと」がただひとつの目標であり、作文は才能ではなく技術。

---

## 1. 修飾語の順序（最重要）

### 修飾語順の4原則

| 優先度 | 原則 | 内容 |
|--------|------|------|
| 1位 | 節を先に、句をあとに | 動詞・形容詞を含む節（クローズ）は、単純な句（フレーズ）より前に置く |
| 2位 | 長い修飾語ほど先に | 物理的な長さで順序を決める（ほぼ同等の重要性） |
| 3位 | 大状況・重要内容ほど先に | より広い文脈・重要度の高いものを前に |
| 4位 | 親和度（なじみ）の強弱 | 直後の語と「なじみ」の強い単語を引き離す |

### 原則1: 節を先に、句をあとに

```
❌ 白い厚手の横線の引かれた紙
                 ↑ 「横線が厚手」に見える

✅ 横線の引かれた白い厚手の紙
   ↑ 節（横線の引かれた）を先、句（白い・厚手の）をあとに
```

動詞修飾の例:
```
❌ 速くライトを消して止まらずに走る
   （「速く消す」と誤読される）

✅ ライトを消して止まらずに速く走る
   （節を先に、句「速く」をあとに）
```

### 原則2: 長い修飾語ほど先に

```
❌ 私はたぶん大雨になると思った。
（「私は」が短いのに前、「大雨になると」が長いのに後）

✅ たぶん大雨になるのではないかと私は思った。
（長い修飾語が先）
```

主述関係の神話を排せよ: 「主語と述語を近くせよ」という指導は誤り。
問題の本質は長短関係であって、主述関係ではない。

```
❌ 明日は大雨だとこの地方の自然に長くなじんできた私は思った。
   （「主語・述語が近い」が読みにくい）

✅ この地方の自然に長くなじんできた私は明日は大雨だと思った。
   （長い修飾語が先）
```

### 原則3: 大状況から小状況へ

```
❌ 豊かな潤いをもえる若葉に初夏の雨が与えた。
   （小状況→大状況の逆順）

✅ 初夏の雨がもえる若葉に豊かな潤いを与えた。
   （大状況「初夏の雨」→小状況「若葉」→細部「潤い」）
```

### 原則4: 親和度（なじみ）の強弱

なじみの強い単語を被修飾語の直前に置くと誤読が生じる。

```
❌ 初夏のみどりがもえる夕日に照り映えた。
   （「みどり」と「もえる」が強い親和性 → 「みどりがもえる」と誤読される）

✅ もえる夕日に初夏のみどりが照り映えた。
   （親和度の強い「もえる」を「みどり」から引き離す）
```

### Before/After 実例集

**実例1: 修飾語の直結の原則**

```
❌ 二日未明、三二人が飛び散ったガラスの破片などで一～二週間のけがをした。
   （「三二人が飛び散った」と誤読）

✅ 飛び散ったガラスの破片などで三二人が一～二週間のけがをした。
   （修飾語を被修飾語に直結）
```

**実例2: 長い順の適用**

```
❌ ＭＰＬＡ指導部の第一陣として一九七四年の暮れに、解放闘争初期からの指導者の一人
   ルシオ・ララにひきいられた代表団がルアンダの空港についたとき、

✅ 解放闘争初期からの指導者の一人ルシオ・ララにひきいられた代表団が
   ＭＰＬＡ指導部の第一陣として一九七四年の暮れに、ルアンダの空港についたとき、
   （長い修飾語を先に）
```

**実例3: 大状況優先**

```
❌ 肉をたっぷり食べているアルゼンチンのセニョリータにくらべると、
   チリ美人はぐっと小柄である。
   （「アルゼンチンの」が「肉」にかかる誤読）

✅ チリ美人は、アルゼンチンのセニョリータにくらべると、ぐっと小柄である。
   （修飾関係を明確にした上で語順整理）
```

### 技術文書・ブログへの応用

- 長い主語（チーム・プロジェクト名など）は最後に置くより前に置く
- 「〜という条件のもとで、〜の場合に、ユーザーが操作すると」の順序は大状況→小状況
- APIドキュメントの例: 「POST /v1/users エンドポイントを使って、認証済みのクライアントが」より
  「認証済みのクライアントが POST /v1/users エンドポイントを使って」の方が読みやすい

## 2. 句読点（テン）の原則

### テンの二大原則

| 原則 | 略称 | 内容 |
|------|------|------|
| 第一原則 | 長い修飾語の原則 | 長い修飾語が2つ以上あるとき、その境界にテンをうつ |
| 第二原則 | 逆順の原則 | 語順が（原則に反して）逆順になったときにテンをうつ |

これ以外の構文上「必要な」テンは存在しない。文部省の13か条の基準もこの2原則に吸収される。

### 第一原則: 長い修飾語の境界

n個の長い修飾語があるとき、(n-1)個のテンが必要。

```
❌ 何も事情を知らない軽薄きわまるＡが思っただけでもふるえるほど大嫌いなＢを
   私の小学校から高校を通じて親友のＣに紹介した。

✅ 何も事情を知らない軽薄きわまるＡが、思っただけでもふるえるほど大嫌いなＢを、
   私の小学校から高校を通じて親友のＣに紹介した。
   （A/B/Cの境界にテン）
```

重文のテンも同じ原則:
```
✅ 父も喜び、母も喜んだ。
   （「父も喜び」「母も喜んだ」の境界 = 第一原則）
```

### 第二原則: 逆順のテン

原則的語順（長い方を前）に反して短い修飾語を冒頭に出した場合、テンが必要。

```
❌ 渡辺刑事は血まみれになって逃げ出した賊を追いかけた。
   （「渡辺刑事は」が短いのに先頭 → テンなしでは誰が血まみれか不明）

✅ 渡辺刑事は、血まみれになって逃げ出した賊を追いかけた。
   （「渡辺刑事は」を逆順で先頭に出したのでテン必須）
```

題目語「○○は」を冒頭に置く文はほぼすべてこの第二原則に該当:
```
✅ 私は、明日はたぶん大雨になるのではないかと思った。
   （「私は」が逆順で冒頭にあるのでテン必要）

（テンが不要な形）
✅ 明日はたぶん大雨になるのではないかと私は思った。
```

### うってはならぬテン

**最も重要な禁止事項**: 連体形の語尾の直後にテンをうたない。

```
❌ わたしをつかまえて来て拷問にかけたときの連中の一人である、特高警察のミンが
   大声でいった。
   （「である」は連体形。テンをうつと文が終わったように見え、構文が崩壊する）

✅ わたしをつかまえて来て拷問にかけたときの連中の一人である特高警察のミンが
   大声でいった。
```

この禁止事項の理由:
1. 終止形と同じ語尾の連体形の後にテンをうつと、文が終わったように誤読される
2. かつ、修飾語の原則（節を先に）を逆用して「節を後に」の反則になる

### 思想のテン（自由なテン）

二大原則以外の自由なテン。筆者の強調・間合いを示す。

```
✅ 父は死んだ。（単なる事実の記述）
✅ 父は、死んだ。（「死んだ」ことへの筆者の感情・強調）
```

注意: テンが多すぎると効果が消える。全文の各単語後にテンを打つと、
逆にテンがないのと同じ状態になる。

### Before/After: テンの実例

**実例1: 誤ったテンが論理を壊す**

```
❌ 病名が心筋梗そくだと元気にまかせて、過労をかさねたのではないかと思い
   （「元気にまかせて」と「過労をかさねた」の間にテン → 誰が元気なのか不明）

✅ 病名が心筋梗そくだと、元気にまかせて過労をかさねたのではないかと思い
   （「心筋梗そくだと」と「元気にまかせて過労...」の境界にテン）
```

**実例2: 構文上のテンを優先する**

```
❌ 戦前からの業界の流れを知る幹部も、若手も今年の漁獲やかつての北洋について
   聞くと、うしろめたそうな顔になった。
   （「幹部も若手も」がひとくくりになり、若手も「戦前から」知っていることになる）

✅ 戦前からの業界の流れを知る幹部も、若手も、今年の漁獲やかつての北洋について
   聞くとうしろめたそうな顔になった。
   （「幹部も」「若手も」「今年の...」の3つの修飾語の境界を明確化）
```

### 技術文書・ブログへの応用

- 長い箇条書き内でもテンの原則は同じ
- 「設定ファイルを編集し、サーバーを再起動した後に、動作確認する」 → テン2か所は第一原則
- コードコメントでも「短い修飾語を逆順に先頭に出したらテンを打つ」
- テンを打つ理由が説明できないテンは打たない

## 3. 助詞の使い方

### 「は」の二大機能

| 機能 | 名称 | 意味 |
|------|------|------|
| 題目のハ | 提題・主題 | 文の話題・テーマを示す（格助詞ガ・ノを兼務） |
| 対照のハ | 限定・対照 | 「〜については」「他と区別して」の意味 |

**題目のハ**: 「象は鼻が長い」の「は」
- 「象の鼻が長い」という事実から「象」を話題として取り出す
- 「象が」ではなく「象は」にすることで、他の話題（キリン・ゾウ）と対比も示す
- 「甲が乙に丙を紹介した」から題目を取り出すと:
  - 「甲は乙に丙を紹介した」（甲を題目に）
  - 「乙には甲が丙を紹介した」（乙を題目に）
  - 「丙を、甲は乙に紹介した」（丙を題目に）

**対照のハ**: 「蛙は腹にはへそがない」の二つの「は」
- 最初の「は」= 題目（カエルについて話す）
- 「腹には」の「は」= 対照・限定（腹に「は」ない、他の部分は別）

### 「が」と「は」の使い分け

| 助詞 | 用法 | 例 |
|------|------|-----|
| が | 焦点・主格（新情報） | 「太郎が来た」（誰が来たか） |
| は | 題目（既知情報・話題） | 「太郎は来た」（太郎についていえば来た） |

注意: 文章での「は」と「が」の選択は、何を「文の題目」とするかの問題であり、
「主語」という概念とは別次元。

### 接続助詞「が」の問題

逆接の「が」と、単なる接続の「が」を混用すると読者を惑わす。

```
❌ 昨日会議があったが、結論は出なかったが、次回に持ち越しになった。
   （どこが逆接か不明）

✅ 昨日会議があったが、結論は出なかった。次回に持ち越しになる。
   （逆接の「が」を1か所に限定）
```

「が」の安易な使用は読者の思考の流れを一瞬乱す（逆接かと身構えさせる）。
接続として「が」を使う場合は、逆接と紛れない文脈に限定する。

### 「の」の連続

所有格「の」が3つ以上連続すると読みにくくなる。

```
❌ 東京大学理学部化学科の教授の論文の内容
✅ 東京大学理学部化学科・田中教授の論文が述べた内容
```

### 並列の助詞: ナカテンとテンの使い分け

- 短い単語の並列 → ナカテン（・）: `松・杉・ヒノキ・ケヤキ`（テンを使うと構文上のテンと混乱）
- 長い修飾語つきの並列 → テン: `『資本論』を書いたカール・マルクス、『国富論』を書いたアダム・スミス`

## 4. 漢字とカナのバランス

### 基本原則: 漢字カナまじりは「わかち書き」の代替

日本語の「漢字+カナ」の組み合わせはローマ字の「分かち書き」と同じ役割を果たす。
視覚的に意味の固まり（単語）を絵画的に認識させる。

```
❌ げんだいのにっぽんぶんは  （カナばかりで一字ずつバラバラに拾い読みになる）
❌ 大体平仮名其混具合        （漢字ばかりで区切りが不明）
✅ 現代の日本文は            （漢字とカナで視覚的まとまりができる）
```

### 漢字/カナの選択基準

**一律の統一はNG**: 前後の文脈によって同じ語でも漢字かカナかを変える。
「今」は常に漢字・常にひらがなと統一するのは誤り。文脈ごとに視覚的バランスを考える。

### カナのみが続く場合の対処

わかち書き目的のテンは打たない。代わりに:
漢字を使う → カタカナ変換 → 傍点 → 本当のわかち書き（半角スペース）の順で検討。

「こん虫」式（一部だけカナ）は最悪。`昆虫（こんちゅう）` か `昆虫` とする。

## 5. 欠陥文パターンと修正法

### パターン1: 修飾・被修飾の離れすぎ

被修飾語（かかる先）から修飾語が離れると、どちらにかかるか不明になる。

```
❌ 渡辺刑事は血まみれになって逃げ出した賊を追いかけた。
   （誰が血まみれか不明）

修正の方向:
1. 直結させる → 修飾語を被修飾語の直前に移動
2. 語順を変える → 長い修飾語を先に

✅ 血まみれになって逃げ出した賊を、渡辺刑事は追いかけた。
```

### パターン2: 入れ子文（ネスト構文）

修飾語がさらに別の修飾語を含む入れ子になると、極端に読みにくくなる。

```
❌ 愛国武装諸勢力と人民の連続的勝利は、愛国武装諸勢力の兵士多数を殺害してこれを
   せん滅し……みじめな失敗に終わったことをしめしている。

修正の方向:
- 入れ子をはずして、主語を述語の直前に移動する
✅ ……みじめな失敗に終わったことを、愛国武装諸勢力と人民の連続的勝利はしめしている。
```

### パターン3: 底抜け文（かかる先のない修飾語）

「〜は」「〜ということは」という題目を提示したのに、それを受ける述語がない。

```
❌ ここで重要なのは、非単系の社会に血縁集団が存在するばあい……必要だと思われる。
   （「重要なのは」が「必要だと思われる」で終わっているが、「重要な点は何か」が示されていない）

修正:
1. 底をつける → 「必要だと思われる点がここで重要だ」
2. 入れ子を外す → 「ここで重要なのは次の点である。すなわち……」
```

### パターン4: 否定の誤った修飾

「〜ない」「〜しない」などの否定語に何がかかるかを誤ると、意味が逆転する。

```
❌ 肩書きだけで一生暮らせない
   （「一生」が「暮らせない」全体にかかり、「少しも暮らせない」になる）

意図: 「肩書きだけの一生（=生き方）は続かない」

✅ 肩書きだけの一生は続かない  または
✅ 一生、肩書きだけで暮らすことはできない
```

### パターン5: 紋切型（手垢のついた表現）

以下の表現は避ける:

| 紋切型 | 問題点 |
|--------|--------|
| 「唇を噛んだ」 | 実際の取材確認なしに使う |
| 「ガックリ肩を落とした」 | 事実か不明な慣用表現 |
| 「このおばさん、ただのおばさんではない」 | 文自体が笑っている |
| 「……昨今である」「……今日このごろ」 | 陳腐な締め |
| 「どうしてどうして」「そんじょそこらの」 | 低劣な口語 |
| 「複雑な表情」 | 何も描写していない |

**原則**: 紋切型は思考を停止させる。その言葉が本当の事実を示しているか問え。

### パターン6: 不必要な繰り返し

```
❌ 起きて、顔を洗って、そして歯を磨いて、そして御飯を食べて、そして……
❌ 〜と思う。〜と思う。〜と思う。（文末の「と思う」の連続）
❌ 〜だった。〜だった。〜だった。（文末の過去形の連続）
```

対策:
- 接続表現を変える: 「しかし・けれども・ところが・だが・が・にもかかわらず」を使い分ける
- 文末表現を変える: 「〜した」「〜である」「〜に至った」「〜といえる」を混用

### パターン7: 体言止めの多用

```
❌ 満開の桜。遠くから眺める都市の風景。そして、静かに流れる川。

問題: 体言止めの多用は単調で「下品」。
特にルポルタージュ（事実を伝える文章）での過去形も同様に注意。
```

### 修正の手順（実戦編より）

欠陥文を直す際の基本手順:

1. **なぜ欠陥文か を特定する**（修飾関係? テン? 助詞? 底抜け?）
2. **必要最小限の変更**で直す（原文を解体しない）
3. **直した後、別の欠陥が生じていないか**確認する

## 6. 実戦チェックリスト

### 文を書いた直後のセルフチェック

```
修飾語の順序:
[ ] 節（動詞・形容詞を含む固まり）は句より前に置いているか
[ ] 長い修飾語ほど前に置いているか
[ ] 大きな状況・重要な内容ほど前に置いているか
[ ] なじみの強い単語を被修飾語の直前に置いていないか

テン（読点）:
[ ] 長い修飾語が2つ以上ある場合、その境界にテンを打っているか（第一原則）
[ ] 短い修飾語を逆順で先頭に出した場合、テンを打っているか（第二原則）
[ ] 連体形の語尾（〜である・〜した・〜な）の直後にテンを打っていないか
[ ] 理由のないテン（単なる口調・わかち書き目的）を打っていないか

助詞:
[ ] 「が」と「は」の機能を明確に使い分けているか
[ ] 「が」を逆接でも単純接続でも無差別に使っていないか
[ ] 「の」が3つ以上連続していないか

漢字・カナ:
[ ] 前後の文脈に応じて漢字/カナを使い分けているか（一律統一はしない）
[ ] カナばかり・漢字ばかりが続く箇所はないか
[ ] 「こん虫」式の漢字分割をしていないか

表現:
[ ] 紋切型の表現を使っていないか
[ ] 同じ文末表現・接続詞の繰り返しはないか
[ ] 体言止めを多用していないか
```

### 文全体のレビュー基準

- **読みながら声に出せるか**: テンのところで自然に息がつけるか
- **一読で理解できるか**: 読み返しが必要なら構造に問題あり
- **修飾語のかかり先が明確か**: 「誰が」「何を」が曖昧なら修正
- **否定の対象が明確か**: 「〜ない」が何を否定しているか確認

### 技術文書・ブログ執筆への応用ポイント

| 状況 | 本多勝一の原則の応用 |
|------|---------------------|
| 長い文章をリファクタリングする | 入れ子構文を外し、修飾語を直結させる |
| 技術仕様を説明する | 大状況（背景・目的）→小状況（詳細）の順で配置 |
| エラーメッセージを書く | 主語（何が）→述語（どうした）を最短距離で結ぶ |
| 手順書を書く | 各ステップの修飾語の順序を「長い方から先に」に統一 |
| レビューコメントを書く | 「〜な問題がある」より「〜が〜なので問題」と具体的修飾語を先に |

## 1. 大学教員は何を見ているか

### 1.1 教員の評価の本音

大学教員は毎年何百本もの論文・レポートを採点している。読み手として何を求めているかを理解すると、自然に高い評価に近づく。

**教員が読んで気持ちよい論文の特徴**:
- 冒頭を読めば「何を問い、何を主張するのか」がすぐわかる
- 読み進めながら「論理の流れ」が予測できる
- 終わりまで読んで「問いに正面から答えている」と感じる

**教員が辛いと感じる論文の特徴**:
- 何が問いかわからず、どこへ連れていかれるかわからない
- 読み終わっても「結局何が言いたいのか」がわからない
- 引用が正確でなく、検証できない

### 1.2 「論文の評価のほとんどは論証で決まる」

論文を書く者が知っておくべき最重要の事実:

> **結論の「正しさ」は評価の主要因ではない。論証が正しくなされているかどうかが重要だ。**

担当教員が熱心な捕鯨反対論者だとしても、「イルカが可哀想だから捕鯨に反対です」という論文より、「捕鯨は条件付きで存続すべきだ。なぜなら……」と論証した論文の方が高く評価される。自分と違う結論でも、論証がきちんとしていれば評価する——それが学術的評価の原則。

## 2. 評価基準の体系

以下は標準的な大学論文の評価基準（★★★＝最低条件、★★＝良い論文の条件、★＝卒論等の大論文のみ適用）。

### 2.1 評価以前の常識（★★★：必須）

| 項目 | チェック内容 |
|-----|------------|
| **剽窃** | 他人の文章・アイデアを無断で使用していないか |
| **書式** | 指定の形式（ホチキス綴じ、余白、フォント等）に従っているか |
| **提出情報** | 所属・学年・氏名・学籍番号が正確に記載されているか |
| **課題対応** | 与えられた課題・テーマに答えているか |
| **制限字数** | 指定の字数・分量を守っているか（±10%が目安） |

### 2.2 内容の評価基準

#### 2.2.1 序論・アブストラクト

| 重要度 | 項目 |
|-------|------|
| ★★★ | 論文の目的・問いが明確に述べられているか |
| ★★★ | 本文の構成が簡潔に説明されているか |
| ★★ | 問いの重要性・意義が説得力を持って述べられているか |
| ★ | テーマにオリジナリティがあるか |

#### 2.2.2 本論

| 重要度 | 項目 |
|-------|------|
| ★★★ | 問いが予備知識のない読者にも丁寧に解説されているか |
| ★★★ | 主張は問いに正面から答えているか |
| ★★★ | 主張は明確で、はぐらかしがないか |
| ★★★ | テキスト・データはきちんと読解・分析されているか |
| ★★★ | 論拠から結論が論理的に帰結するか（議論の妥当性） |
| ★★★ | 扱っている分野の基本概念を正しく理解しているか |
| ★★★ | 矛盾した主張が混在していないか |
| ★★★ | 各パラグラフは議論の流れがわかるように配置されているか |
| ★★ | 論拠は十分に集められているか |
| ★★ | データの信頼性が吟味されているか |
| ★★ | 調査の質・量・分析方法は適切か |
| ★★ | 反対意見・反証がきちんと扱われているか |
| ★ | 先行研究は体系的に把握されているか |
| ★ | 問い・主張・論証に斬新な視点はあるか |

#### 2.2.3 結論・まとめ

| 重要度 | 項目 |
|-------|------|
| ★★★ | 序論で立てた問いに対応した形でまとめられているか |
| ★★★ | 何が明らかになり、何が未解決かを述べているか |

### 2.3 形式の評価基準

| 重要度 | 項目 |
|-------|------|
| ★★★ | パラグラフ・ライティングがなされているか |
| ★★★ | 文頭と文末の対応がとれているか（主語・述語の一致） |
| ★★★ | 誤字・脱字が少ないか |
| ★★ | 語の選び方・日本語表現能力は適切か |
| ★★ | 注は適切に付けられているか |
| ★★ | 引用の仕方は適切か |
| ★★ | 参考文献は適切に挙げられているか |

## 3. 成績評価の段階別基準目安

| 評価 | 状態 |
|-----|------|
| **D（不可）** | 問いがない、または剽窃がある |
| **C（可）** | 問いはあるが論証が不十分。読めば何が言いたいか伝わる |
| **B（良）** | 問い・主張・論証が揃っている。論理的に書かれている |
| **A（優）** | 上記に加え、論証が充実しており、反論も扱われている |
| **S（秀）** | 先行研究を体系的に把握し、独自の視点・知見がある |

**重要**: CからSの違いは「論証の充実度」と「問いの独自性」による。基本的な条件（問い・主張・論証の三点セット）を満たすことがまず必須。

## 4. よくある減点ポイント

### 4.1 問いに関する問題

#### パターン1: 問いがない・不明確

```
問題例:
「今日の若者の読書離れについて述べる。現代では……」

問題: 何が問いなのか不明確。記述と感想が続くだけ。

修正例:
「本論文は、なぜ近年の大学生の読書量が減少しているのかを
 メディア環境の変化との関係から考察する。」
```

#### パターン2: 大きすぎる問い

```
問題例:
「現代日本の少子化問題について論じる。」

問題: 何年分、何を、どの角度から論じるのか不明。
     何でも書けるが、何も深められない。

修正例:
「2010年代以降の女性の就業率上昇が、第一子出産の
 タイミングに与えた影響を分析する。」
```

#### パターン3: 問いらしき問い

```
問題例:
「自由とは何か」「教育とは何か」

問題: これは問いの「見出し」。具体的に答えられない。

修正例:
「なぜ日本では義務教育の年限が6・3制に定められたのか」
「経済的自由と政治的自由はどのような関係にあるか」
```

### 4.2 内容・論証に関する問題

#### 感想文になっている

| 感想文 | 論文 |
|-------|------|
| 「この問題は非常に難しいと思います」 | 問題の複雑さの具体的な説明と分析 |
| 「読んで感動しました」 | 感動した理由を論拠とともに分析 |
| 「もっと皆が考えるべきだと思いました」 | 社会的意義を論拠とともに主張 |

#### 根拠なしの主張

```
問題例:
「若者はSNSに多くの時間を費やしすぎている。」
（根拠・データが全くない）

修正例:
「総務省（2022）の調査によれば、20代のSNS利用時間は
 1日平均2.7時間で、10年前と比較して1.5倍に増加した。」
```

### 4.3 引用・出典に関する問題

| 問題 | 正しい対処 |
|-----|-----------|
| 出典を示さずに他人の主張を使う（剽窃） | 必ず「著者（年）」「（著者, 年）」等を明記 |
| ウェブサイトのURLだけを示す | 著者・サイト名・公開日・アクセス日を記録 |
| Wikipediaをそのまま引用 | 学術論文・公的文献に当たり直す |
| 参考文献リストに掲載のない文献を本文で引用 | 本文とリストを照合して一致させる |
| 引用ページ数の省略 | 直接引用には必ずページ数を明記 |

### 4.4 構成の問題

```
よくある構成ミス:

① 序論で結論を詳しく書いてしまう
   → 序論では「問いと方法」を示す。結論は結論部で。

② 本論の論点が散漫
   → 各章・各節が問いに直結しているか常に確認する。

③ 結論で新しい論点を持ち出す
   → 結論は「まとめ」。新しい議論は本論に入れるか削除。

④ 「はじめに」が自己紹介的
   → 「この論文の問いは〜」から始める。個人的な動機は
      必要最小限に。
```

### 4.5 表記・書式の問題

| 問題 | 対処 |
|-----|------|
| 表紙なし、またはページ番号なし | 提出要件を再確認 |
| 参考文献リストの形式が不統一 | 同一形式に統一（著者, タイトル, 出版社, 年） |
| 誤字・脱字 | 提出前に印刷して声に出して読む |
| 「の」の三連鎖以上 | 文の構造を組み換える |
| フォント・余白の不統一 | 一度全選択して書式をリセット |

## 5. 良い論文の条件

### 5.1 論文の三要素（絶対条件）

```
論文 = 問い + 主張 + 論証

問い  : 「なぜ〜か」「〜はどのように変化したか」等の明確な疑問
主張  : 問いに対する答え（賛成/反対、〜である、〜によってなど）
論証  : 主張を裏づける証拠と論理の配列
```

この三要素のどれかが欠けると「論文」として成立しない。

### 5.2 良い論文に見られる特徴

| 特徴 | 具体的な表れ |
|-----|-----------|
| **問いが小さく絞られている** | 一論文で完全に答えられるサイズ |
| **問いと答えが対応している** | 序論の問いに結論で正面から答えている |
| **論証が積み重なっている** | 根拠→主張が連鎖して結論に向かう |
| **反論を想定している** | 「〜という反論もあり得るが、〜ゆえに否定される」 |
| **先行研究を批判的に使っている** | 先行研究の成果を踏まえつつ、自分の立場を示す |
| **文章が明確** | 一文の主語と述語が対応し、長すぎない |

### 5.3 「知的エンターテインメント」としての論文

良い論文には「笑いどころ」と「突っ込みどころ」がある。

- **笑いどころ**: 読者が「なるほど！」「そんな視点があるのか！」と感じる瞬間。意外な切り口、鋭い分析、驚きのデータなど。
- **突っ込みどころ**: 読者が「それはどういうことだ？」「本当にそう言えるか？」と考えを深めるきっかけ。批判的思考を促す問い・論点。

## 6. 自己チェックリスト

### 6.1 提出前の必須確認（30項目）

#### 全体構成
- [ ] 問いが序論で明示されているか
- [ ] 問いに答える形で結論が書かれているか
- [ ] 各章・各節が問いへの答えに貢献しているか
- [ ] 本論で展開された内容が結論に反映されているか
- [ ] 結論で新しい論点・データを持ち出していないか
- [ ] 章立て（目次）が論文の論理の流れを反映しているか

#### 論証の質
- [ ] すべての主張に根拠（データ・文献・論理）がついているか
- [ ] 根拠から結論が論理的に導けるか
- [ ] 「〜と思う」「〜ではないだろうか」の代わりに根拠が示されているか
- [ ] 想定される反論が少なくとも一か所で扱われているか
- [ ] 先行研究との関係（一致・相違・発展）が明示されているか

#### 引用と参考文献
- [ ] すべての引用に出典が示されているか
- [ ] 参考文献リストに本文で引用した文献がすべて載っているか
- [ ] 参考文献リストに本文で引用していない文献が混入していないか
- [ ] 引用形式が統一されているか（著者・年・タイトル・出版社等）
- [ ] 直接引用のページ番号が記されているか
- [ ] ウェブサイトの引用にアクセス日が記されているか

#### 文章・表現
- [ ] 一文の主語と述語が対応しているか
- [ ] 「〜と思いました」「〜ではないかなと思います」等の感想表現を排除したか
- [ ] 「たくさん」「すごく」「とても」等の曖昧な表現を具体的な数値に換えたか
- [ ] 「の」が3つ以上連続している箇所を修正したか
- [ ] 指示語（これ・それ・あれ）が何を指すか明確か
- [ ] 一文が3行以上になっていたら分割を検討したか
- [ ] 「ここで〜を見てみる」「では〜について」等のヘタ夫表現を排除したか

#### 書式・形式
- [ ] 表紙（所属・学籍番号・氏名・タイトル・提出日）が整っているか
- [ ] ページ番号が付いているか
- [ ] 指定字数・分量を守っているか
- [ ] 章・節の番号に抜け・重複がないか
- [ ] 図表に通し番号とキャプションが付いているか
- [ ] フォント・行間・余白が統一されているか

### 6.2 最終確認（提出直前）

```
1. 印刷して声に出して読む
   → 脳内で補完してしまうミスを防ぐ

2. 問いと結論を抜き出して並べる
   → 対応しているか確認

3. 各パラグラフの最初の文だけを読む
   → 論文の論理の流れが見えてくる

4. 参考文献リストを照合する
   → 本文の引用との一致確認

5. 別の人に読んでもらう（可能なら）
   → 書き手にわかっている「当然のこと」が
     読み手に伝わっていないことが判明する
```

## 参考: 教員の評価コメントの読み方

| よくもらうコメント | 意味するところ | 修正の方向 |
|-----------------|-------------|-----------|
| 「問いが不明確」 | 論文として成立していない | 序論を全面的に書き直す |
| 「論証が弱い」 | 主張の根拠が少ない・不適切 | データ・文献を追加、論理を整理 |
| 「感想文になっている」 | 分析・論証がない | 「なぜ」「どのように」を論じる |
| 「先行研究との関係が不明」 | 既存研究を参照していない | 文献レビューを追加・深化 |
| 「引用・参考文献の不備」 | 書式・記載が不完全 | 引用規則を確認して修正 |
| 「構成が散漫」 | 論文の流れが論理的でない | 問いに沿って章立てを再設計 |

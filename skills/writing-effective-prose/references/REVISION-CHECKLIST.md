# 推敲・校正リファレンス

推敲と校正の技術および完全チェックリスト。
文章を書いた後、提出・公開前に必ず参照する。

---

## 1. 推敲の重要性

推敲していない原稿は単なる下書きに過ぎない。最終品質は推敲の質で決まる。

### なぜ推敲が難しいか

- **書き手は結論を知っている**: すでに内容を理解しているため、自分の文章のわかりにくさに気づけない
- **フィードバックが得られない**: 文章スキルは経験だけでは身につかない。意識的な推敲なしに上達しない
- **慣れ**: 書いた直後は内容に馴染みすぎて客観視できない

### 推敲の4技術

| 技術 | 具体的な実践方法 |
|------|----------------|
| **批判的な態度** | 「誤りを必ず発見するぞ」という姿勢で読む。慈悲的に読まない |
| **客観的な態度** | 時間を空ける（理想は3日）・印刷する・音読する・読み手を想像する |
| **他人に読んでもらう** | 対象読者と同じ知識量・理解度の人に依頼する |
| **ポイントを絞って複数回読む** | 1回目: 内容の過不足 / 2回目: 表現 / 3回目: 別のポイント |

## 2. パラグラフ要約文チェック（最重要確認法）

**各パラグラフの先頭文だけを読んで意味が通るかチェックする。**

これが最も効果的な推敲法であり、構成の問題を即座に発見できる。

### 判定基準

| チェック結果 | 意味 |
|------------|------|
| ✅ 意味が通る | パラグラフ構成と要約文が正しく書けている証拠 |
| ❌ 意味が通らない | 1パラグラフに複数トピックが混在、またはトピック文が冒頭にない |

### よくある失敗パターン

```
❌ 悪い例（先頭文が要約になっていない）
先頭文: 「昨年度の売上は前年比115%を達成しました。」
2文目: 「これは新製品Aの好調が主因です。」
3文目: 「また、既存製品の価格改定も貢献しました。」
→ 先頭文は事実の報告であり、パラグラフ全体の要約になっていない

✅ 良い例（先頭文が要約文）
先頭文: 「売上増加の主因は新製品Aと価格改定の2点です。」
2文目: 「新製品Aは前年比150%の売上を記録しました。」
3文目: 「既存製品の価格改定も収益改善に貢献しました。」
```

## 3. 推敲の媒体

### 印刷が最も効果的

- 画面上での修正は「推敲」と呼べない
- **A5サイズ推奨**: A4は1行が長く集中しにくい
- PDFに出力してタブレットで読む方法も有効

### 推敲のタイミング

1. まず勢いで書ききる（完璧を目指さない）
2. 時間を置く（最低でも数時間、理想は翌日〜3日後）
3. 印刷またはPDF化して推敲する

---

## 4. 校正段階で避けるべき修正

校正は「すでに書いた文章を正確にする」段階。以下は推敲不足のサインであり、校正段階でやってはいけない:

| 避けるべき修正 | 理由 |
|--------------|------|
| 新しいアイデアによる大幅加筆 | 推敲段階で解決すべき問題 |
| 大意の変わらない言い換え（「しかし」→「ところが」等） | 本質的な改善ではない |
| トピックの追加 | 構成の再設計が必要 |

## 5. セルフエディティング 完全チェックリスト

### 【A】構成チェック

- [ ] テーマを「問い」に置き換えてあるか
- [ ] 文章の冒頭に重要情報の概略（総論）があるか
- [ ] 総論で述べた順に各論が展開されているか
- [ ] **各パラグラフの先頭文だけ読んで意味が通るか**（最重要）
- [ ] 結論は何か？明確か？
- [ ] 結論を直接支える根拠（2段目）が存在するか
- [ ] 根拠はMECEにグループ分けしてあるか

### 【B】導入部チェック

- [ ] テーマと期待する反応を明示してあるか
- [ ] テーマ設定の背景を説明してあるか
- [ ] 書き手・読み手を明確にしてあるか

### 【C】視覚化チェック

- [ ] 表題・見出しを明記してあるか
- [ ] 見出しだけで内容がわかるか
- [ ] スペース・記号で論理的階層を示しているか
- [ ] 文頭で説明の切り口を明示しているか

### 【D】論理表現チェック

- [ ] 具体的か（変化のbefore/after、「誰が何をどうする」が明示されている）
- [ ] 論理的な関係を正しく表現しているか
  - [ ] 並列する見出しの表記が揃っているか
  - [ ] 「また」で切り口が埋没していないか
  - [ ] 「以下のとおり」でSo What?を放棄していないか
  - [ ] 接続詞（しかし/ただし）を正しく使っているか
- [ ] 簡潔か
  - [ ] 一文が2行以内か（主語+述語の錯綜がないか）
  - [ ] 冗長表現（「〜を行う」等）がないか
  - [ ] 受け身の乱用がないか

### 【E】文レベルチェック

- [ ] 一文一義になっているか
- [ ] 修飾語と被修飾語が近いか
- [ ] 読点は適切か（1文に1〜3個）
- [ ] 接続助詞「が」を使っていないか
- [ ] 同じ助詞が連続していないか
- [ ] 主語と述語が対応しているか
- [ ] 二重否定はないか
- [ ] 二通りの意味に解釈できる箇所はないか

### 【F】表現チェック

- [ ] 漢字とひらがなのバランスは適切か
- [ ] 1つの対象に1つの表現を使っているか（同義語バリエーション回避）
- [ ] 冗長表現を削除したか（「〜することができる」等）
- [ ] テン・マルのバランスは適切か
- [ ] 難解な漢字・専門用語を避けているか（または定義しているか）
- [ ] 俗語・話し言葉を使っていないか
- [ ] 「以上・以下・未満」を正確に使っているか

### 【G】文書全体チェック

- [ ] 表記が1文書内で一貫しているか
- [ ] 文体（ですます/である）が統一されているか
- [ ] 見出しの文体・番号体系が統一されているか
- [ ] 情報が既知→未知の流れになっているか
- [ ] 並列情報がパラレリズムで揃っているか

---

## 6. 5分間クイックチェック（最低限確認すべき5項目）

時間がないときでも、最低限これだけは確認する:

1. **パラグラフ先頭文チェック**: 先頭文だけ読んで意味が通るか
2. **総論チェック**: 冒頭の総論で内容の全体像がつかめるか
3. **一文一義チェック**: 長すぎる文がないか（2行超を探す）
4. **連続チェック**: 「が」「て」「の」が連続していないか
5. **冗長表現チェック**: 「〜を行う」「〜することができる」が残っていないか

## 7. 「著者の帽子」と「読者の帽子」

推敲の本質は、著者が「読者の帽子」をかぶり直し、自分の文章を他人の目で検証する反復作業。

```
プロセス:
1. 書く（著者の帽子）
2. 読み返す（読者の帽子に切り替え）
3. ずれを発見（書きたかったこと ≠ 文章に書かれたこと）
4. 修正（著者の帽子に戻す）
5. 繰り返す（複数の観点で）
```

**実践のコツ**:
- 書いた直後は著者の帽子が外れにくい。時間を置くほど読者の帽子をかぶりやすくなる
- 「自分が伝えたいこと」と「文章に実際に書かれていること」のずれを探す
- ずれが小さくなるにつれて推敲は終わりに近づく

## 8. 読者が迷う6つの場面

| # | 迷いの場面 | 原因 | 対処 |
|---|----------|------|------|
| 1 | 似ている語句で迷う | 「ビット列」「ビット」等の表記揺れ | 同一概念に同一表記を徹底 |
| 2 | 長い文で迷う | 複文で記憶負荷増加 | 短文に分割 |
| 3 | 言葉の不足で迷う | 前提知識の提供不足 | 定義・背景説明を追加 |
| 4 | 無駄な言葉で迷う | 冗長表現による焦点ぼけ | 不要語を削除 |
| 5 | 指示語で迷う | 「それ」「これ」の先行詞不明 | 指示語を具体的な語句に置換 |
| 6 | 言外の意味で迷う | 暗黙の前提の不一致 | 前提を明示 |

**活用法**: 推敲時にこの6パターンを順番にチェックし、該当箇所を修正する。

## 9. 推敲の3大作戦

| 作戦 | 方法 | 目的 |
|------|------|------|
| ローラー作戦 | 全文をまんべんなく読む | 誤字脱字、文法ミスの検出 |
| フェーズ作戦 | 観点を変えて繰り返し読む（1回目:論理、2回目:表現、3回目:校正） | 構造的改善 |
| メリハリ作戦 | 修正箇所とその周辺を念入りに読む | 修正の副作用確認 |

**使い分けの目安**:
- 初稿完成直後 → ローラー作戦で全体を把握
- 構成・内容の改善段階 → フェーズ作戦で観点を分けて精査
- 修正後の確認 → メリハリ作戦で副作用をチェック

## 10. 推敲の時間戦略

| タイミング | 確認対象 | 効果 |
|-----------|---------|------|
| 直後 | タイプミス、明白な誤り | 基本的な間違いの除去 |
| 数時間後 | 文法、文の構成 | 細部の改善 |
| 1日後 | 論理性、読みやすさ | 新鮮な目で読み直し |
| 1週間後 | 全体バランス、不要な説明 | 俯瞰的な判断 |
| 提出直前 | 最終確認 | フォーマット・体裁 |

### スケジュール管理の原則

- 締切りから推敲時間を逆算して執筆開始時期を決める
- 最低基準: 締切り2日前に執筆完了、翌日確認、提出直前に最終チェック（最低2回読み返し）
- 書く時間と同等かそれ以上に推敲時間を確保する

## 11. 具体的な推敲チェック語句リスト

推敲時に特に注意すべき語句:

| 注意語句 | 問題点 | 改善案 |
|----------|--------|--------|
| 「それぞれ」 | 対応関係が曖昧 | 「AはX、BはY」と明示 |
| 「基本的には」 | 例外の範囲が不明 | 具体的な条件を明示 |
| 「ある意味では」 | 逃げ道的表現 | 「どの意味か」を正確に述べる |
| 「本当に」 | 中身のない強調 | 削除するか数値化 |
| 「こと」の多用 | 冗長 | 「〜することができる」→「〜できる」 |
| 二重否定 | 認知負荷増加 | 肯定形に変換 |
| 「〜の方が」 | 「方」の意味曖昧 | 「AよりBが優れている」等に |

**使い方**: 文章を書き終えたら、上記の語句を検索して一括チェックする。特に「それぞれ」「基本的には」は使いやすいが曖昧になりがちなため要注意。

# 表現と文体

文章の骨格（論理・構成）が整ったあとに磨く「表層の技術」。同じ内容でも、表現次第で読者の受け取り方は大きく変わる。簡潔さ・リズム・語彙・強調の技術を身につけることで、文章の質を一段引き上げる。

---

## 1. 簡潔さ（削ぐ力）

「伝わる文章」の最大のポイントは簡潔さにある。難しそうな文章を書くのは簡単だ。シンプルでわかりやすい文章こそ難しい。

### 原則

- **不要な情報をカット**: 読み手のニーズに合わない情報は入れない
- **前提知識をカット**: 読み手がすでに知っていることは繰り返さない
- **無駄な修飾語を削る**: 「ものすごい」「ビックリするほど」「非常に」は文章を安っぽくする
- **骨格だけ残す勇気を持つ**: 削ることへの恐れを克服する

### 冗長表現の除去パターン

| Before（冗長） | After（簡潔） |
|--------------|-------------|
| 〜することができる | 〜できる |
| 〜と思われる | 〜である（言い切る） |
| 設定を行う | 設定する |
| 確認を行った | 確認した |
| 一番最初 | 最初（類語の繰り返し） |
| まず最初に | まず / 最初に |
| 適切な対処を行うことができるように | 適切に対処できるように |
| 〜することが可能となっている | 〜できる |
| 〜という点において | 〜において |
| 〜に関しては | 〜は |
| 〜の点については | 〜については |

### Before/After 例

**Before（冗長）:**
> 大量生産の機械加工では決して表現することができないであろう、熟練の職人技だからこそ実現できる曲線の美しさ。キーボードばかり打っていてペンを握る習慣のない手にも心地よくなじむであろう、ストレスを与えることのない握りやすさ。

**After（簡潔）:**
> 機械では表現できない、職人技による曲線の美しさ。手に心地よくなじむ握りやすさ。そして、すらすらと流れていくような書き心地。

## 2. リズム（グルーヴ感）

文章にはリズムがある。適切な言葉選択・テン・マルの使いこなし・韻の意識によって、文章は音楽のような流れを持つ。

### リズムのない文章の特徴

- 「思った」が多用されている
- 形容詞が乱発されている
- 文が長すぎる（あるいは短すぎる）
- すべての文が同じ長さ

### リズムを生み出す方法

1. **無駄を省く** ── 冗長な表現を削ると自然にテンポが生まれる
2. **テンの位置を修正する** ── 息継ぎの場所を意図的に設計する
3. **文の長短を交互にする** ── 長い文のあとに短い文を置くとメリハリが生まれる

### Before/After 例

**Before（単調なリズム）:**
> このプロダクトは多くの機能を持っており、ユーザーはさまざまな操作を行うことができ、またカスタマイズも自由に行うことができ、さらに他のツールとの連携も簡単に行うことができます。

**After（リズムある文章）:**
> このプロダクトは多機能だ。ユーザーは自由にカスタマイズできる。他ツールとの連携も簡単。

## 3. テンとマルの戦略的使用

### テン（読点）の5つの機能

| 機能 | 説明 | 例 |
|------|------|-----|
| ① 時間の経過 | 動作の前後関係を明確にする | 「資料を読み、内容を確認した。」 |
| ② かかり受けの明確化 | どの言葉がどこにかかるかを示す | 「静かな、部屋の隅に座った。」 |
| ③ 強調 | テンの前後の言葉を際立たせる | 「重要なのは、速さではなく精度だ。」 |
| ④ 伝達事項の分かち書き | 複数の情報を区切る | 「日時は3月10日、場所は会議室A、参加者は5名。」 |
| ⑤ リズム生成 | 読む速度と呼吸をコントロールする | 「走って、転んで、立ち上がった。」 |

### テンの実践ルール

- **一文にテンは原則1個、多くても2〜3個**
- テンが少なすぎると息苦しく読みにくい
- テンが多すぎると散漫になり意味が分散する
- 勢いで読ませたい場合はあえてテンを減らす

### マルの戦略的使用

- **長い文はマルで分割する** ── 一文が50〜100文字を超えたらマルで切る
- **マルを多用すると文章が締まる** ── 短い文の連続は力強さを生む

### Before/After 例

**Before（テンが多すぎ）:**
> このシステムは、大量のデータを、高速に、正確に、処理することが、できます。

**After（適切なテン）:**
> このシステムは、大量のデータを高速かつ正確に処理できます。

## 4. 漢字とひらがなのバランス

文章全体を「面」として見たとき、漢字とひらがなのバランスが読みやすさに直結する。

- **漢字多すぎ** ── ぎっちりして読む気が失せる
- **ひらがな多すぎ** ── 読みにくく、相手を子ども扱いするような印象を与える

### 習慣的に「ひらく」漢字一覧

| 漢字表記 | ひらがな | 備考 |
|---------|---------|------|
| 例えば | たとえば | 接続表現 |
| 特に | とくに | 強調副詞 |
| 但し | ただし | 接続表現 |
| 尚 | なお | 接続表現 |
| 更に | さらに | 接続表現 |
| 因みに | ちなみに | 接続表現 |
| 事 | こと | 形式名詞 |
| 訳 | わけ | 形式名詞 |
| 出来る | できる | 補助動詞 |
| 様な | ような | 形式名詞 |
| 為 | ため | 形式名詞 |
| 時 | とき | 形式名詞 |

### 判断基準

- 独立した意味を持つ実質的な名詞・動詞 → 漢字でよい
- 文法的な役割を果たすだけの形式名詞・補助動詞 → ひらがな

## 5. 具体的に書く

抽象的な表現は読者の頭に絵が浮かばない。具体性が文章の説得力を決める。

### 変化はbefore/afterを示す

**悪い例（抽象）:**
> ライフスタイルの変化によって腕時計需要が低迷した。

**良い例（具体）:**
> 携帯電話の普及とともに時計機能を携帯電話に代替させる層が拡大し、腕時計需要が低迷した。

---

### アクションは「誰が何をどうする」を明示する

**悪い例（誰が不明）:**
> ピーク時対応を図る。

**良い例（誰が何をどう）:**
> 代表オペレータが兼務する受付業務のピーク時間に、オペレータ応援要員を確保する。

---

### 判断基準は「何がどうなればよしとするか」まで書く

**悪い例（基準が曖昧）:**
> リスク対応を実施する。

**良い例（基準が明確）:**
> 安定供給上のリスク対応力が高い仕入先を選定し、代替仕入先を2社以上確保する。

---

### 数値・固有名詞を活用する

| Before（曖昧） | After（具体） |
|--------------|-------------|
| 多くのユーザーが使用 | 月間アクティブユーザー1万人が使用 |
| 処理が速くなった | 処理時間が3秒から0.2秒に短縮された |
| 最近リリースされた | 2024年11月にリリースされた |
| 大幅なコスト削減 | 年間200万円のコスト削減 |

## 6. ですます調 vs である調

### 比較表

| | ですます調（敬体） | である調（常体） |
|--|-----------------|---------------|
| 印象 | 語りかけるような親しみ | 強さ・断定感 |
| 適する場面 | Webメディア、ブログ、メール | 新聞、雑誌、企画書、論文 |
| 注意点 | 単調になりやすい | 冷たく感じやすい |

### 原則

- **1つの文章内で混ぜない** ── 読み手に違和感を与える
- **両方書けるようにしておく** ── 媒体・読者に合わせて選択する

### Before/After 例

**混在（NG）:**
> このツールは非常に使いやすい。機能が豊富で、初心者でも扱えます。

**統一（OK）:**
> このツールは非常に使いやすい。機能が豊富で、初心者でも扱える。（である調）
> このツールは非常に使いやすいです。機能が豊富で、初心者でも扱えます。（ですます調）

## 7. 語彙の選択

### 基本原則

- **一般的な言葉を使う** ── 専門用語は定義してから使う
- **多義語を避ける** ── 「適当」は「適切」か「いいかげん」か読み手には判断できない
- **一対象には一表現** ── 同義語をバリエーションで使うと混乱を招く

### 俗語・話し言葉を書き言葉へ

| 俗語・話し言葉 | 書き言葉 |
|-------------|---------|
| 普通に使えます | 十分役立ちます |
| デフォのまま | 初期設定のまま |
| めちゃくちゃ速い | 非常に速い / 大幅に高速化された |
| ざっくり言うと | 概要を述べると |
| いい感じ | 適切 / 良好 |
| 〜っていうか | 〜というよりも |

### 注意すべき表記：「以上・以下・未満」

| 表現 | 意味 | 例 |
|------|------|-----|
| 以上 | その数値を含む（≥） | 18歳以上 = 18歳も含む |
| 以下 | その数値を含む（≤） | 10件以下 = 10件も含む |
| 未満 | その数値を含まない（<） | 18歳未満 = 17歳まで |
| 超える | その数値を含まない（>） | 100を超える = 101から |

## 8. 強調の技術

強調を多用すると強調の意味がなくなる。**絞ること**が重要。

### 4つの強調手法

1. **語順を工夫する**
   - 強調したい要素を**文末**に置く
   - 日本語は文末が最も印象に残る

2. **繰り返す**
   - 重要な内容を複数の箇所で述べる
   - ただし過剰な繰り返しは読者を疲れさせる

3. **視覚的効果を使う**
   - 太字（**太字**）
   - 下線（<u>下線</u>）
   - 括弧（「重要」）
   - 文字サイズ変更（見出しレベルの活用）

4. **強調箇所を絞る**
   - 1段落に強調は1〜2箇所まで
   - 全部を強調すると何も強調されない

### Before/After 例

**Before（強調が多すぎ）:**
> **このシステム**は**非常に重要**で、**すべてのユーザー**が**必ず使わなければなりません**。**機能A**と**機能B**と**機能C**が**特に重要**です。

**After（絞った強調）:**
> このシステムはすべてのユーザーが必ず使う基盤ツールだ。中でも**機能Aは日常業務に直結**するため、優先的に習得してほしい。

## 9. 能動態と受動態の使い分け

「受動態を避け能動態を使え」というアドバイスは半分正しく、半分誤りだ。

### 一文単位では：能動態が読みやすい

| 受動態 | 能動態 |
|-------|-------|
| このシステムはエンジニアによって開発された | エンジニアがこのシステムを開発した |
| 結果が報告された | 担当者が結果を報告した |

### 文章全体では：情報の流れが優先される

**既知→未知の情報の流れ**を守ることが最重要ルール。無理に能動態にすると文章全体の表現力が落ちる場合がある。

**受動態が自然な例:**
> 昨日、会議室で重要な発表があった。**その発表は**、全社員に向けて行われた。

この文脈では「その発表は」が既知情報として文頭に来るため、受動態が自然。無理に能動態にすると「全社員に向けてその発表を行った」と「誰が」が不明になる。

### 判断基準

1. 一文だけを見たとき → 能動態が読みやすければ能動態
2. 前後の文脈で「既知情報が主語になるか」 → 受動態が自然なら受動態
3. 動作主を省略したいとき → 受動態が有効（「このフォームは毎月更新されます」）

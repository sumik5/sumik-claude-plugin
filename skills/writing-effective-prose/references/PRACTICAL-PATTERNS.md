## 1. 上司を説得する文章

### 目的

読み手（上司・権限者）の判断を変え、自分の提案を採用させる。
感情的アピールでなく、論理と論拠で意思決定を促す文章。

---

### 改善の5ステップ

| ステップ | 内容 | ポイント |
|---------|------|---------|
| 1. 結果をイメージする | 最高の結果を具体的に思い描く | 「何が実現するか」を先に描く |
| 2. 論点を決める | 否定→提案に変換する | 「人事判断が間違い」→「成功させるには誰がリーダーか？」 |
| 3. 意見をはっきりさせる | 具体的な提案を一文で表明 | 「田中さんが適任です」 |
| 4. 論拠を用意する | データ・実績を収集する | 数字・事例・証言 |
| 5. アウトラインをつくる | 論点→論拠→意見の順に組む | 最後に意見を置く構造 |

---

### グレードアップ5ポイント

1. **自分の論に自分で反論する**
   想定される反論を先取りして論じることで、読み手の疑問を先回りする。

2. **相手の論拠を押さえる**
   相手が何を根拠にしているか、できれば直接聞く。「なぜそう考えているか」を知ることで、的外れな反論を避けられる。

3. **視野を広げる**
   - 時間軸: 過去の経緯・将来予測を加える
   - 他社事例: 同様の状況での成功・失敗事例
   - 文献: 客観的なデータや研究知見

4. **再反論のあるアウトラインをつくる**
   反論→再反論まで盛り込んだ構成で、より堅牢な説得力を持たせる。

5. **相手の仕事への共感点を文頭に置く**
   読み手が「この人はわかってくれている」と感じると、信頼性が上がり意見を受け入れやすくなる。

---

### 基本アウトライン（説得文）

```
【共感・問題認識の共有】
相手の立場・仕事への理解を示す。

【論点の提示】
「本質的な問いは何か」を明示する。

【自分の意見】
具体的な提案を明言する。

【論拠①②③】
データ・実績・事例を並べる。

【想定される反論と再反論】
相手の疑念を先取りして論じる。

【結論（意見の再確認）】
最終的に取るべき行動を明示する。
```

## 2. 依頼文（お願いの文章）

### 目的

相手に「やりたい」と思わせる。自分の都合を押しつけず、相手のやる気に火をつける構成。

---

### 基本構成

| 順序 | 要素 | 内容 |
|-----|------|------|
| 1 | 挨拶 | 丁寧な書き出し・関係性の確認 |
| 2 | 自己紹介 | 氏名・所属・志（なぜこの仕事をしているか） |
| 3 | 依頼内容 | 何をお願いしたいか、具体的に |
| 4 | 依頼理由 | なぜ「あなた」に頼むのか（特定の理由） |
| 5 | 条件 | 別紙要項（詳細は手紙と分離） |
| 6 | 返事の方法 | 連絡先・期限・方法の明示 |

---

### やる気を引き出す3つの動機

1. **志に共感させる**
   自分が何のためにこの仕事をしているかを伝える。相手の価値観と重なる部分を見つけて示す。

2. **面白いと感じさせる**
   「この仕事ならではの面白さ」を具体的に伝える。

3. **やる必然性を感じさせる**
   「なぜあなたでなければならないか」を明確に伝える（「誰でもいい」という印象を与えない）。

---

### やる気をなくすNG表現

- ❌ 自分の都合ばかり並べる（「急ぎです」「こちらの事情で…」）
- ❌ 誰でもいいという扱い（「どなたかお願いしたい」「適任者を」）
- ❌ 手紙に詳細を詰め込みすぎる（読む前に疲れさせる）

---

### 手紙と要項を分ける理由

| 文書 | 目的 | 内容 |
|-----|------|------|
| 依頼手紙 | やる気を引き出す | 志・共感・理由・感謝 |
| 別紙要項 | やることを明示する | 日時・場所・内容・条件・謝礼 |

2つを一緒にすると、手紙の感情的な効果が薄れる。

## 3. 議事録

### 最重要ポイント

> 発言をそのまま記録するのではなく、**議題を「問い」の形で頭に大きく書く**。

「問いに対して何が決まったか」を軸に構造化することで、後から読んでも意味がわかる議事録になる。

---

### 基本構造

| 要素 | 内容 | 記述形式 |
|-----|------|---------|
| **議題** | 会議で答えるべき問い | 疑問文で明示（例: 「Q&A動画を作成すべきか？」） |
| **決定事項** | 問いに対する答え | 端的に。「〜することとなった」 |
| **会議の流れ** | 検討の手順・論点の変化 | 時系列ではなく論理的な流れで |
| **議事の要点** | 何が大切にされたか・何が保留か | 価値判断・背景を含めて |

---

### グレードアップ

**前後の流れを明示する**
```
前回の議題:（前回答えようとした問い）
本日の議題:（本日答えようとした問い）
次回の予定:（次に答えるべき問い）
```

**会議の位置づけを書く**
- 決定の会議（合意形成）
- ブレインストーミング（アイデア発散）
- 共有・報告（情報の共有）

**今後の課題を疑問文で書く**
「〜をどうするか」の形で書くことで、次の行動方向が明確になる。

---

### Quick Reference（議事録テンプレート）

```
# 議事録

**日時**: YYYY-MM-DD HH:MM〜HH:MM
**参加者**: 〇〇、〇〇、〇〇
**ファシリテーター**: 〇〇
**記録者**: 〇〇

---

## 前回の流れ
（前回の問い・決定事項の概要）

## 本日の議題（問い）
Q: ？

## 決定事項
→ （答え）

## 会議の流れ
1. 〜を確認した
2. 〜について議論した
3. 〜が明らかになった

## 議事の要点
- 重視した観点: 〜
- 保留事項: 〜

## 今後の課題
- 〜をどうするか？（担当: 〇〇、期限: 〇〇）

## 次回の議題（予定）
Q: ？
```

## 4. お詫び文

### 謝罪の3要件

お詫び文として機能するには、以下の3つがすべて必要。どれが欠けても不完全。

| 要件 | 意味 |
|-----|------|
| **反省** | 何が問題だったかを自覚し、認める |
| **謝罪** | 迷惑・不快をかけたことへの詫び |
| **償い** | 今後どうするか・何を取り戻すか |

---

### 7段落構成

```
① 謝罪（開幕で謝る）
　→ 冒頭に謝罪の言葉を置く。

② 相手側から見る（相手の迷惑・気持ちを具体的に書く）
　→ 自分の事情ではなく、相手にとって何が大変だったかを書く。

③ 罪を積極的に認める
　→「私のミスです」「私の責任です」と明言する。曖昧にしない。

④ 原因を究明する（言い訳でなく、自分の怠慢・力不足で語る）
　→「〜のため」ではなく「私が〜を怠ったため」という主語で書く。

⑤ 対策を立てる（具体的に）
　→ 精神論は避ける。「いつ・誰が・何をどう変えるか」を明示する。

⑥ 償いをする
　→ 創意工夫を心がける。「どう埋め合わせるか」を示す。

⑦ 再度謝罪（締めくくる）
　→ 改めて謝罪の言葉で締める。
```

---

### NGパターン

| NG | 問題点 |
|----|--------|
| 謝罪のみで反省・償いなし | 形式的で誠意が伝わらない |
| 「全員一丸」「チームで改善」の精神論 | 誰が責任者か不明。対策が空虚に見える |
| 「ご指導ご鞭撻をお願いします」 | 責任を相手に転嫁している印象を与える |
| 1人称を「私たち」「弊社一同」にする | 責任の所在が曖昧になる |

## 5. 志望理由書

### キーを見つけることが最重要

志望理由書の核心は「なぜここでなければならないか」のキーワード（接点）を見つけること。

| 対象 | キーの見つけ方 |
|-----|--------------|
| 大学（学部・学科）| 学部・学科名がキー。そこで何を学ぶかから逆算する |
| 会社（就活）| 「分野×業種」または「分野×職種」がキー |

---

### 3要件

1. **志望分野をとりまく社会認識**
   その分野の現状・問題・理想を自分の言葉で語る。
   「社会はどうなっているか、どうあるべきか」

2. **志望分野との関係における自己認識**
   自分がその分野に至った動機、自分の資質、貢献できる点を示す。
   「私はなぜここに来たのか、何ができるのか」

3. **先方を選んだ理由**
   その大学・会社の理念・方針・特色への共感を具体的に書く。
   「なぜ他ではなくここか」

---

### やってはいけない論法

**なし崩しの論法**（他がだめだからここにした）
- ❌「第一志望に合格できなかったため」
- ❌「他の会社に断られたため〜が魅力に見えてきた」
- ✅「〜に惹かれたため、最初からここを目指していた」

先方を選んだ理由は「積極的な選択」として書く。

---

### アウトライン（志望理由書）

```
【社会認識】
・この分野をとりまく現状
・問題点・課題
・理想の姿（どうあるべきか）

【自己認識】
・この分野への関心を持った動機
・自分の資質・経験・強み
・貢献できること

【先方を選んだ理由】
・この大学/会社の理念・方針
・共感した具体的なポイント
・ここでなければならない理由

【まとめ・抱負】
```

## 6. メール

### シーソーの原則

> 書き手が考える手間を省くと、読み手の負担が増える。
> 読み手の負担を減らすには、書き手が考え抜く必要がある。

---

### 2ステップ

**Step 1: 一番言いたいことを決める**
- 「誰が何をどうするか」まで打ち出す
- 意見・結論を文頭に置く（報告か・相談か・お願いかを明確に）

**Step 2: 論拠を決める**
- 優先順位が高いものから並べる
- メールは「頭に大事なことを書く」原則

---

### グレードアップ

**相手にとっての意味を明示する**

| 種類 | 冒頭の一言 |
|-----|-----------|
| 報告 | 「〇〇についてご報告します。」 |
| 相談 | 「〇〇についてご相談したいことがあります。」 |
| お願い | 「〇〇についてご対応をお願いしたく連絡しました。」 |
| 返信期限 | 「〇月〇日までにご回答いただけますと幸いです。」 |

**タイトルの設計**
タイトル = 「論点」＋「相手にとっての意味」

```
【例】
× 「予算について」
○ 「至急！ 秋の特別号予算アップのお願い（要回答）」
```

---

### 文字量の目安

| 媒体 | 目安 |
|-----|------|
| ビジネスメール | 2,000字がギリギリの上限 |
| 理想 | スクロール不要な範囲（30字×30行 ≒ 900字） |

長くなる場合は添付資料に分離し、メール本文は要点のみに絞る。

## 7. 報告書・提案書（ビジネス文書の型）

### 文書タイプ別アウトライン

**提案書**
```
背景（現状の問題点）
→ 提案内容
→ メリット①②③
→ デメリットへの反論
→ 結論（アクション）
```

**調査報告書**
```
現状
→ 問題
→ 解決策
→ 効果
→ 課題
```

**技術報告書**
```
背景
→ 目的
→ 調査方法
→ 調査結果
→ 考察
```

---

### 文章作成の11手順

| # | 手順 | 内容 |
|---|------|------|
| 1 | 総論を書く | 全体の主張・結論を最初に書く |
| 2 | 全体構成をパラグラフ単位で検討 | 何を書くかのリスト化 |
| 3 | パラレリズムできるパラグラフを検討 | 並列できる情報を見つける |
| 4 | 各パラグラフの要約文を作成 | 1段落=1メッセージを決める |
| 5 | 各パラグラフを展開 | 要約文に肉付けする |
| 6 | 総論を見直す | 各論と矛盾していないか確認 |
| 7 | 既知→未知の流れを確認 | 読み手が知っていることから始まっているか |
| 8 | パラレリズムの確認 | 並列構造が表現まで統一されているか |
| 9 | 一文一ポイントの確認 | 1文に複数の情報が混在していないか |
| 10 | 簡潔さの確認 | 不要な語句を削る |
| 11 | 時間をあけて見直し | 可能なら3日後に再読 |

## 8. 企画書

### 目的

読み手に「これはやる価値がある」と感じさせ、動いてもらう。
論理と魅力を両立させる文書。

---

### 5段階構成

```
① 企画意図（なぜこの企画を書こうと思ったか）
　→ 背景や問題意識を簡潔に。冗長にしない。

② 現状分析（いまどうなっているか）
　→ 感情でなく、データ・事実に基づいた冷静な分析。

③ 問題点の抽出（現状から導き出される課題）
　→ 「だから何が問題か」を一文で言える形に。

④ 打開策（問題解決のアイデア提示）
　→ 具体的なアイデアを示す。「なぜこれが有効か」も添える。

⑤ さりげないアピール（企画の有用性・面白さ）
　→ くどくならず、シンプルに。「なぜいま・なぜこれか」を残す。
```

---

### 企画書を書くときの注意

- 「説得しよう」と意識しすぎると力みが出る。「伝えよう」と意識する。
- 数字・データは「支持材料」として使う。数字で押しつけない。
- アピールは「さりげなく」。過度な自画自賛は逆効果。

## 9. 「つかみ」のテクニック

### 冒頭で読み手を引き込む

> 読み手に「なんだろう？」「もっと知りたい」と思わせる冒頭が良い「つかみ」。

---

### 4つのアプローチ

| アプローチ | 方法 | 例 |
|-----------|------|-----|
| **効能を示す** | 「読むと何が得られるか」を冒頭に置く | 「この文章を読めば、説得力が3倍になる」 |
| **切実な話題とリンク** | 読み手が今困っていることと接続する | 「会議でいつも意見が通らないと感じていないか」 |
| **タイムリーな話題** | 今起きていること・流行と結びつける | 「AIが台頭する今、書く力はより重要になった」 |
| **独特の世界を醸す** | 著者独自の視点・表現で世界観を作る | 文体・比喩・語り口による没入感 |

---

### 注意点

- 「狙った感」が露骨にならないよう注意する。
  テクニックが透けて見えると、読み手は冷める。
- あくまで「本文への入り口」として機能させる。
  つかみが派手でも、本文に続く価値がなければ意味がない。

## 10. 引用のバランス

### 原則

- **引用は全体の約3割に抑える**
  残り7割は自分の言葉・分析・解釈。

- **「どこを引用するか」が書き手の個性**
  同じ本から同じ部分を引用しても、選ぶポイントが違えば別の文章になる。
  引用の選択眼こそが書き手の視点を示す。

- **あれもこれも引用しない**
  印象的な部分を厳選する。「フック」となる一節だけを使う。

---

### 引用の使い方

| 用途 | 方法 |
|-----|------|
| 権威付け | 専門家・公的機関の発言・データを引く |
| 具体化 | 抽象的な主張に実例・発言を添える |
| 共感の喚起 | 読み手が「そうそう」と感じる言葉を引く |
| 反論の提示 | 自分と異なる立場の意見を引いてから論じる |

引用後は必ず「自分の解釈・コメント」を続ける。引用だけで段落を終わらせない。

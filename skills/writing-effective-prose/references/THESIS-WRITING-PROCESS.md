## 1. 卒論とはどういうものか

### 1.1 レポートとの本質的な違い

| 項目 | レポート | 卒業論文 |
|------|----------|----------|
| **問いの設定** | 与えられることが多い | 自分で立てる（必須） |
| **先行研究** | 部分的に参照 | 体系的に整理・批判的検討が必須 |
| **オリジナリティ** | 求められないことも多い | 何らかの新しい知見・視点が必要 |
| **分量** | 数枚〜20枚程度 | 40〜100枚以上（学部・分野による） |
| **構成の複雑さ** | 序論・本論・結論 | 複数の章・節に細分化 |
| **研究期間** | 数日〜数週間 | 1年〜2年 |

### 1.2 卒論が課される理由

卒論は「4年間の学修の総まとめ」ではなく、**一つの研究プロジェクトを企画・実行する能力の実証**である。問いを立て、先行研究を整理し、独自に調査・分析して結論を導く——この一連のプロセスを自力で完遂できるかどうかが問われている。

「どれだけたくさん調べたか」や「どれだけ大きな研究をしたか」ではなく、**一つのプロジェクトを設計して実行できる力があるか**が評価の基準になる。

## 2. 全体スケジュールの設計

### 2.1 1年間の標準的な流れ

```
[前期]
4〜5月  テーマの絞り込み、先行研究の調査開始
6月     先行研究の整理、問いの定式化
7月     中間報告・ゼミ発表（仮テーマ・問いの発表）
8〜9月  夏休み：本格的な資料収集・データ収集、下書き開始

[後期]
10月    章ごとの草稿作成
11月    全体の草稿完成、指導教員のフィードバック
12月    推敲・修正（繰り返し）
1月前半 最終版の完成
1月末   提出
2月     口頭試問（学校・学部による）
```

### 2.2 時間配分の目安

| フェーズ | 目安期間 | 主な作業 |
|---------|---------|---------|
| **テーマ探索・問いの定式化** | 2〜3ヶ月 | 広く読む、問いを絞る |
| **本格的な資料調査** | 2〜3ヶ月 | 文献収集、データ取得、一次資料分析 |
| **執筆（初稿）** | 2〜3ヶ月 | 章ごとに書く |
| **推敲・修正** | 1〜2ヶ月 | 読み返し、構成の見直し、文章の磨き込み |

**注意**: 初稿を完成させるだけでは不十分。提出の3週間前には初稿を完成させ、推敲に十分な時間を確保すること。

### 2.3 逆算スケジューリング

提出日から逆算して締め切りを設定する。

```
提出日 ← 最終確認（1週間） ← 最終版完成 ← 推敲（2週間）
    ← 初稿完成 ← 第3章執筆（2週間） ← 第2章執筆（2週間）
    ← 第1章執筆（2週間） ← 序章執筆（1週間）
    ← 章立ての確定 ← 資料収集完了 ← ...
```

## 3. テーマ設定の進め方

### 3.1 テーマ選びの大原則

**「自分が本当に興味を持てるか」が最重要の基準**。論文を書く作業が楽しくなるかどうか、最終的に良い論文が仕上がるかどうかは、すべてこの一点にかかっている。卒論は長期間の作業であるため、自分の興味から外れたテーマを選ぶと途中で苦痛になる。

「人からすすめられて……」「就職に役立つから……」といった安易な決め方はしない。

### 3.2 テーマを絞る手順

| ステップ | 作業内容 | チェックポイント |
|---------|---------|----------------|
| **1. 広い関心を確認する** | 自分が最も楽しいと感じる分野・活動を挙げる | 「小さい頃から何をしている時が楽しいか」を自問 |
| **2. 研究可能かを確認する** | 文献・データが入手できるか調べる | 先行研究が0件では書けない、多すぎると絞れない |
| **3. 問いの形にする** | 「〜とは何か」ではなく「〜はなぜか」「〜はどのように変わったか」に変換 | 「〜とは何か」という問いは問いではない |
| **4. 小さく絞る** | 「えっ、こんなに小さくていいの？」と思うくらい絞り込む | 小さく絞った問いは深めることができる |

### 3.3 論文テーマの2類型

#### 論争型論文
賛否両論のある社会的テーマを扱い、論争の経緯を整理しながら自分の立場を主張する。

- **利点**: 論点が明確、自分の意見を主張しやすい
- **注意**: あまりにもホットな直近の問題は資料が少ない
- **適したテーマ例**: 「○○政策の是非」「△△現象の社会的影響」

#### 歴史・記述型論文（伝記型論文）
ある人物・事象・制度の歴史や変遷を体系的に記述する。

- **利点**: 資料が見つかりやすい、論証の方法が明確
- **注意**: 「まとめただけ」にならないよう、独自の分析視点が必要
- **適したテーマ例**: 「○○の発展史」「△△制度の変遷」

### 3.4 避けるべきテーマの例

| 問題パターン | 具体例 | 理由 |
|------------|-------|------|
| **大きすぎる問い** | 「日本社会の変化について」 | 何も書けず、結局浅い論文になる |
| **問いらしき問い** | 「権利とは何か」「芸術とは何か」 | これは問いの「見出し」であって、問いではない |
| **資料がない** | 非常にタブー的なトピック | 先行研究なしでは書けない |
| **刻々と変わる問題** | 今まさに展開中の政治問題 | 信頼できる資料が少ない、書き終わる前に状況が変わる |

## 4. 資料調査と文献レビュー

### 4.1 資料収集の2段階

| 段階 | 目的 | 使う資料 |
|-----|------|---------|
| **問いを見つけるための読書** | 興味のある分野の概観をつかみ、「引っかかり」を見つける | 新書・文庫・入門書・事典 |
| **問いを解決するための読書** | 自分の問いに直接答えるための証拠・論拠を集める | 専門書・学術雑誌・原典 |

### 4.2 文献調査の進め方

**スノーボール法**: 良い論文・本の参考文献リストから、さらに関連文献を探す。

1. 指導教員に「この問いに関する基本文献を教えてください」と聞く（最短経路）
2. 図書館の検索システムでキーワード検索
3. CiNii（国内論文）、Google Scholar で論文を探す
4. 見つけた論文の参考文献から芋づる式に探す
5. 英語文献も確認する（日本語では研究が少ない分野でも英語では豊富なことがある）

### 4.3 ノートの取り方

```
カードの記録内容（1枚に1情報）:
- 書誌情報（著者・書名・出版年・ページ数）← 後で必ず必要になる
- 内容の要約（自分の言葉で）
- 直接引用する場合はカギ括弧で囲む
- 自分のコメント・疑問点（別の色で書くと区別できる）
```

**絶対ルール**: 文献を読んだ瞬間に書誌情報を完全に記録する。後から探し直すのは非常に手間がかかる。

### 4.4 文献レビューの書き方

文献レビューは「読んだ本の紹介」ではなく、**自分の問いとの関係で先行研究を整理する作業**。

```
良い文献レビューの構造:
1. この問いに関してどのような研究があるか（研究の全体像）
2. 各研究が何を明らかにしたか（成果と限界）
3. まだ明らかにされていないことは何か（研究の空白）
4. 自分の研究がその空白にどう貢献するか（自分の研究の位置づけ）
```

## 5. 指導教員との関わり方

### 5.1 相談のタイミング

| タイミング | 相談すべき内容 | 準備するもの |
|-----------|--------------|-------------|
| **テーマ設定段階** | テーマの方向性、先行研究の有無 | 自分の興味・関心のメモ |
| **問いの定式化段階** | 問いが適切かどうかの確認 | 仮の問い3〜5案 |
| **資料収集段階** | 重要な文献の紹介依頼 | 既に集めた文献リスト |
| **章立て確定後** | 構成の妥当性チェック | アウトライン（目次案） |
| **初稿完成後** | 内容・論理の確認 | 章ごとの草稿 |

**重要**: 「ある程度自分で調べてから」相談する。何も準備せずに「どんなテーマにしたらいいですか」と聞いてはいけない。

### 5.2 報告の仕方

```
効果的な報告の構成:
1. 現在の状況（どこまで進んでいるか）
2. 取り組んでいる問題点（何で詰まっているか）
3. 自分なりの解決策の仮説（〜と思うのですが、どうでしょうか）
4. 具体的なアドバイスの依頼（〜について教えていただけますか）
```

### 5.3 指導教員との関係で注意すること

- フィードバックをもらったら、次の面談までに必ず何らかの修正・進展を見せる
- 「わかりました」で終わらせず、具体的に「〜をします」と答える
- 指導教員の専門外の問いを立てた場合は、早めに相談して調整する
- 締め切りを守る（守れない場合は事前に報告する）

## 6. 卒論の構成と章立て

### 6.1 標準的な構成

| 章・節 | 内容 | 分量の目安 |
|-------|------|----------|
| **序章（第1章）** | 問いの提示、研究の背景・意義、論文の構成 | 全体の10〜15% |
| **第2章（文献レビュー）** | 先行研究の整理と批判的検討 | 全体の20〜25% |
| **第3章（方法・分析枠組み）** | 調査方法、使用するデータ・資料、分析の手続き | 全体の10〜15% |
| **第4〜5章（結果・分析）** | 調査・分析の結果、事実の記述と解釈 | 全体の30〜40% |
| **第6章（考察）** | 結果の意味、先行研究との関係、限界 | 全体の10〜15% |
| **結章（終章）** | 全体の要約、問いへの回答、今後の課題 | 全体の5〜10% |
| **参考文献リスト** | 本文で引用・参照したすべての文献 | — |

### 6.2 序章で書くべきこと

```
序章の4要素:
1. 問いの明示（この論文は何を問うのか）
2. 研究の背景・必要性（なぜこの問いが重要なのか）
3. 研究の方法・対象（どのように調べるのか）
4. 論文の構成（各章で何を論じるのか）
```

**注意**: 序章で結論を書いてはいけない。しかし「この論文は〜を主張する」という形で問いへの答えを示すことはよい。

### 6.3 考察（ディスカッション）の書き方

考察は「まとめ」ではなく、**結果の意味を解釈して主張を論証する**場所。

- 調査・分析で明らかになったことが問いにどう答えるかを明示する
- 先行研究と自分の結果がどう関係するか（支持、修正、反証）を論じる
- 「思った通りだった」は考察ではなく感想。「なぜそうなったか」を論じる
- 自分の研究の限界（できなかったこと）も正直に書く

## 7. 中間発表・口頭試問への準備

### 7.1 中間発表（ゼミ発表）

| 段階 | 発表内容 | 準備のポイント |
|-----|---------|--------------|
| **テーマ発表（5月〜6月）** | 問い・先行研究の概要・方法 | 問いの独自性と実現可能性を示す |
| **進捗発表（10〜11月）** | 第1〜2章の草稿、調査進捗 | 論理の流れを口頭で説明できるようにする |
| **最終発表（1月）** | 全体の要約・主張・結論 | 質問への回答を準備しておく |

**発表スライドの構成（15分発表の場合）**:
```
1. 問いと背景（2〜3分）
2. 先行研究と本研究の位置づけ（3分）
3. 調査方法（2分）
4. 結果・分析（5分）
5. 結論と今後の課題（2〜3分）
```

### 7.2 口頭試問への準備

口頭試問では主に以下が問われる:

| よく聞かれる質問 | 準備すべき回答 |
|----------------|--------------|
| なぜこの問いを選んだのか | 問いの意義・独自性を1〜2分で説明できる |
| 先行研究との違いは何か | 自分の研究がどこに新しさがあるかを明確に |
| なぜこの方法・資料を使ったのか | 方法の選択理由を論理的に説明できる |
| 限界・課題は何か | 正直に認め、今後どう克服できるか示す |
| 〜という反論についてはどう考えるか | 想定される批判への回答を準備しておく |

**口頭試問前日までにやること**:
- 自分の論文全体を1〜2時間で読み直す
- 各章の主張を一文で言えるようにする
- 予想質問を自分で作り、声に出して答える練習をする

## 8. 卒論と修士論文の違い

| 観点 | 卒業論文（学部4年） | 修士論文 |
|-----|------------------|---------|
| **独自性の要求水準** | 何らかの新しい視点・知見があればよい | より高度なオリジナリティが必要 |
| **先行研究の扱い** | 主要文献を網羅的に参照 | 当該分野のほぼすべての先行研究を把握・批判的検討 |
| **研究方法** | 既存の方法を適切に使えばよい | 方法論的な議論も含むことが多い |
| **分量** | 40〜80枚（学部・分野による） | 80〜200枚（分野による） |
| **審査** | ゼミ・口頭試問（学部による） | 公開発表・審査委員会での審問 |
| **社会への貢献** | 求められないことが多い | 学術的貢献の明示が求められる |

**修士論文特有の要件**:
- 「なぜこの研究が学術的に重要か」を先行研究の文脈で明示する必要がある
- 一次資料（原典・データ）を自ら収集・分析することが多い
- 理論的な貢献と実証的な貢献の両方を意識する

## 9. 進捗管理と書けない時の対処法

### 9.1 書けない原因と対処法

| 書けない理由 | 対処法 |
|------------|-------|
| **最初の一行が書けない** | 「人に話す」練習をする。話した内容の最初の文がそのまま書き出しになる |
| **何を書けばいいかわからない** | アウトライン（箇条書き）を先に作り、各項目の「見出し」から書く |
| **論理が整理できていない** | 紙に「問い→根拠1→根拠2→結論」を書き出し、可視化する |
| **完璧主義で手が動かない** | 「下書き」と割り切って、とにかく書く（後で直す） |
| **気が乗らない** | 書く時間帯を固定する（脳が「その時間は書く時間」と認識する） |

### 9.2 進捗を維持するコツ

**小さく区切る**: 「論文を書く」ではなく「今日は第2章の第1節を500字書く」という目標を設定する。

**毎日書く**: たとえ100字でも毎日書くことで、論文との距離を縮める。長期間書かないと書き出しが難しくなる。

**書く環境を整える**: 特定の場所（図書館の特定の席など）と結びつけると習慣化しやすい。

**章の順番にこだわらない**: 書きやすい部分から書き始めていい。序章は最後に書く人も多い。

### 9.3 行き詰まった時のチェックリスト

- [ ] 問いが明確に定式化されているか
- [ ] 書こうとしていることが問いへの答えに直結しているか
- [ ] 書けない部分の「なぜ書けないか」を紙に書き出してみたか
- [ ] 指導教員・先輩・友人に相談したか
- [ ] 関連する良い論文を読み直したか（構成・論証の参考に）

## 10. ゼミ・研究室での発表技法

### 10.1 ゼミ発表で評価される点

| 評価ポイント | 良い発表の特徴 |
|------------|-------------|
| **問いの明確さ** | 発表の冒頭に「この研究の問いは〜です」と一文で言える |
| **先行研究の理解** | 自分の研究がどの議論に位置するかを示せる |
| **論理の一貫性** | 問い→方法→結果→結論が一本の線でつながっている |
| **時間管理** | 指定時間内に収まり、重要な点に時間を配分している |
| **質問への対応** | 批判的な質問を防御的にならず建設的に受け止める |

### 10.2 発表後のフィードバックの活用

発表後の質問・コメントは、論文を改善するための貴重なリソース。

```
フィードバック活用の手順:
1. 発表中は質問をメモする（記憶は信用できない）
2. 発表直後に「これはどういう意図の質問だったか」を考える
3. 論文の該当箇所を確認し、修正が必要か判断する
4. 次の発表やゼミで「前回のご指摘を踏まえて〜を修正しました」と伝える
```

## 付録: 執筆スケジュール管理シート（例）

| 週 | 目標 | 実績 | メモ |
|----|------|------|------|
| 第1週 | 第1章序論 草稿 | | |
| 第2週 | 第1章序論 修正版 | | |
| 第3週 | 第2章文献レビュー1節 | | |
| ... | ... | | |

目標は「書く量」より「論文の論理的完成度」で測る。「第2章1節完成（問いとの関係が明確に書けている）」のように。

# わかりやすい文章を書く技法集

## 概要

本ファイルは以下の4冊の知見を統合・再構成したリファレンスである。

- 大久保進『「わかりやすい」文章を書く全技術100』
- 小西利恵子『文章を整える技術 書いたあとのひと手間でぜんぜん違う』
- 藤吉豊・小川真理子『日本人のための「書く」全技術【極み】』
- 三宅香帆『「好き」を言語化する技術』

対象は実用文（説明文・ビジネス文書・レポート・SNS投稿など）。文学的表現とは目的が異なるため、「わかりやすさ・明快さ」を最優先とする。

## 1. 文の要素を明確にする

### 伝わりやすい語句の選び方

| 避けるもの | 具体例 | 対処法 |
|-----------|--------|--------|
| 専門用語 | 「バミっておく」「利食い売り」 | 一般的な言葉に置き換える |
| 難解な漢語 | 「地理的連続性がある」 | 「互いに接していて繋がっている」 |
| 外来語（馴染みのないもの） | 「ブレイクスルー」「スクリーニング」 | 日本語に言い換えるか説明を添える |
| 略語 | 「BtoB」「SEO」 | 初出は正式名称（略語）の形で書く |
| 多義語 | 「適当」「もう一杯」「れる・られる」 | 曖昧さが生じない言葉に置き換える |
| 誤用 | 「すべからく（→すべて）」「役不足（→力不足）」「煮詰まる（→行き詰まる）」 | 正しい意味を確認して使う |

**原則**: 読み手が「知らないかも」と思う表現は、分かりやすい言葉に置き換えるか、直後に括弧で説明を添える。

```
【悪い例】フィヨルド沿岸の町は、海路によって結ばれている。
【良い例】フィヨルド（U字型の谷に、海水が侵入してできた湾）沿岸の町は、海路によって結ばれている。
```

### 指示語の使い方

指示語（これ・それ・あれ・この・その・あの）は誤読の温床になる。

| NG パターン | 改善策 |
|------------|--------|
| 指示語が何を指すか不明 | 指示語の後に指示対象を明記する |
| 指示対象から指示語が遠い | 指示語を指示対象の近くに移動させる |
| 同一対象に異なる言葉を使う | 一貫して同じ言葉で指す |

```
【悪い例】暗黒な天体の数は、光り輝くものの数を凌駕していた。もっとも、これらは必ずしも質量が小さいわけではない。
【良い例】暗黒な天体の数は、光り輝くものの数を凌駕していた。もっとも、これらの暗黒の天体は必ずしも質量が小さいわけではない。
```

### 表記の統一

同音・同義の語に複数の表記が混在すると読み手を混乱させる。

| 表記ゆれの例 | 統一の考え方 |
|-------------|-------------|
| 「私」「わたし」「ボク」 | 一つに統一する |
| 「デイビット」「ディヴィット」「デイヴィット」 | 一つに統一する |
| 「様々」「さまざま」 | 表記ルールを事前に決める |
| 「予め」「あらかじめ」 | 表記ルールを事前に決める |

### 言葉の切れ目を明示する

| 手法 | 使いどころ |
|------|-----------|
| ひらがな・カタカナ・漢字を混ぜる | 「ここではきもの」→「ここで履物を」|
| 読点（、）を打つ | 文節の区切り、誤読が起きやすい箇所 |
| 括弧でくくる | 一単語だと示したい場合（「にしんみそ」）|
| ナカテン（・）で区切る | 並列語・固有名詞の複合語 |

## 2. 文を短くする

**文章上達の最重要ルール**: 一文が長いと理解しにくくなる。短い文章は情報量が少なく、構造も単純になるため、読み手の理解負荷が下がる。

ベストセラー書（『窓ぎわのトットちゃん』『道をひらく』）は一文が短い。20字未満の文が文章全体の約半数を占める。

### 一文の目安

| 媒体・場面 | 目安 |
|-----------|------|
| 一般文書・ビジネスメール | 60字以内を目標 |
| Webコンテンツ | 40〜50字を目標 |
| SNS・短文 | 30字以内を意識 |

スティーブン・キングが実践した公式: **二次稿 = 一次稿 × 0.9**（10%削る）

### 余計な言葉の削り方

「この言葉は伝えたい内容に本当に必要か」を問いかける。

| 削れる表現 | 理由 |
|-----------|------|
| 「ような」（「共通する**ような**面影」） | 「共通する」だけで伝わる |
| 「基本的には」 | 曖昧さを加えるだけ |
| 「の方」（「片付け**の方**を」） | 不要な丁寧語 |
| 「新たに」（「新たに開店した」） | 「開店した」に「新たに」は重複 |
| 「自分の店舗の」（「顧客が奪われた」） | 文脈から明らか |

### 短い言い回しへの変換表

| 長い言い回し | 短い言い回し |
|------------|------------|
| 調査を行う | 調査する |
| 変更が実施された | 変更された |
| ということもあり | のため |
| どのくらい売れたか | 売上 |
| 腹落ちするような気持ちだった | 腹落ちした |
| 豊富に存在する | 豊富にある／豊富だ |
| 物言いをする | 言う |

### 重複語・冗語の変換表

| 重複表現 | 修正後 |
|---------|--------|
| まず最初に | まず／最初に（どちらか一方） |
| 各支社ごとに | 各支社で |
| 頭痛が痛い | 頭痛がする／頭が痛い |
| 日記を日々綴る | 日記を綴る |
| 議決を決める | 議決する |
| 豊富に存在する | 豊富にある |
| 物事にはっきりとした物言いをする | 物事にはっきり言う |

### 接続助詞の制御

接続助詞（ので・が・ながら・たり など）は使いすぎると文が長くなる。

```
【悪い例】
先日依頼いただいた件ですが、社内で検討しているのですが、いまだに対応できずにおりますが、もう少々お待ちいただけますでしょうか。

【良い例】
先日依頼いただいた件です。社内で検討しています。ですが、いまだに対応できずにおります。もう少々お待ちいただけますでしょうか。
```

**特に注意**: 接続助詞「が」は逆接と単純接続の2つの意味を持ち、混乱を招きやすい。単純接続の「が」は句点で区切ると読みやすくなる。

### 中止法の節制

「動詞・形容詞の連用形で文を中止してつなぐ」中止法も使いすぎると文が長くなる。

```
【悪い例】管理職になると、責任が重くなり、部下への気配りも必要になり、超過勤務手当がなくなり、余計な支出も増える。

【良い例】管理職になると、責任が重くなる。部下への気配りも必要になる。また、超過勤務手当がなくなる。そのうえ、余計な支出も増える。
```

## 3. 理解しやすい構造にする

### 主語と述語の距離

主語と述語は近づける。主語と述語の間に長い修飾語が入ると、対応関係が分かりにくくなる。

```
【悪い例】南西よりの強風が、インド洋の島国モルディブの首都マレの普段は静かな砂浜に吹き渡った。
【良い例】インド洋の島国モルディブの首都マレの普段は静かな砂浜に、南西よりの強風が吹き渡った。
```

**主語を省略しない場面**:
- 前の文から主語が変わるとき
- 文の途中で主語が変わるとき

### 修飾語の配置

| ルール | 例 |
|-------|-----|
| 修飾語と被修飾語を近づける | 「赤い大きなリンゴ」の「赤い」「大きな」はどちらも「リンゴ」に近く |
| 長い修飾語を前、短い修飾語を後ろ | 「昨年度から継続している」（長）→「大型」（短）→「プロジェクト」 |
| 大状況から小状況の順 | 「3階（大状況）の、窓側（小状況）の席」 |
| 長い修飾語が2つ以上あるとき | 最初の修飾語の後に読点を打つ |

### 助詞の使い方

| 注意点 | 例 |
|-------|-----|
| 同じ助詞を続けない | 「私の母の友人の話」→「私の母の友人が話した」 |
| 意味がわかりにくい助詞は使わない | 「彼は来月」→「彼は来月に」（「に」を補う） |
| 「てにをは」の不一致を直す | 「私に上司にプレゼンを依頼した」→「上司は私にプレゼンを依頼した」 |

### 並列・同格の表現

| 手法 | 使いどころ |
|------|-----------|
| ナカテン「・」 | 語句の並列（例：「書く・読む・話す」） |
| 読点（、） | 主語＋述語が複数続く場合の区切り |
| 箇条書き | 3つ以上の情報を並列するとき |

### 二重否定・曖昧表現を避ける

| 避けるもの | 理由 |
|-----------|------|
| 二重否定（「なくはない」） | 肯定か否定か判断しにくい |
| 二通りに解釈できる修飾 | 「美しい金髪の少女」（「美しい」は少女にかかるか金髪にかかるか不明）|

## 4. 文と文のつながり

### 接続詞の使い方

接続詞の11種類と使いどころ:

| 種類 | 代表語 | 使いどころ |
|------|-------|-----------|
| 順接 | だから・したがって・すると | 前が原因、後が結果 |
| 逆接 | しかし・だが・ところが | 前と異なる内容が続く |
| 添加 | しかも・加えて・それに | 前に別の内容を付け加える |
| 並列 | また・かつ・および | 前と同等の内容を並べる |
| 対比 | 一方・逆に・反対に | 対比関係を明確にする |
| 選択 | あるいは・もしくは | 選択肢を示す |
| 言換 | つまり・すなわち・要するに | 前の内容を言い換える |
| 説明 | なぜなら・というのは | 前の内容を説明する |
| 補足 | ただし・なお・ちなみに | 補足情報を加える |
| 例示 | たとえば | 具体例を示す |
| 転換 | ところで・では・さて | 話題を変える |

**重要な原則**: 接続詞を省いても意味が通じるなら省く。接続詞を多用すると「くどい」印象になる。特に逆接の直後にまた逆接が来る構造は避ける。

```
【過剰な接続詞の例】
つまりAというメソッドは通用しない。実はBが用いられている。要するにAは時代遅れだ。

【修正後】
Aは通用しない。海外ではBが用いられている。Aは時代遅れだ。
```

### 段落の組み立て

| 原則 | 説明 |
|------|------|
| 近い内容はまとめて書く | トピックが変わるところで段落を変える |
| 一段落に一トピック | 複数の話題を一段落に詰め込まない |
| 段落の最初に結論 | 段落の最初の一文で「何について述べるか」を示す |
| 空白行でメリハリ | 場所・時間・主張の転換時に一行あける |

### 情報の順序

| 構成パターン | 適した場面 |
|------------|-----------|
| 結論→補足 | 報告書・ビジネス文書（最も基本的） |
| 全体→細部 | 概要から詳細に向かう説明 |
| 時系列 | 手順・経緯の説明 |
| 問題→原因→解決策 | 分析・提案文書 |

## 5. 全体構成

### 読者の「知りたい順」に書く

読者が文章を読む主な目的は「知りたいことがある」から。
文章の良し悪し = **内容70% : 書き方15% : 見せ方15%**

| チェック項目 | 内容 |
|------------|------|
| 読者に役立つか | 読者のメリットになる情報を含んでいるか |
| 独自性はあるか | 書き手独自の視点・新しい情報があるか |
| 信頼性はあるか | 客観データ・出典が明記されているか |
| 読者に関係があるか | 「自分にも関係がある」と思ってもらえるか |

### 文章構成の流れ

1. **企画立案**: 何を・誰に・何のために書くかを決める
2. **情報収集**: 正確で新しい情報を集める
3. **情報整理**: 使う情報・使わない情報を取捨選択する
4. **執筆**: 構成を決め、表現を選んで書く
5. **推敲**: 読み返して磨き上げる

### 書き出しのパターン

| パターン | 特徴 |
|---------|------|
| よかった要素の描写から始める | 読者を具体的場面に引き込む |
| 自分語りから始める | 親近感・共感を生む |
| 文脈・背景から始める | 問題の位置づけを先に示す |
| 問いかけから始める | 読者に当事者意識を持たせる |

**原則**: 書き出しは「この文章が何について書かれているか」を予感させること。

### 見出しの設計

- 見出しだけを読んでも内容が分かるようにする
- 見出しと本文の内容が一致しているか確認する
- 概要→見出し→本文の三層構造で「一目でわかる文章」を作る

### 結論を先に述べる

日本語の自然な語順（主語→述語）とは逆に、ビジネス文書では**結論を先に述べる**のが基本。

```
【改善前】
Aという問題があります。Bという原因があります。Cという施策が有効です。
→ 結論は最後まで分からない

【改善後】
Cという施策を提案します。理由はAという問題とBという原因があるからです。
```

## 6. 言語化のテクニック

### 細分化：大きな感想を具体的な要素へ

「やばい」「感動した」「よかった」といった大まかな表現で止まらず、**なぜよかったのかを深掘り**する。

**言語化の3ステップ**:
1. よかった箇所の具体例を挙げる（「あのシーン」「この台詞」）
2. 感情を言語化する（「なぜそれがよかったのか」を問う）
3. 忘れないうちにメモを残す

```
【止まっている状態】 「あのドラマ、よかった。やばかった。」

【細分化後】
「ああいう場面で力みなく台詞を言うのは珍しい。
それがリアルな感情に見えた理由かもしれない。
役とキャラクターの相性が良い気もする。
前回より演技が深くなっている。」
```

### 感情のパターン認識

感動の種類を分解すると、言葉が見つかりやすくなる。

| 感情の種類 | 言葉の手がかり |
|-----------|-------------|
| 共感 | 「自分も同じ経験がある」「わかる」「これが言いたかった」 |
| 驚き | 「こんな見方があったのか」「予想外だった」「意外」 |
| 切なさ | 「なぜか胸が締め付けられる」「どこか懐かしい」 |
| 感銘 | 「これを知れてよかった」「考えが変わった」 |
| 不快 | 「なぜ不快なのか」→ 違和感の根拠を探る |
| 退屈 | 「何が引っかかったのか」→ 期待とのズレを探る |

面白さとは基本的に「共感」か「驚き」のどちらかに分類できる。

### クリシェを避ける

「クリシェ」= 使い古されて本来の意味・新鮮さが失われた表現。

| クリシェ表現 | 問題点 |
|------------|--------|
| 「泣ける」 | 何が感動的だったか伝わらない |
| 「考えさせられた」 | 思考停止の言葉。何を考えたのかが不明 |
| 「やばい」（感想として） | 感情の種類も程度も不明 |
| 「全米が号泣」 | 誇張・常套句化 |

**クリシェを避ける訓練**: クリシェ表現を禁止ワードにして、その感情を別の言葉で表現しようとする。それだけで自分の言葉が生まれ始める。

### 自分の言葉をつくる

他人の感想を見る前に、まず自分の感想をメモに残す。SNSやレビューを先に読むと、他人の言葉が自分の感想を上書きしてしまう。

**もやもやを抱えておく力（ネガティブ・ケイパビリティ）**: すぐに白黒つけようとせず、もやもやした感情をそのまま抱えておくことで、自分の本当の感想が見えてくる。

## 7. 推敲のテクニック

### 推敲が必要な理由

書いている最中は「書くこと」で頭がいっぱいで、客観視できない。接続詞の多用・主述の不一致・表記ゆれなどのミスは書いた直後には気づきにくい。

**村上春樹の方法**: 書き終えたら半月〜1ヶ月は引き出しにしまって忘れる。その後、徹底的に書き直す。

### 初級推敲テク5選

| テクニック | 具体的な方法 |
|-----------|------------|
| 1. 一度忘れてから読み返す | 締切の2日前までに書き終え、翌日以降に読み返す。コーヒー休憩・気分転換だけでも効果あり |
| 2. 音読する | 声に出して読む。長文は息が続かず気づく。文末の重複も耳で気づける |
| 3. 句読点・括弧・一行あけを整える | 読点の位置、かぎかっこの使い分け、段落の切れ目を確認する |
| 4. 漢字・略称・専門用語を見直す | 「読み手が読めるか・わかるか」を基準に、ひらがなやルビへ変換する |
| 5. 表記ゆれを統一する | 同音・同義語の表記が揺れていないかチェックする |

### 中級推敲テク5選

| テクニック | 具体的な方法 |
|-----------|------------|
| 1. 文末のアレンジをストックする | 「〜です」が続いたら体言止め・過去形・助詞止めで変化をつける |
| 2. 一文あたりの最長文字数を決める | 例: 60字・80字などの上限を設けて確認する |
| 3. 「より短い言葉にできるか」を意識する | 一文字でも短くできないかを常に問いかける |
| 4. 「本当に必要な情報か」を疑う | 要素を「必須」「補足可」「不要」の3段階に分類し、不要を削る |
| 5. チェックリストを作成する | 自分の文章のクセ（多用する接続詞・フレーズなど）をリスト化する |

### 推敲の順序

1. **黙読で大まかに確認**: 段落のつながり・脱線・不要な内容を見る
2. **音読で細かく確認**: てにをは・誤字脱字・リズム・特定フレーズの多用を見る
3. **少し時間をおいて再確認**: 少なくとも2回は読み返す

**意地悪な読者になりきる**: 「この部分は分かりにくい」「ここに根拠がない」と批判的な視点で読む。

## 8. 冗長表現の変換表

### 動詞の冗長化パターン

| 冗長な表現 | 短縮後 |
|-----------|--------|
| 検討を行う | 検討する |
| 実施をする | 実施する／する |
| 説明をおこなう | 説明する |
| 対応できずにおります | 対応できておりません |
| 腹落ちするような気持ちだった | 腹落ちした |
| 目が覚めるような思いを持った | 目が覚めた |

### 形容詞・副詞の冗長化パターン

| 冗長な表現 | 短縮後 |
|-----------|--------|
| ような面影 | 面影 |
| 基本的には〜だ | 〜だ |
| 新たに開店した | 開店した |
| 共通するような点 | 共通する点 |

### 接続詞の整理

| 削除できるケース | 理由 |
|----------------|------|
| 「つまり」「実は」「要するに」が続く | 1つで十分 |
| 「だから」の後に「しかし」 | 論理が混乱している |
| 「そして」で単純につなぐだけ | 削除しても意味が通じる |

### Before/After 実例

```
【Before】
その年の健康診断では、過去最悪の結果が出てしまった。だから、それから毎日ジョギングに励んだ。すると、ウエストまわりだけでなく脚や二の腕なども引き締まったような気がした。だが、空腹を覚える時間が増えてしまった。そうして食事量が増えたのだが、しかし、その分揚げ物や炭水化物もたっぷりと摂ってしまったため、その結果として体重は以前よりも数キロ増えてしまった。

【After】
その年の健康診断では、過去最悪の結果が出た。毎日ジョギングに励んだところ、ウエストまわりだけではなく脚や二の腕なども引き締まった気がした。だが、空腹を覚える時間が増え、食事量も増えた。揚げ物や炭水化物もたっぷりと摂ったため、体重は以前よりも数キロ増えてしまった。
```

## 9. チェックリスト

### 執筆前

- [ ] 誰に・何のために・何を書くかが明確か
- [ ] 読者にとって役立つ内容か
- [ ] 情報の出典・信頼性を確認したか

### 文レベル

- [ ] 一文は60字以内を目指しているか（長すぎる文を分割したか）
- [ ] 一文に情報を詰め込みすぎていないか
- [ ] 主語と述語の対応が正しいか
- [ ] 修飾語と被修飾語が近くにあるか
- [ ] 同じ助詞が連続していないか
- [ ] 二重否定・二義的な表現がないか
- [ ] 余計な言葉（「ような」「基本的には」「の方」など）が入っていないか
- [ ] 同じ意味の言葉の重複がないか（「まず最初に」「頭痛が痛い」）

### 語句レベル

- [ ] 専門用語・略語を使う場合、説明を添えているか
- [ ] 多義語が誤解なく伝わるか
- [ ] 指示語（これ・その）の指示対象が明確か
- [ ] 一つの対象を異なる言葉で呼んでいないか
- [ ] 表記ゆれがないか（漢字・ひらがな・カタカナの統一）
- [ ] 誤用していないか（すべからく・役不足・煮詰まるなど）

### 構成・段落レベル

- [ ] 結論が先に来ているか（ビジネス文書の場合）
- [ ] 段落ごとに一つのトピックにまとまっているか
- [ ] 接続詞が多すぎないか（削除しても意味が通じるものを排除）
- [ ] 情報の順序が読者の「知りたい順」に並んでいるか
- [ ] 近い内容がまとまって書かれているか

### 推敲・仕上げ

- [ ] 一度時間をおいて読み返したか
- [ ] 音読して違和感のある箇所がないか
- [ ] 文末表現が単調に繰り返されていないか
- [ ] 句読点の位置は適切か
- [ ] 括弧（かぎ・丸・二重かぎ）の使い方は正しいか
- [ ] 誤字脱字がないか
- [ ] 段落の切れ目（一行あけ）が適切か
- [ ] 出典を明記しているか

## 1. テクニカル・ライティングとは何か

テクニカル・ライティングとは、科学技術的な内容を**論理的かつわかりやすい文章に書く技術**のことである。その根本的な目的は、「簡潔明快」に事実と主張を伝えることにある。

### 文章の二大区分

| 種類 | 目的 | 書き方の特徴 |
|------|------|------------|
| **テクニカルライティング（理系文章・ビジネス文書）** | 正確な情報を効果的に伝える | 簡潔明快、論理的、後の展開が予測しやすい |
| **随筆・小説** | 楽しみ・気分転換を提供する | 情緒的、文体の個性、ドンデン返しも可 |

テクニカル・ライティングでは、**なるべく後の展開を予測しやすい文章**こそが良い文章とされる。

### なぜ今テクニカル・ライティングが重要か

- 仕事で交換される文章量が飛躍的に増加している
- 簡潔明快な文章が書けると、広い範囲の人とスムーズに仕事できる
- 理系の文章とビジネス文書は非常によく似ており、一度身につければ一生の財産になる

## 2. 読者志向の原則

テクニカル・ライティングの最重要原則は「読者のことを考えること」である。

### 読者分析の二つの観点

| 観点 | 内容 |
|------|------|
| **知識** | 読者が何を知っていて、何を知らないか |
| **目的** | 読者はなぜこの文章を読むのか |

### 想定読者の設定方法

実験レポートにおける推奨される想定読者の設定：
- **実際の読者（担当教員）ではなく**、「その実験について読んだり聞いたりする前のクラスメート」を想定して書く
- これにより、知識のない読者にも理解できる記述が実現する

### Before / After 例

| Before（悪い例）| After（良い例）|
|----------------|----------------|
| 「実験原理については教科書のXXページを参照のこと」 | 実験原理を読者が理解できる水準で説明する |
| 「先生はご存知だと思いますが」 | そのような前置きは省き、直接内容を書く |

## 3. 簡潔性・正確性・客観性

### 三原則

| 原則 | 説明 | 実践のポイント |
|------|------|--------------|
| **簡潔性** | 冗長な表現を排除する | 1文に1つの情報を入れる。接続助詞の多用を避ける |
| **正確性** | 曖昧な表現を使わない | 数値で示せるものは数値で。「多い」→「X個（X%）」 |
| **客観性** | 事実と意見を区別する | 推測・判断が入る場合は明示する |

### 主観的表現と客観的表現の比較

| Before（主観的・曖昧）| After（客観的・正確）|
|----------------------|---------------------|
| 「かなり大きな値であった」 | 「XXの値はYY±ZZであり、先行研究のAA（参考文献1）と比較して約BB倍であった」 |
| 「良い結果が得られた」 | 「目的の化合物XXが収率YY%で得られた」 |
| 「図1を見ると、気温が高いことがわかる」 | 「図1に示すように、7月1日の最高気温は35.0℃である」または「7月1日の最高気温は35.0℃である（図1）」 |
| 「放射冷却が気温変化に大きな役割を果たしていると、私は考える」 | 「この結果は、放射冷却の特徴とよく一致している」 |

**重要**: 「図Xを見ると、...がわかる」という表現は避けること。事実を直接述べる表現に置き換える。

## 4. 科学的記述の原則

### 再現可能性の確保

科学的文章の根本は「読んだ者が同じ手順で同じ結果を得られる」記述である。

- 実験方法の記述は**読者が実験を再現できる程度の詳しさ**を目標とする
- 複数の方法がある場合は「どの方法を選んだか」を明記する
  - 例：「5分間加熱する」ではなく「沸騰したお湯に入れて5分間加熱する」

### 定量的記述の徹底

| 定性的（避けるべき）| 定量的（望ましい）|
|--------------------|-----------------|
| 「温度が大きく上昇した」 | 「温度が20℃から45℃に上昇した」 |
| 「多くのサンプルで」 | 「20サンプル中15サンプル（75%）で」 |
| 「かなりの時間が経過した後」 | 「3時間後に」 |

### 事実・考察・感想の区別

これらをごちゃ混ぜにしないことが重要。

| 種類 | 定義 | 書く場所 |
|------|------|---------|
| **事実（結果）** | 測定・観察した客観的データ | 「結果」の節 |
| **考察** | 結果の解釈・先行研究との比較・意義 | 「考察」の節 |
| **感想** | 個人的な印象・難しさの感想 | 「感想」（任意、指定されれば） |

## 5. 数値・単位の表記ルール

### 基本ルール

| ルール | 例 |
|--------|-----|
| 数値と単位の間にスペースを入れる | 25 ℃、100 mL |
| 単位はSI単位系を基本とする | m, kg, s, A, K |
| 有効数字を揃える | 測定精度に合わせて桁数を統一 |
| 不確かさ（誤差）を明記する | 25.3 ± 0.2 ℃ |

### 数値の表記

- 本文中で1桁の数値は漢数字・アラビア数字どちらも可だが、統一する
- 測定値は有効数字に注意：「3.0」と「3.00」は意味が異なる
- 百分率は「%」を使い「パーセント」とは混用しない
- 指数表記：1.23 × 10³（「1.23E3」は数式処理ソフト向け表記）

### 単位の注意点

| 注意事項 | 悪い例 | 良い例 |
|---------|--------|--------|
| SI接頭辞は小文字 | KG（間違い） | kg |
| リットルは大文字も可 | l（紛らわしい） | L または mL |
| 温度は℃またはK | °C（読みにくい） | ℃ |

## 6. 図表の作成ルール

### 図表の役割

図表は**それ自身と説明文（キャプション）だけで内容が理解できる**ように作成する。

### 図（グラフ・写真・模式図）

| 項目 | ルール |
|------|--------|
| **キャプション位置** | 図の**下**に記載 |
| **番号** | 「図1」「図2」と通し番号 |
| **軸ラベル** | 物理量の名称と単位を必ず記載（例：温度 [℃]、時間 [s]） |
| **凡例** | 複数データを重ねる場合は必ず凡例を付ける |
| **出典** | 引用・転載した場合は出典を明記 |

### 表

| 項目 | ルール |
|------|--------|
| **キャプション位置** | 表の**上**に記載 |
| **番号** | 「表1」「表2」と通し番号 |
| **単位** | 列ヘッダーに単位を記載（例：温度 [℃]） |
| **有効数字** | 列内で揃える |

### 本文での図表の参照

- 図表は本文で必ず参照する：「図1に示すように」「表2に示した」
- 「図1の通り」だけでは不十分。本文で**特徴を述べる**こと
- 「図1は○○の図である」だけで特徴を書かないのは不適切

### 図表の説明文の書き方

```
【悪い例】
図1 実験結果

【良い例】
図1 気温の日変化（2023年7月1日、東京）。実線は晴天時、破線は曇天時を示す。
エラーバーは標準偏差（n=3）を示す。
```

## 7. 科学用語の使い方

### 初出時の定義

専門用語・略語は**初出時に定義（展開）**してから以降は略称を使う。

```
【良い例】
本研究では、遺伝子発現プロファイリング（Gene Expression Profiling, GEP）を用いた。
GEPは...（以降は「GEP」で統一）

【悪い例】
本研究ではGEPを用いた。（初出で説明なし）
```

### 用語の一貫性

一度使った術語は全文で統一する。

| 注意点 | 悪い例 | 良い例 |
|--------|--------|--------|
| 同じ概念に複数の言葉を使わない | 「反応率」と「収率」が混在 | どちらかに統一する |
| 和語・外来語の混用に注意 | 「コンピュータ」と「計算機」が混在 | どちらかに統一 |

### 確実性の表現

科学的記述では、確実性の程度を適切に表現することが重要。

| 表現 | 使う場面 |
|------|---------|
| 「〜である」「〜が示された」 | 直接測定・観察した事実 |
| 「〜と考えられる」「〜と思われる」 | 解釈・推測 |
| 「〜の可能性がある」「〜が示唆される」 | 間接的な証拠からの推測 |
| 「〜という仮説を立てた」 | 検証前の仮説 |

## 8. 論理と接続表現

### 接続表現の正しい使い方

接続表現は文と文のつながりを示す。使い方を間違えると論理が崩れる。

| 接続表現 | 意味・使い方 | 例 |
|---------|------------|-----|
| **したがって**・**ゆえに** | 原因→結果の論理的帰結 | 「温度が上昇した。したがって、反応速度が増加した」 |
| **一方**・**これに対し** | 対比・比較（前後対等） | 「Aは増加した。一方、Bは減少した」 |
| **しかし**・**ところが** | 逆接（後の文に重みがある） | 「高い収率が期待された。しかし、実際は低収率であった」 |
| **ただし** | 前の文に重みを置きつつ条件・例外を示す | 「高い収率が得られた。ただし、温度条件が20℃の場合に限る」 |
| **また** | 付加・並列 | 「高い収率が得られた。また、純度も高かった」 |
| **さらに** | 強調付加（程度が増す） | 「高い収率が得られた。さらに、従来法より速かった」 |

### 避けるべき論理的な誤り

1. **接続詞の不適切な使用**: 「原因と結果」の関係がないのに「したがって」を使う
2. **根拠なしの断言**: 「〜に違いない」「〜は明らかである」を証拠なしに使う
3. **接続のしすぎ**: 8つ以上の主張を連鎖させると、全体の確率が著しく低下する

## 9. 既知から未知への構成

### 「ワイングラス」モデル

理系論文の全体構成はワイングラスの形に例えられる。

```
「はじめに」  ─── 広い（一般から個別へ）
     |
     |  ホップ：大きな研究領域の重要性
     |  ステップ：未解決の個別問題の提示
     |  ジャンプ：本研究の目的と手段
     |
「方法・結果」 ─── 狭い（個別の事実）
     |
     |  具体的なデータ・手法
     |
「考察」      ─── 広い（個別から一般へ）
              先行研究との比較、学術的意義
```

### 段落・文の構成の原則

- 各段落の**最初の文（トピック・センテンス）**にその段落の主題を書く
- 既知の情報→新しい情報の順で書く
- 各文が前の文を受けて次の文への橋渡しをする

### 「はじめに」の三段階構成

| 段階 | 内容 | 文の方向性 |
|------|------|-----------|
| **ホップ** | より大きい研究領域の重要性を主張 | 一般的・広い |
| **ステップ** | 研究が不十分な個別テーマを提示 | 「しかし〜が行われていない」 |
| **ジャンプ** | 本研究の目的と手段の概要 | 具体的・狭い |

## 10. チェックリスト

### テクニカル・ライティング最終チェック

**読者・目的**
- [ ] 想定読者の知識レベルに合わせて記述されているか
- [ ] 読者の目的に応えているか

**簡潔性**
- [ ] 冗長な表現は削除されているか
- [ ] 1文に1つの情報が収まっているか
- [ ] 不必要な修飾語・接続助詞は削除されているか

**正確性**
- [ ] 定量的記述になっているか（数値・単位が正確に記載されているか）
- [ ] 有効数字は揃っているか
- [ ] 単位はSI単位系で統一されているか

**客観性**
- [ ] 事実と推測・考察が区別されているか
- [ ] 「私は〜と考える」を「結果」節に書いていないか
- [ ] 「図Xを見るとわかる」という表現を使っていないか

**図表**
- [ ] 図キャプションは図の下にあるか
- [ ] 表のキャプションは表の上にあるか
- [ ] 軸ラベルに単位が記載されているか
- [ ] 図表はそれ単体で意味が理解できるか
- [ ] 本文で図表を参照し、特徴を述べているか

**用語**
- [ ] 専門用語・略語は初出で定義されているか
- [ ] 同じ概念に同じ言葉を使っているか

**論理**
- [ ] 接続表現は論理的関係を正確に示しているか
- [ ] 各段落の最初の文に主題が書かれているか
- [ ] 構成は「一般→個別（はじめに）」「個別→一般（考察）」になっているか

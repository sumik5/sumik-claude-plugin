## 1. 剽窃（ひょうせつ）とは何か

### 定義

剽窃とは、**他者の文章・アイデア・データを、出典を示さずに自分の著作として発表すること**である。

アカデミックな世界（学術・教育）において、剽窃は最も深刻な不誠実行為の一つとみなされる。

### 剽窃の四類型

| 類型 | 内容 |
|------|------|
| **丸写し（コピペ）** | 他者の文章をすべてまたは一部コピーして、自分の書いたものとして提出する |
| **自己剽窃** | 自分が過去に書いたものを、新たな著作であるかのように再提出する |
| **無断借用** | 重要な論点やアイデアを、出典を示さずに他の著作から使う |
| **モザイク剽窃** | 複数の文献の文章をつなぎ合わせ、多少変えて提出する（出典を示しても問題が残る場合あり）|

## 2. なぜ剽窃がいけないのか

### 学術的誠実性の観点

アカデミックな世界の基本ルール：

> 「人がそれなりの努力を傾注して調べたり考えたりして到達した真理・知識は、人類すべてのものとして共有されるべきである。しかし、その代わりに、それを生みだした人にはそれ相当の尊敬が払われなければならない。」

剽窃はこのルールに正面から違反する。

### 自分自身を損なう観点

論文（レポート）を書くことの目的は**自分の思考力・文章力を鍛えること**にある。丸写しすることで、以下の機会を自ら放棄することになる：

- 問題を分析し、自分の考えを形成するプロセス
- それを文章にまとめる修業のプロセス
- これらを通じて自分を賢くするチャンス

### 実際上の危険性

剽窃は**ほぼ確実にバレる**：

1. 教員は毎年多数の論文を読んでいるため、学生の文体と異なる文章を識別できる
2. 同じ文献に複数の学生が行き着くため、複数の論文が同じ文章になりやすい
3. 教員も検索エンジンで容易にソースを見つけられる
4. iThenticateなどの剽窃検出ソフトウェアを導入する大学が増加している

## 3. 剽窃になる行為の具体例

### コピペ（ネットからの無断転載）

```
【剽窃の例】
インターネット上の記事の文章をそのままコピーして、
自分の論文として提出する。

→ 出典を明示すれば引用として扱える。ただし、引用のルール
（短い引用はカギ括弧、長い引用はインデント、出典表示）に従うこと。
```

### 出典なしの要約

```
【剽窃の例】
「ある研究によれば、日本の大学生の約60%がXXを経験している」と
書いたが、どの研究か出典を示さない。

→ 自分の調査・研究でない限り、必ず出典を示す必要がある。
「ある研究によれば」という曖昧な表現は剽窃に当たる。
```

### 他人のアイデアの無断使用

```
【剽窃の例】
ある論文の独自のフレームワークや分類体系をそのまま使い、
「○○という3つの要素に分類できる」と書いたが、出典を示さない。

→ アイデア・枠組み・分類体系も「他者の知的財産」。
必ず出典を示すこと。
```

### 自己剽窃

```
【剽窃の例】
前の学期のレポートを、別の授業のレポートとして再提出する。
または、前の学期のレポートの一部を流用して新しいレポートを作成する。

→ 自分の過去の著作も、新しい著作として提出することは認められない。
どうしても以前の内容を参照したい場合は、自分の名前で出典として引用する。
```

## 4. 適切な引用の方法

剽窃を避けるためには、**自分の記述と他者の記述を明確に区別**することが必要。

### 判断の基準

| 状況 | 対応 |
|------|------|
| 自分の考え・発見 | そのまま書く（出典不要）|
| 他者の事実・データ・考え | 必ず出典を示す（直接引用または間接引用）|
| 常識的に知られた事実（例：「太陽は東から昇る」）| 出典不要 |
| 専門分野で広く共有された定説（例：「Einsteinの相対性理論」）| 出典不要（ただし特定の文献の主張を取り上げる場合は必要）|

### 引用の正式な手順

```
1. 引用する箇所を特定する

2. 引用の種類を選ぶ:
   - 短い（2行以内）→ カギ括弧「」でくくって本文に埋め込む
   - 長い（3行以上）→ インデントして独立した引用ブロックにする
   - 内容だけ → 間接引用（パラフレーズ）

3. 出典を明示する:
   - 著者・年・ページ番号（文中注方式）
   または
   - 注番号（脚注・後注方式）

4. 参考文献リストに完全な情報を記載する
```

### 引用のルールまとめ

| ルール | 内容 |
|--------|------|
| 短い引用 | カギ括弧「」の中に原文を入れる |
| 長い引用 | インデント（2〜5字）して原文を示す |
| 省略 | （中略）を使う |
| 加筆 | （括弧内引用者）と注記する |
| 出典 | 著者・年・ページを必ず示す |
| 参考文献リスト | 文末に完全な文献情報を掲載する |

## 5. パラフレーズの技法

### パラフレーズとは

他者の記述の**内容（意味）を維持しながら、表現を自分の言葉に変換すること**。

### パラフレーズが必要な理由

- 原文を直接引用すると論文が断片的になる
- 原文の内容は重要だが表現は自分のものにしたい
- 論文全体の文体を統一したい

### 正しいパラフレーズの手順

1. 原文をよく読んで**完全に理解**する
2. 原文を閉じる（見ないようにする）
3. **自分の言葉**で内容を書く
4. 原文と照合して意味がずれていないか確認する
5. **出典を示す**（パラフレーズでも出典は必要）

### パラフレーズのBefore / After

**原文:**
```
「学術研究は、他人がすでに明らかにしてくれた事実の莫大な蓄積の上に、
ちょこっと、自分がはじめて明らかにしたことを付け加えることによって進む」
```

**不適切なパラフレーズ（単語を入れ替えただけ）:**
```
学術研究は、他の研究者が以前に解明した知識の大きな積み重ねの上に、
少し、自分が初めて発見したことを追加することで発展する。
（→ これはほぼ丸写し。出典を示しても問題が残る）
```

**適切なパラフレーズ（自分の言葉で内容を再構成）:**
```
研究における「独自性（オリジナリティ）」とは、先行研究の積み重ねを
前提にした上での小さな前進のことである（著者名 年, p.XX）。
```

### パラフレーズ後も出典は必須

自分の言葉で書き換えても、アイデアは他者のものである。必ず出典を示すこと。

## 6. 引用と参照の区別

### 「引用」と「参照」の違い

| 種類 | 意味 | 表示方法 |
|------|------|---------|
| **直接引用** | 原文をそのまま転載する | カギ括弧またはインデント＋出典 |
| **間接引用（パラフレーズ）** | 内容を自分の言葉で書く | 地の文＋出典 |
| **参照** | 内容は書かずに出典だけ示す | 「〜については（著者 年）参照」 |

### 参照の使い方

```
【参照の例】
この問題については多くの先行研究がある（Smith 2020; Tanaka 2021; Lee 2022）。

XXXの測定方法については（見延 2003, pp. 87-94）に詳述されている。
```

## 7. AI生成テキストと剽窃の新しい問題

### ChatGPT時代の新しい課題

AI生成テキスト（ChatGPTなどのLLMが生成した文章）の使用は、学術界では急速に問題化している。

### AI生成テキストの主な問題点

| 問題点 | 内容 |
|--------|------|
| **自分の思考の代替** | AIが書いた文章は自分の思考プロセスを経ていない |
| **学習機会の喪失** | 書くことで自分が育つ機会を失う |
| **誤情報のリスク** | AIは「ハルシネーション」（もっともらしい嘘）を生成することがある |
| **出典の不在** | AI生成テキストに引用された文献・データは実在しない場合がある |
| **著作権の曖昧さ** | AIは学習データから学んでいるが、その境界が明確でない |

### 大学・教育機関の対応動向（2024年時点）

- **全面禁止**: AI使用を一切認めない
- **部分的許可**: アイデア出しや翻訳補助などには許可、文章の執筆は不可
- **開示義務**: AI使用を開示すれば許可
- **ルールなし**: 教員・機関によって対応が異なる

**重要**: 課題を提出する際は、**提出先の機関・教員のルールを必ず確認**すること。

### AIを「使う」と「頼る」の違い

| 使い方 | 判断 |
|--------|------|
| アイデアのブレインストーミングに使う | 多くの場合許容される（機関のルール確認必要） |
| 文章の骨格を作らせ、自分で大幅に書き直す | 機関による（開示が必要なことが多い） |
| AIが生成した文章を最小限の修正で提出する | 多くの機関で禁止されている可能性が高い |
| AIが生成した文章をそのまま提出する | ほぼすべての機関で禁止 |

### AI使用の開示方法（使用が許可されている場合）

```
【開示の例】
本稿の下書き作成にはChatGPT（OpenAI、2024年3月版）を補助的に使用した。
生成されたテキストはすべて著者が確認・修正・加筆しており、内容の責任は
著者が負う。
```

## 8. 剽窃チェックの実践

### 提出前のセルフチェックリスト

**引用のチェック**
- [ ] 他者の文章をそのまま使っている箇所はカギ括弧またはインデントで示したか
- [ ] すべての直接引用に出典（著者・年・ページ）が示されているか
- [ ] 間接引用・パラフレーズにも出典が示されているか
- [ ] 参考文献リストに全文献が記載されているか

**自分の記述と他者の記述の区別**
- [ ] 「〜である」「〜した」と断言している内容は、自分の調査・実験・論証によるものか
- [ ] 他者のアイデア・発見・データを使っている箇所に出典があるか
- [ ] 「常識」と「要出典の事実」を区別できているか

**AI使用チェック（使用した場合）**
- [ ] 提出先機関のAI使用ポリシーを確認したか
- [ ] AIが生成した内容の事実確認を自分で行ったか
- [ ] 必要な場合、AI使用を開示したか

### 剽窃検出の仕組みを知る

教員・査読者はどのように剽窃を見つけるか：

1. **文体の違和感**: 学生らしくない高度な語彙・表現、突然変わる文体
2. **キーワード検索**: 不自然な言い回しを検索すると原典が見つかる
3. **複数の同一文章**: 同じ課題で複数学生が同じ文献から引用した場合に一致
4. **専用ソフト**: Turnitin、iThenticate などの剽窃検出ソフトウェア

### 「剽窃にならないようにする」のではなく「正しく引用する」

ゴールは「バレない剽窃」ではなく「適切な引用によって自分のオリジナルな貢献を明確にすること」。

引用のルールを正しく守れば：
- つまらない内容であっても「ただのつまらない論文」ですむ
- 引用・参考文献のルールに違反すると「盗作（剽窃）」と判断される

正しい引用は**学術的誠実性（Academic Integrity）の基盤**である。

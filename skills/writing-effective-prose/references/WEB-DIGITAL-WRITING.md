# Web・デジタル媒体向けライティング リファレンス

## 1. デジタルライティングの前提

### 「読者は文章の20%しか読まない」

ウェブユーザビリティ研究によれば、2,400文字の文章のうち実際に読まれるのは480文字程度に過ぎない。スマートフォンの普及以降、この割合はさらに低下していると考えられる。

- 人間の脳は省エネ設計のため、積極的に読むことを避ける
- デジタル化による情報量の爆発的増加が、集中力をさらに低下させている
- 読み手はスクロールしながら「スキャン」している

### スキャンリーディング前提で構成する

読み手は文章を読むのではなく、**見て**情報を取得する。

- 視線は「F字型」に動く（上→右→下→右）
- 最初の数行で離脱するかどうかを判断する
- 見出し・太字・箇条書きだけで内容が把握できる構成が理想

### 「読んでもらえない」前提で書く

「自分の文章は読まれる」という前提を捨てることが出発点。

- コピーライターの基本姿勢: 広告は誰も読みたくない
- 読まれない前提で、どう伝えるかを考える
- 読み手の負担を最小化するホスピタリティとして文章術を捉える

## 2. 読み手ファースト 7つのルール

| # | ルール | 要点 |
|---|--------|------|
| 1 | 見た目が9割 | レイアウト・改行・余白の最適化。行間・改行・段落分けが三種の神器 |
| 2 | 短く書く | 1文は短く（40〜60文字目安）、段落も短く。短さは読みやすさに直結する |
| 3 | リズムが命 | 文の長短に変化をつける。書いたら音読してリズムを確認する |
| 4 | シンプルに | 複雑さを排除、1メッセージ1段落（ワンセンテンス・ワンメッセージ） |
| 5 | 要点を先に | 結論ファースト（逆ピラミッド構造・アンサーファースト） |
| 6 | 視覚的に整理 | 箇条書き・表・太字で重要点を強調。箇条書きが最強の整理手段 |
| 7 | アクションを明確に | 読者が次に何をすべきかを示す |

### ルール1: 見た目が9割

文章の内容より先に、見た目が読み手の「読む/読まない」を決める。

- 行間・改行・段落分けを適切に設定する
- 余計な強調・装飾をしない（装飾が多すぎると何も強調されない）
- 強調ルールを統一する（太字/下線/色の使い分けをルール化）

### ルール2: 短く書く

短い文章のほうが読み手のクリック率・完読率が高いことはA/Bテストで繰り返し実証されている。

**短くするための2つのアプローチ:**

1. **「何を言うか」を絞る**: 訴求ポイントを削る（多くても3つ）
2. **「どう言うか」を改善する**: 同じ内容をより少ない文字で伝える

**内容を変えずに短くする4テクニック:**
- 一文を短くする（ワンセンテンス・ワンメッセージ）
- 短い言葉に言い換える
- 体言止めを活用する
- 丁寧すぎる言葉づかいを避ける

### ルール3: リズムが命

リズムの悪い文章は読み手を疲れさせる。

- 短文と長文を意図的に混ぜる
- 書いたら必ず音読してリズムを確認する
- 読点は「読みやすい見た目になるように」「音読して息継ぎする場所に」打つ

### ルール4: シンプルに

> 「シンプルさはエレガンスの鍵」

複雑な文章は読み手の認知負荷を高める。

- 1段落に1つのメッセージ
- 横文字・専門用語は必要最小限に（横文字禁止令: 日本語で言える場合は日本語を使う）
- ひらがな7割・漢字3割が黄金比（「こと」「もの」「ください」はひらがなに）

### ルール5: 要点を先に

読み手は最初の数行で離脱するかどうかを決める。

- **逆ピラミッド構造**: 重要度の高い情報から順に書く
- 「Fの形」で読まれることを意識する（上部に最重要情報を置く）
- 「突然ですが」「まず初めに」などの前置きは削除する
- あいさつや自己紹介から始めない

### ルール6: 視覚的に整理

箇条書きは文章をもっとも読みやすくする道具。

- 並列する項目は箇条書きに変換する
- 表を使って比較・整理する
- 強調（太字）は1段落に1つまで

### ルール7: アクションを明確に

読み手に「次に何をすればよいか」を明示する。

- CTA（Call to Action）を明確に配置する
- 「詳しくはこちら」は禁止（具体的な動詞で書く）
- 読み手にとってのメリットを示してから行動を促す

## 3. UXライティングの原則

UXライティングはユーザーの体験を設計する文章。UIテキスト・エラーメッセージ・ヘルプテキストなど、デジタルプロダクトのあらゆる文字情報が対象。

### 機能的ライティングと情緒的ライティング

| 種類 | 目的 | 例 |
|------|------|-----|
| 機能的ライティング | 情報を正確に伝える | エラーメッセージ、ヘルプテキスト、操作ガイド |
| 情緒的ライティング | 感情に訴える | マーケティングコピー、ブランドメッセージ |

ほとんどのビジネス文書に求められるのは機能的ライティング。「エモい文章」「バズる文章」が必要な場面は限られている。

### UXライティングの基本原則

- **お客様の本当の目的を言語化する**: 機能の名前ではなく、ユーザーが達成したいことを言葉にする
- **記憶に残らない文章の技術**: 読み手がストレスなく操作・理解できることが最高の体験
- **文字数の目安**: タイトル・キャッチフレーズは全角15.5文字以内（Yahoo! ニュースの見出しに基づく研究結果）

## 4. PREP法（論理的構成）

ビジネス文書・ブログ記事・プレゼンなど、説得力が求められる文章に使う基本フレームワーク。

| ステップ | 内容 | 説明 |
|---------|------|------|
| P（Point） | 結論 | 最も伝えたいことを冒頭に置く |
| R（Reason） | 理由 | なぜそう言えるのかを示す |
| E（Example） | 具体例 | 数値・事例・エピソードで裏付ける |
| P（Point） | 結論再述 | 冒頭の主張を繰り返し、記憶に定着させる |

**適用例（記事本文の各セクション）:**

```
【結論】本ツールを使うと作業時間が50%削減できます。
【理由】手動で行っていた3ステップが自動化されるためです。
【具体例】A社では導入後、月20時間の工数削減を達成しました。
【結論再述】このように、作業効率を大幅に改善できます。
```

## 5. 「ポイントは3つ」の法則

訴求点は3つに絞ることが最も記憶に残りやすい。

**なぜ3つか:**
- 人間の短期記憶に楽に保持できる項目数は3〜4つ
- 5つ以上になると受け取る側がキャッチできなくなる（ジョブズの「5つの玉」の逸話）
- 「3」は文化的・生物的に強い構造を持つ

**実践方法:**
- 伝えたいことが多い場合は「最も重要な3つ」に絞り込む
- 見出しも3つ程度を目安にする
- 「ポイントは3つです」と最初に宣言すると読み手の理解度が上がる

## 6. 強調・装飾のルール

装飾は「情報の重要度を視覚的に伝えるためのルール」として統一する。

### 強調ルールの一例

| 強調方法 | 使用場面 | 1段落あたりの上限 |
|---------|---------|-----------------|
| **太字** | 最重要の単語・フレーズ | 1箇所 |
| 下線 | リンクのみ（本文中の強調には使わない） | - |
| 色 | 警告・注意（赤系）、ポジティブ（緑系） | 統一して使う |

### 強調の失敗パターン

- 1段落に太字が多すぎる → 何も強調されていないのと同じ
- 複数の強調方法を混用 → ルールなく使うと視覚的なノイズになる
- 装飾で文章の質を補おうとする → 装飾は内容の補助であり代替にはならない

## 7. SEO意識のライティング（基本）

SEOとユーザー体験は矛盾しない。ユーザーの検索意図に応える良質なコンテンツが検索順位を上げる。

### 基本チェックリスト

- **タイトル**: SEOキーワードを含め、32〜40文字程度で具体的に
- **メタディスクリプション**: 120文字以内で記事の要点と読む理由を
- **見出し（H2/H3）**: 結論がわかる見出しにする。キーワードを自然に含める
- **本文**: 読者の検索意図（何を知りたいか・何をしたいか）に正面から答える

### 検索意図に応える文章

読者の検索意図は主に3種類:

| 検索意図 | 特徴 | 対応する文章 |
|---------|------|------------|
| 知りたい（情報収集） | 「〇〇とは」「〇〇の方法」 | 網羅的な説明・解説 |
| したい（課題解決） | 「〇〇できない」「〇〇するには」 | 手順・解決策の提示 |
| 買いたい（比較検討） | 「〇〇 おすすめ」「〇〇 比較」 | 比較・レビュー・理由の提示 |

## 8. 推敲サイクル

デジタルでも推敲は必須。「書き上げ→寝かせる→読み返す」のサイクルが品質を高める。

### 推敲チェックリスト（デジタル向け）

**構成レベル**
- [ ] 結論が冒頭に来ているか
- [ ] 訴求ポイントは3つ以内に絞られているか
- [ ] PREP法の構造になっているか

**文章レベル**
- [ ] 1文が長すぎないか（40〜60文字の目安）
- [ ] 一文一メッセージになっているか
- [ ] 前置き・あいさつから始まっていないか
- [ ] 横文字・専門用語は必要最低限か

**視覚レベル**
- [ ] 箇条書き・表を適切に使っているか
- [ ] 強調は1段落1箇所以内か
- [ ] スマートフォンでの表示を確認したか

### 「一晩寝かせる」の効果

書き上げた直後は自分の意図が先行して、読み手の視点を失いやすい。時間を置いて読み直すと：
- 前提を共有していない読み手が迷う箇所が見えてくる
- 冗長な表現・重複した内容が目につく
- 「本当に伝えたいこと」が明確になる
